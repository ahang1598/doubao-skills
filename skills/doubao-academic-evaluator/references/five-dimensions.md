# 五维打分框架

这个框架来自给强基线生成渐进式想法的方法论：不要拿着解法去找新问题，而是拿一个定义清楚的基线，问它在五个方向上哪个还能往上推。每个方向都有典型的入手策略和具体例子。反过来用，它就成了评判的透镜：给一个想法，看它在这五维上各自把现状推进了多少。强想法通常在某一维上压倒性地强，再在至少一维上站得住；弱想法在四维以上都含糊。

## 五个维度

### 更准（Higher）：效果、准确率、质量
入手策略：
- **信号/模态增广**：基线是不是忽略了一个本可以帮上忙的输入信号？（Text-to-SQL 只用 schema 加问题，喂进数据分布或领域文档后可能大幅提升。）
- **反馈驱动的精修**：执行反馈（编译错误、运行异常、测试失败）能不能驱动自我修正？（把执行环境的报错返回给 agent 做反思，代码生成成功率会跳升。）
- **错误根因打靶**：跑基线、聚类失败、找出主导失败模式、专门做一个模块去打它。（如果 Text-to-SQL 的失败集中在复杂 JOIN，就设计一个分解 JOIN 的子 agent。）

打分：9-10 = 提出一个无人用过的、有原理的新信号或反馈回路，有望带来多个点的准确率提升；6-8 = 把已知信号或机制用到新设定，提升不大但真实；3-5 = 普通的 prompt 或微调变体，提升多半淹没在噪声里；1-2 = 看不出对效果有任何推进。

### 更快（Faster）：时间、token、显存、算力
入手策略：
- **缓存与经验复用**：缓存成功的轨迹，重复任务不必从头重规划。
- **并行与解耦**：把长串行流水线拆成独立子任务，交给并行的专门 agent。
- **早退与动态路由**：简单的走便宜模型，只有难的才升级到大模型。

打分：9-10 = 找到一个大且量化清楚的效率缺口，用有原理的机制把它关上（3 倍以上加速且不掉效果）；6-8 = 针对已知瓶颈、机制可信（1.5 到 3 倍）；3-5 = 提了效率但没有具体机制；1-2 = 不涉及效率。

### 更稳（Stronger）：鲁棒性与泛化
入手策略：
- **噪声/容错**：优雅地处理畸形或歧义输入。（意图澄清模块，遇到歧义时反问用户而非静默失败。）
- **异常恢复**：检测 API 失败、意外输出、工具崩溃，换备选重试。
- **表示解耦**：把通用推理和领域知识分开，让同一个推理模块零样本迁移到新领域。

打分：9-10 = 直击一个已知的脆弱点（歧义、分布外、域偏移），有原理的机制，且计划做跨域评测；6-8 = 提了机制但只在单域评测；3-5 = 嘴上说鲁棒、没机制；1-2 = 不涉及。

### 更省（Cheaper）：数据、训练、部署成本
入手策略：
- **LLM 数据合成**：用强基座模型合成带标数据或模拟轨迹，减少人工标注。
- **带人在环的主动学习**：挑最小的、最能提升模型的样本给人审，而不是标全量。
- **知识蒸馏**：把大模型的推理路径蒸到一个小的可部署模型里，保住精度、降低推理成本。

打分：9-10 = 以朴素标注的零头成本做出高质量数据集，或以低一个数量级的成本部署；6-8 = 用已知机制把成本降到 1/2 到 1/5；3-5 = 提了降本但没量化；1-2 = 不考虑成本。

### 更广（Broader）：跨域与统一
入手策略：
- **跨域移植**：把一个领域的成熟技术搬到另一个领域。（数据库查询优化器的成本估计和计划缓存，搬到 LLM agent 的规划上。）
- **抽象与统一**：把一族任务（Text-to-SQL、Text-to-Python、Text-to-Chart）统一到一个共享底层结构的框架下。

打分：9-10 = 非平凡方向上移植一个成熟技术，或统一三个以上原本零散的任务族；6-8 = 移植一个已知想法，或统一两个任务族；3-5 = 只提到和别的领域有关联、没有具体机制；1-2 = 孤立。

## 打分纪律

- **默认每维 5 分**，往上加分必须有依据。
- 加分的依据有两种都算：对方给出的**实测结果/数据**（引他原话），或**讲得通的机制论证**（标明"基于机制，未经数据证实"）。
- 别为了显得慷慨全打 7、8（信号毁了）；也别为了显得严格把机制扎实的强项压到 7（错杀）。
- 某一维既无数据又无机制论证，封在 5，写明"未给依据"。
- 提升建议要点名上面的某个入手策略，"换个 prompt 试试"不算建议。

## 怎么聚合

- 最高维到了 8、第二维到了 6 以上：这两维就是论文的论点，建议引言重点突出。
- 三维以上都在 5 或以下：想法偏薄，要么找出它真正发光的那一维去磨，要么换方向。
- 五维全在 4 或以下：换个方向。
- 没有任何一维上得了 7：想法太含糊，先请对方指出最强的那一维再重评。

## 关于归因（最容易出错的地方）

如果一个亮眼的结果可能其实来自外围环节，比如某个路由步骤、某步后处理、挑过的样本、更强的底座、换了基线，而不是声称的核心机制，那这个增益就还没被证实。把对应维度封住，直到一个消融实验证明确实是核心机制本身在起作用。这一条各学科通用：化学里产率提升可能其实来自多加的一步纯化（而非新催化剂）；社科里的效应可能其实由样本选取驱动（而非声称的干预）。处理这类的办法都一样：定位"去掉那一步/那一选择后，核心机制还剩多少"，剩不下，归因就不成立。
