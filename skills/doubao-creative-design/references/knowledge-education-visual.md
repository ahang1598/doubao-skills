# 知识教育视觉 Prompt 改写

## 主 Skill 路由适配

本 ref 只提供知识教育视觉物料的产物规则。被 `doubao-creative-design` 路由调用时，工具协议、澄清轮次、核心资产记录、张数策略和 prompt 总规则以主 `SKILL.md` 为准；本文件只补充知识结构判断、教育图表类型选择、可读性规范、学科准确性和知识图文字红线。

## 产物 Skill 边界

知识教育视觉是独立产物 ref。它的目标是把知识内容转化为清晰、准确、可教学的视觉图像，包括知识科普海报、教材插图、课件配图、思维导图、知识图谱、流程图、科学结构图、数据图表、工程示意图、公式推导图、受力分析图、电路图和知识图优化。不要把知识教育图扩展成品牌全案、电商商品图、营销封面或通用插画；如果用户的核心目标是点击传播、商品转化或品牌触点，应回到主 `SKILL.md` 的路由冲突规则重新选择 ref。

## 概述

本技能用于知识教育场景的工具选择与高质量 Prompt 改写，解决用户需求过短、知识结构不完整、画面文字不清晰、工具选择不准确的问题。

核心原则：
- 只做知识教育视觉内容的工具路由与 Prompt 扩写，不处理通用营销、电商、品牌、娱乐图片。
- Prompt 必须扩写成完整设计稿描述，不能只写关键词。
- 用户只给主题时，主动补齐标题、结构、节点、步骤、说明文字、配色、字体、构图和质量要求。
- 所有需要出现在画面中的文字必须使用中文双引号“”包裹。
- 最终输出应能直接传入对应工具，不暴露内部判断过程。

## 使用场景

应该触发：
- 用户要生成或编辑知识教育图像：知识科普海报、科普长图、健康科普图、操作指南海报、产业链科普图、逻辑关系图、科学结构图、数据可视化图、工程图纸、时空演化图、UI/票据/凭证教学图、日常知识图、艺术文化图、理科计算图、教材插图、课件配图。
- 用户给出简短主题，希望扩写为可生成图片的 Prompt，例如“做英语语法思维导图”“做请假流程图”“做一张光合作用知识卡片”。
- 用户上传知识图、课件图、草图、截图，希望纠错、重绘、补节点、改结构、提高清晰度，优化为更准确、更专业、更适合教学的图像。
- 用户希望把知识图变成动态视频，例如节点逐步点亮、流程箭头流动、公式分步显现、时间轴推进。

不应该触发：
- 用户只要纯文字讲解、教案撰写、题目解析，不需要图像或视频 Prompt。
- 用户要做电商主图、品牌 VI、Logo、包装、社媒营销图、人物写真、通用海报。
- 用户要求伪造证件、成绩单、公章、票据、真实政治人物、侵权 IP、平台水印或商标。

## 核心流程

1. 理解用户意图  
   先判断用户真正要解决的知识表达任务，而不是直接照抄用户说的图型。需要识别用户是要讲清概念、梳理分类、表达关系、展示流程、解释结构、呈现数据、展示时间变化、说明空间路径、拆解公式，还是优化已有图片。

2. 匹配可视化形式  
   根据用户任务选择最匹配的图像形式。用户明确指定图型时优先尊重；如果用户只说“做个图”“做成知识图”“可视化一下”，必须主动选择更合适的可视化形式。

   - 分类、层级、知识总览：思维导图、知识图谱、树状结构、矩阵卡片。
   - 步骤、审批、操作、业务流：流程图、泳道图、步骤图。
   - 因果、问题分析、策略拆解：鱼骨图、因果图、策略地图。
   - 实体关系、系统关系、组织关系：ER 图、架构图、系统协作图、用例图。
   - 结构组成、器官、装置、物体拆解：结构示意图、剖面图、爆炸图、标注图。
   - 数据对比、趋势、占比、分布：柱状图、折线图、饼图、热力图、散点图。
   - 时间变化、演化、项目进度：时间轴、阶段图、生命周期图。
   - 空间移动、路线、迁徙、运输：路径图、路线图、地图示意图。
   - 界面结构、票据/凭证版式教学：UI 架构图、组件说明图、票据结构示意图、凭证结构说明图。
   - 公式推导、受力、函数、电路、化学配平：推导图、函数图、受力分析图、电路图。
   - 教材讲解、课堂展示、课件页面：教材插图、板书式解析图、PPT 课件配图、课堂练习讲解图。
   - 快速记忆、复习总结、重点提炼：知识卡片、复习卡、对比卡片。
   - 面向大众解释复杂知识、健康常识、行业链路、操作指南：知识科普海报、分区信息海报、科普长图。

3. 识别产物类型  
   判断用户要的是图片生成、图片编辑，还是知识图动态化。被主 `doubao-creative-design` 路由时，默认只规划图片生成 / 图片编辑；只有宿主环境明确提供视频工具且用户明确要求视频时，才规划动态化。

4. 选择工具  
   - 无参考图、从零生成知识图像：使用 `image_gen`。
   - 有上传图、历史图、草案图，或用户说“基于这张改”：使用 `image_edit`。
   - 无参考图，从文本生成知识视频：仅在宿主工具提供 `text_to_video` 时使用，否则说明当前只能交付知识图像 prompt 或静态图。
   - 基于图片生成动态知识视频：仅在宿主工具提供 `image_to_video` 时使用，否则说明当前只能交付知识图优化或动态分镜 prompt。

5. 明确比例和数量  
   - 横版知识图、课堂 PPT、流程图：优先 `16:9`。
   - 知识卡片、复习卡、竖版信息图：优先 `3:4`。
   - 思维导图、知识图谱、结构总览：优先 `1:1` 或 `4:3`，节点多时用 `16:9`。
   - 手机竖屏视频或竖版知识图：优先 `9:16`。
   - 用户明确指定比例或数量时，遵从用户。

6. 扩写 Prompt  
   按“图像类型 → 知识主题 → 画面文字 → 构图结构 → 信息层级 → 视觉风格 → 色彩字体 → 图形元素 → 质量要求”扩写。

7. 输出工具参数和 Prompt  
   被主 `doubao-creative-design` 路由时，按主 skill 的方式直接调用图片工具，不把内部工具参数模板暴露给用户；如果只需交付 prompt，则输出工具、比例、数量或时长、参考图信息、保留项/修改项、Prompt。不要解释内部路由过程。

## 通用执行内核

本 ref 可脱离主 MD 独立参考。被主 skill 路由时，执行顺序固定为：理解知识表达任务 -> 选择可视化形式 -> 确定比例和张数 -> 补全可渲染文字清单 -> 写独立 design_brief/prompt -> 调用 `image_gen` 或 `image_edit` -> 结果检查。

- `image_gen` 只在没有参考图、历史图或核心资产，且需要从零生成知识视觉时使用；入参为独立 `prompt` 和 `ratio`。
- `image_edit` 用于所有已有上传图、历史图、草图、截图、课件图或知识图优化任务；入参为独立 `prompt`、`ratio` 和主 skill 要求的参考图字段。
- 有参考图时，必须在 prompt 中说明保留项、修改项和目标画面；不能用纯文生图重画导致知识结构或原有文字丢失。
- 多图任务必须逐张规划，每张只承载一个明确教学目标；不要把多张正式产物合并到同一画布，除非用户明确要求总览图。
- 高密度知识图必须先建立 `visible_text_inventory`：标题、节点、标签、步骤、公式、坐标轴、刻度、单位、图例、说明卡和结论必须逐条明确。最终 prompt 只能渲染清单内文字。
- 所有画面文字必须逐字明确，并用中文双引号包裹；用户提供的标题、公式、数据、节点、步骤必须完整保留。
- 公式、单位、坐标轴、刻度、图例、实验器材、结构名称、流程顺序、箭头方向、比例关系和数据含义不能随意编错；缺少关键真实数据时，用明确的教学示例或要求用户补充，不能伪造真实世界结论。
- 传入工具前自检：prompt 中没有工具字段、路径、模型名、画幅参数、像素尺寸或内部 skill 名称；所有中文双引号内容都是最终画面文字；没有使用“XX”“若干文字”“相关说明”等占位符。

## 输入

可接受输入：
- 纯文本知识主题，例如“英语语法思维导图”“企业请假流程图”“温度对酶活性影响折线图”。
- 带具体文字的图像需求，例如标题、节点、步骤、公式、数据、说明文案。
- 上传图或历史图，用于优化、改风格、补节点、提高清晰度、改成教材风。
- 视频需求，例如“把这张流程图做成 6 秒动画”“做一个公式推导短视频”。

关键信息缺失时的处理：
- 能合理补全时直接补全，不追问。
- 缺少学科主题、核心内容、参考图作用，且无法判断画面结构时，只问 1 个核心问题，并给 2-3 个选项。

## 输出形式

被 `doubao-creative-design` 主 skill 路由并实际生图时，不需要把以下模板原样展示给用户；它们用于内部规划和 prompt 生成。若用户要求“只给 prompt / 工具参数”，可按以下格式输出。

图片从零生成：

```text
工具：image_gen
比例：16:9
数量：1
Prompt：一段完整的图像生成 Prompt
```

图片编辑：

```text
工具：image_edit
参考图片 ID：用户上传图
比例：沿用参考图
数量：1
保留项：保留原图的主题、主体、结构、文字、布局和核心风格
修改项：写清要修改、补充、优化的部分
Prompt：一段完整的图片编辑 Prompt
```

文本生成视频：

```text
工具：text_to_video
比例：16:9
时长：6 秒
Prompt：一段完整的视频生成 Prompt
确认点：需要准确呈现的文字、公式、数据或步骤
```

仅当宿主环境明确提供 `text_to_video` 时使用；否则改为输出知识图 prompt 或视频分镜 prompt，并说明当前不可直接生成视频。

图片生成视频：

```text
工具：image_to_video
参考图片 ID：上一张知识图
比例：沿用参考图
时长：6 秒
保持主体：保留原图标题、结构、文字、配色和布局
Prompt：一段完整的图片转视频 Prompt
```

仅当宿主环境明确提供 `image_to_video` 时使用；否则改为输出图片动态化分镜 prompt，并说明当前不可直接基于图片生成视频。

多图任务必须逐张输出工具参数和 Prompt，不要把多张图合并成一张。

## 工具与脚本

| 工具 | 使用条件 | 不要用于 | 必填规划信息 |
| --- | --- | --- | --- |
| `image_gen` | 无参考图、从零生成知识图像 | 有上传图、历史图、核心资产，需要保持一致或局部修改 | 主体、风格、构图、色彩、文字、比例、数量 |
| `image_edit` | 有上传图、历史图或核心资产，需要修图、改风格、换背景、扩图、延展、保持主体一致 | 纯文本从零生成、无资产依赖任务 | 参考图片 ID、保留项、修改项、输出数量、每张目标 |
| `text_to_video` | 宿主明确提供该工具，且用户要求无参考图知识视频 | 需要沿用图片主体、海报、角色或产品的视频；宿主未提供视频工具时不要使用 | 主题、时长、比例、分镜、动作、场景、确认点 |
| `image_to_video` | 宿主明确提供该工具，且用户要求基于上传图、历史图或知识海报生成视频 | 脱离参考图重新生成无关主体；宿主未提供视频工具时不要使用 | 参考图片、保持主体、动态方式、镜头运动、时长、比例 |

工具参数规则：
- `ratio` 单独写在比例字段，不写入 Prompt。
- Prompt 不写工具名、模型名、参数字段、内部说明。
- 单次工具调用只产出 1 张图片或 1 条视频。
- 有参考图时，不要用纯文本工具重画；图片任务应使用 `image_edit`，视频任务仅在宿主提供 `image_to_video` 时使用该工具。
- 被主 `doubao-creative-design` 路由时，实际工具字段以主 `SKILL.md` 的工具协议为准；本表只说明知识教育视觉的工具选择逻辑。

## Prompt 改写规范

### 基础规则

Prompt 必须是一段自然、完整、可生成的中文画面描述，不能是关键词堆砌。

必须包含：
- 图像类型：逻辑关系图、结构科学图、数据图表、工程图纸、时空演化图、UI/票据教学图、艺术文化图、公式推导图等。
- 核心主题：明确学科、知识点、用途或教学目标。
- 画面文字：标题、节点、标签、步骤、说明卡、公式、图例。
- 构图结构：左右树状、中心放射、从左到右流程、上下分区、网格卡片、时间轴、剖面标注。
- 信息层级：主标题、一级节点、二级节点、辅助说明、底部标签。
- 视觉风格：现代教育信息图、教材矢量风、科学可视化、商务办公风、复习卡片风。
- 色彩字体：主色、辅助色、分区色、背景色、无衬线字体、黑体、标题加粗、正文规整。
- 图形元素：节点、箭头、连线、图标、标注框、引出线、局部放大框、图例。
- 质量要求：高清矢量质感、清爽整洁、高可读性、无水印、无乱码、无截断。

### 文字规则

- 所有画面文字必须用中文双引号“”包裹。
- 用户提供的文字必须完整保留，不得删改。
- 不使用“XX”“某某”“标题文案”“待补充文字”“若干文字”等占位符。
- 用户只给主题时，必须主动补全明确可渲染文字。

### 禁止写入 Prompt 的参数

Prompt 只写视觉内容，不写工具参数。

禁止写入 Prompt：
- 尺寸、分辨率、像素、DPI、PPI、宽高、文件格式，例如“2048x2048”“300dpi”“PNG输出”。
- 比例参数，例如“16:9画面”“3:4比例”。比例只能写在 `比例：` 字段。
- 色值参数，例如 Hex、RGB、CMYK、HSL、Pantone。不要写“#1E3A8A”“RGB(30,58,138)”。
- 模型参数，例如 seed、steps、CFG、采样器、模型版本、质量档位。
- 镜头参数数字化堆砌，例如“8K”“超高清 4K”“85mm f/1.4”，除非用户明确要求摄影风格且不属于工具参数。

允许写入 Prompt：
- 自然语言视觉描述，例如“深蓝与浅灰色块”“低饱和蓝绿色”“高清矢量质感”。
- 学科内容必要数值，例如“37℃”“9.8m/s²”“0-50mL”“相对酶活性(%)”“s = v₀t + 1/2at²”。

### 抽象词转译

- “清晰”转为：节点间距均匀、线条连续、层级分明、文字对齐、留白充足。
- “专业”转为：平面无透视、教材式布局、现代无衬线字体、标注规范、低饱和配色。
- “好看”转为：主辅色明确、分区配色、圆角卡片、细线连接、图标统一。
- “有逻辑”转为：从左到右、从上到下、中心到分支、主线到细节的阅读路径明确。

### 专业化提升策略

- 结构优先：先确定阅读路径，再写装饰。每张图必须有明确第一视觉，例如中心节点、流程主线、核心公式、最大数据点、主体剖面。
- 信息分层：至少区分主标题、一级信息、二级信息、辅助说明。重要信息用大小、粗细、色块、位置建立层级。
- 教学闭环：尽量包含“概念/结构/过程/关系/结论/易错点”中的 2-3 层，避免只展示表面元素。
- 标注规范：标签靠近对象但不遮挡主体；引出线清晰、不交叉；图例固定在空白区。
- 可读性优先：文字短句化，节点间距均匀，留白明确，主线连续，避免线条穿插和内容拥挤。
- 学科准确：公式、单位、坐标轴、实验器材、结构名称、流程顺序不能随意编错。

### 多样性策略

同一主题需要多张方案时，必须让每张有明确差异，不能只是换颜色。

可从以下维度做差异化：
- 结构差异：中心放射、左右树状、矩阵卡片、横向流程、纵向步骤、时间轴、剖面标注。
- 风格差异：教材矢量风、现代教育信息图、商务办公风、复习卡片风、科学示意图风、历史人文线描风、手账科普风、纸张笔记风。
- 信息重心差异：总览版、步骤版、对比版、易错点版、记忆卡版、数据分析版。
- 色彩差异：马卡龙、蓝灰商务、低饱和绿色、黑红灰理科、米白历史人文、浅灰科技风、暖黄色亲和科普、青绿色清爽科普。

## 子场景规范

### 3.1 逻辑与关系图表

覆盖：架构图、流程图、思维导图、知识图谱、策略地图、分类判断图、ER 图、时序图、用例图、系统协作图、鱼骨图、因果图、矩阵关系图。

Prompt 要点：
- 写清核心主题、关系类型、节点层级、连接方向、主线与分支。

### 3.9 知识科普海报

覆盖：健康科普海报、生活常识科普、行业知识科普、科技知识科普、操作指南海报、风险提示海报、科普长图、面向大众的一图读懂内容。

Prompt 要点：
- 用强标题区建立第一视觉：大标题、副标题或一句话钩子，标题短而明确。
- 用导语区解释核心概念：在标题下方放一个浅色说明框，用 1-2 句说明“这是什么、为什么重要”。
- 用 3-6 个模块分区承载知识点：每个模块包含“模块标题 + 2-3 条短句 + 对应插画/图标/示意元素”。
- 图文必须绑定：症状、原因、步骤、风险、建议、结构、链路等内容都要有对应图标、人物动作、物品示意或流程箭头，避免纯文字堆叠。

结构策略：
- 大众健康/生活常识：暖色或清爽底色，卡通人物/器官/物品插画，症状、原因、建议分区。
- 快速科普/一分钟了解：纸张笔记底、编号模块、虚线框、重点词高亮、轻量插画点缀。
- 行业链路/复杂系统：上中下游或分层结构，箭头流向、图标系统、关键结论条。
- 操作指南/标准步骤：大标题、步骤卡片、操作插画、注意事项栏，重点步骤编号清晰。

避免：把海报写成纯文字文章；模块过多导致拥挤；插画与文字无关；医学健康内容绝对化；标题过长不可读；把科普海报做成商业促销图。

## 图片编辑规范

使用 `image_edit` 时，必须写清：

- 保留项：参考图的主体、知识结构、原有文字、构图比例、色彩基调、风格、布局关系。
- 修改项：要改变的内容，例如“提升文字清晰度”“改为教材矢量风”“补全缺失节点”“统一配色”“增加底部说明卡片”。
- 质量要求：非修改区域保持不变，新增文字清晰准确，无水印、无乱码、无截断。

知识图纠错、重绘、结构优化规则：
- 先识别并保留原图的教学目标、主标题、节点关系、阅读方向和关键文字；不要脱离原图重做成另一套内容。
- 纠错优先级高于美化：先修正错误文字、公式、单位、箭头方向、坐标轴、图例、节点归属、流程顺序，再调整版式、配色和图标。
- 重绘时必须说明“保持知识结构不变”或逐条列出允许调整的结构；不确定是否为错误时，不擅自改知识结论。
- 结构优化只做教学表达优化，例如合并重复节点、补充分组标题、调整阅读顺序、增加说明卡；不能新增未经用户确认的真实数据、结论、出处或权威背书。
- 课件图优化必须保持原页比例和主讲内容，优先提升投屏可读性、标题层级、图文绑定和留白。

示例结构：

```text
工具：image_edit
参考图片 ID：用户上传图
比例：沿用参考图
数量：1
保留项：保留原图主题、主要结构、节点关系、阅读方向和已有文字。
修改项：将整体改为现代教育信息图风格，统一节点样式，提升文字清晰度，补充缺失说明标签。
Prompt：基于参考图进行知识图优化，保留原图的主题、主要结构、节点关系、阅读方向和已有文字不变；将视觉风格统一为现代教育信息图，白色背景，低饱和分区配色，所有节点改为圆角卡片，连线改为细线或连续箭头，提升文字清晰度和对齐关系。补充必要说明标签，新增文字必须完整清晰，整体信息层级分明，高可读性，高清矢量质感，非修改区域保持一致，无水印、无乱码、无截断。
```

## 视频规范

知识视频只做轻量动态化，不改写成复杂剧情。

文本生成视频必须写清：
- 知识主题。
- 镜头顺序。
- 每个镜头呈现的知识内容。
- 文字如何出现。
- 动态方式，例如节点点亮、箭头流动、公式逐步显现、时间轴推进。

图片生成视频必须写清：
- 参考图保持不变的部分。
- 动态化方式。
- 镜头运动。
- 不改变原图知识结构和文字。

示例结构：

```text
工具：image_to_video
参考图片 ID：上一张知识图
比例：沿用参考图
时长：6 秒
保持主体：保持原图标题、节点结构、配色、文字和布局不变。
Prompt：基于参考图生成轻量知识讲解动画，保持原图标题、节点结构、配色、文字和布局不变。镜头从整体轻微推近，中心标题先清晰出现，然后各主分支节点按顺时针顺序逐个点亮，连接线出现柔和流动效果，最后底部总结标签轻微高亮。整体节奏平稳，适合课堂展示，不改变任何文字内容，不新增无关元素，无水印、无乱码。
```

## 最终检查

输出前检查：
- 工具是否选对。
- 是否写明比例、数量或时长。

- Prompt 是否没有混入尺寸、分辨率、色值、模型参数、比例字段等工具参数。
- 画面文字是否全部用中文双引号“”包裹。
- 是否补齐标题、节点、分支、步骤、说明、图例等关键内容。
- 知识科普海报是否具备标题区、导语区、分区模块、图文绑定和底部结论。
- 是否避免占位符、平台名、水印、乱码、截断。
- 有参考图时是否使用了 `image_edit` 或 `image_to_video`，并写清保留项与修改项。
