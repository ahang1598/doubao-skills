# 全平台营销配图与文案生成器

## 主 Skill 路由适配

本 ref 只提供多平台媒体创意的产物规则。被 `doubao-creative-design` 路由调用时，工具协议、澄清轮次、核心资产记录、张数策略、prompt 总规则和关键实体信息红线以主 `SKILL.md` 为准；本文件只补充平台视觉特征、内容传播策略、媒体封面结构、小红书/H5/长图策略、平台禁忌和传播文字规则。

本 ref 不强行限制媒体物料张数。用户明确要求多页、套图、长图文或跨平台分发时，按内容结构规划足够页数；用户未指定张数时，先按完成传播目标所需的最小张数规划，避免为了凑数量生成重复图。

## 概述
用于生成符合 B站、抖音、快手、微信朋友圈、微信公众号、小红书等平台视觉规范的营销配图（封面、长图文、笔记配图等），以及跨平台通用的 Banner（横向焦点图）与 海报（竖版 Poster）、H5页面（移动端互动长页）等，及配套发布文案，帮助用户快速创建高质量的社媒视觉内容。
核心原则：严格遵守各平台独家视觉指纹（如小红书的高信息密度四区网格、B站的弹幕梗图、抖音的破屏透视等），保证对应赛道特征与系列感，同时强制遵循文字渲染规范与各平台合规红线。

## 使用场景
**何时触发该skill：**
- 用户的任务描述中包含“B站”、“抖音”、“快手”、“朋友圈”、“公众号”、“小红书”、“笔记”、“H5”、“种草”、“OOTD”、“探店”、“视频封面”、“私域流量图”、“Banner”、“焦点图”、“轮播图”、“海报”、“宣传海报”等相关关键词。
- 用户明确要求生成具有上述特定社媒平台风格的配图或排版内容。
- 在多轮对话中，当前轮次为上述配图任务的延续、修改或补充说明时（即使未显式提及上述关键词）。

**何时不触发该skill：**
- 纯文字翻译、纯代码编写等与社媒配图/视觉营销/发布文案完全无关的任务。

## 输入
- **目标平台**：优先识别用户指定的平台（B站、抖音、快手、朋友圈、公众号、小红书、H5）。若未指明但可从“竖版短视频封面、笔记图、公众号头图、朋友圈图、社媒长图”等语义推断，直接内部选择；只有多个平台会明显改变比例、信息密度和视觉语言且无法判断时，按主 `SKILL.md` 只问一轮关键问题。
- **发布主题**：一句话说明"你想让读者看完得到什么"。用户未明确时，根据已给主体、用途、参考图和平台语境内部推断；只有主题缺失到无法确定画面核心时，按主 `SKILL.md` 只问一个关键问题。用户说“你自己定/随便定”时，直接选择最贴近上下文的主题并执行。
- **关键实体信息确认与保留（如需）**：
  - 当用户的主题或需求暗示了必须出现具体的活动日期、具体的人名、活动地点、商品价格等关键实体信息时，按主 `SKILL.md` 的澄清规则处理。禁止使用“xx”、“某某”等指代词；用户未提供且无法确认时，不编造真实日期、价格、地点、人名、认证编号或监管声明，可改用非具体表达或不在画面中渲染该信息。
  - **⚠️专有名词保留约束**：如果用户的 prompt 中已经提供了具体的品牌名、活动名、产品全称等关键实体信息（如“怡宝矿泉水”），**绝对禁止在推演和撰写提示词时对其进行省略或概括**（如省略为“矿泉水”），必须原封不动地保留并应用到画面主体描述或文字渲染方案中。
- **参考图**：根据用户上传/提供的图片进行意图推断（如品牌素材、产品展示、人物姿态、场景氛围、构图、光影、色彩、风格等，详见参考文件中的 [附表2：参考图推演与高级审美风格提升规则](#附表2参考图推演与高级审美风格提升规则)）。
  - *注：若是"随手拍"低质产品图，需主动提升质感；不确定参考意图时需向用户提问确认。*
- **生成张数**：
  - 未指定时，文生图通常规划 **1-4 张**，图生图通常规划 **1 张**；若主题天然需要封面、过程、清单、结尾 CTA 等连续阅读，可增加页数，不设置机械硬上限。
  - 用户未指定生成单张长图，但明确要求生成多图（如“完整的多页图文”、“全套配图”）时，默认采用**分页多图**形式。根据用户的意图自行判断用多少张图能表达清楚用户的主题，一般遵循：**主题能展开说的信息越多，所需的张数越多** 的规律；页数必须服务内容结构，不能重复凑数。
  - 用户明确要求生成多张或特定张数的图时（如“3张图分享”）时，严格按用户要求规划/生成张数。
  - **️长图推演要求**：必须按照用户需求产出对应的整张长图，不要进行分页。但在设计方案推演时，要保证整张图的信息密度与逻辑连贯性和分页保持一致，长图各板块（通常是从上至下垂直排布，引导向下滑动阅读）必须有序衔接，自然过渡，文案详实具体。*

## 核心流程
**Step 1: 需求分析与提示词处理**
- 确认或内部推断用户发布主题；只有无法确定传播核心时才按主 `SKILL.md` 澄清。
- 根据用户 prompt 长短进行**核心提炼**或**合理扩充**。
- 必须将需求收束为以下 7 项分析结果：
  1. **内容类型**：从赛道中定 1-2 个主类型。
  2. **钩子潜力**：识别钩子类型（数字/痛点/好奇/利益/身份），给出 1 个更强的标题钩子备选。
  3. **受众**：主要受众 + 次要受众 + 他们最在意的 1 句话。
  4. **互动信号**：收藏理由/分享触发/评论引导（各 1 条）。
  5. **视觉表现**：把内容元素映射到视觉载体（数据→大数字；对比→左右分屏；步骤→1→2→3流程；清单→列表卡片；情绪→表情/场景）。
  6. **滑动流**：封面→铺垫→核心（每页 1-2 要点）→收获→结尾 CTA。
  7. **发布文案规划**：规划发布文案的正文内容（包括标题、正文开头、正文主体、正文结尾、话题标签）等。

**Step 2: 匹配平台视觉特征与赛道核心**
必须根据目标平台应用视觉共性（详见 [附表1：各平台视觉特征与赛道核心](#附表1各平台视觉特征与赛道核心)）。

**Step 3: 推演设计方案**
遵循三层推演流程：明确骨架 -> 场景语义推演 -> 实例填充。
- **⚠️色彩反刻板原则**：在推演配色方案时，**绝对禁止对季节、特定概念进行低级的视觉刻板映射**（例如：春天/护肤必用大面积绿色、秋天必用黄色、科技必用深蓝色）。当参考图中包含高饱和度的产品/主体色时（如绿色的瓶子），在设计背景与全局主色调时，必须**主动采用中性色（如临床白、高级灰、莫兰迪色、奶油白）或低饱和辅助色进行视觉中和**，严禁直接提取主体高亮色作为大面积背景色，坚决避免造成“绿上加绿”等色彩泛滥与审美疲劳。
- **⚠️文字渲染强制规则**：所有待渲染的可视化文字都要用中文双引号`“”`框定。非生成文字内容绝对不能用`“”`表示强调。
- **多图串行推演**：如果生成多图，必须继承前一张的视觉资产，确保系列感一致。
- **T2I（文生图）方案**：按“总览句 + 分区扫描主段落（基础9维，具备极高的空间描述和量化密度）+ 条件收束 + 画幅比例”输出（详见附表3）。
- **I2I（图生图）方案**：按“编辑目标 + 保留项 + 编辑项 + 空间线性描述”输出，整合成一段连贯的话（详见附表4）。

**Step 4: 红线与禁忌审查**
- 检查是否存在版权IP、敏感词、不良视觉元素、违反公序良俗的内容（详见 [附表6：硬性红线与禁忌项](#附表6硬性红线与禁忌项)）。
- 检查是否存在占位符（如`[文案]`，必须写出完整明文）、站外引流、明显水印、违规医疗/功效宣传、极限词等违规元素。
- **⚠️平台隔离原则**：设计方案中**绝对禁止出现平台名称**（如“抖音风”、“B站风”、“小红书”），也不要出现“笔记”、“爆款”等平台强相关的运营词汇，需将这些概念转化为对应的纯视觉描述。
- 检查设计方案中是否包含了未与用户澄清的编造事实信息（如自己编造的日期、地点、人名、价格、认证等），若有，必须移除、改为非具体表达或按主 `SKILL.md` 的澄清规则处理，绝不允许出现占位符。

**Step 5: 调用工具生图与审查**
- **⚠️多图生成与串联执行规则**：不管要求生成几张图，对每一张图都必须独立进行完整的方案规划。在调用生图工具时，**必须多次调用，每次调用只生成一张图**。
  - **系列感保持约束**：当需要生成同系列多图时，必须**串行执行**：即先完成第一张图的规划与生图，审查合格后再进行第二张图的规划与生图。后续生图必须将前一轮成功生成的图片作为参考图传入，以保持系列感一致性。
- 必须多次调用生图工具，每次只生成一张图。
- **合格生成图要求**：
  - 用户生图需求中要求的图片、文字/任意元素已正确生成，不存在物体/文字结构崩坏。
  - 画面文字与方案中列出的“文案内容”完全一致，无变形、无乱码、无裁切、无重复生成。
  - 画幅比例正确，无平台水印或违规元素。
- **【审查生图结果】**
  - 未达标需修正方案重新生图，**废稿静默丢弃，禁止在最终回复中展示**。
  - 连续两次未达标则停止生图并仅交付最后一次结果。

**Step 6: 输出与交付**
- **交付图片**：按顺序编号展示最终生成的图片，说明其对应平台的用途（如“B站视频封面”、“朋友圈首图”、“公众号长图”）及与文案的对应关系。
- **发布文案（按需输出）**：仅当用户明确要求生成文案时输出。文案必须根据目标平台动态调整语气与格式（例如：B站带梗、抖音/快手短平快、朋友圈极简原生、公众号克制专业、小红书真人分享感），严禁跨平台生搬硬套。
- **细节反馈引导**：
  - 若本次任务仅生成了配图，主动追问用户：“是否需要为您生成适配该平台的发布文案？”
  - 提供 1-3 个维度的效果延展、形式变化或多平台分发建议（例如：“当前海报可调整为 16:9 用于 B站视频封面，是否需要为您生成？”）。保持简短，无明确增量时不展开。

## 参考附表

### 附表1：各平台视觉特征与赛道核心
#### 1. B站 (哔哩哔哩) - 默认 16:9
- **平台共性**：二次元/ACG视觉锚点、左右分栏对抗式构图、极高信息密度、弹幕梗图互动、粗黑描边+鲜亮主标。
- **核心赛道**：
  - **数码/测评**：硬核参数跑分对决，UP主半身特写+硬件抠图左右排布，暗红+科技蓝对比。
  - **干货洞察**：核心结论+数据化表达，学术感深色背景极细网格，顶部超大标题。若用户明确要求知识科普海报、教学图、公式/图表准确性，回到 `knowledge-education-visual.md`。
  - **ACG/游戏**：二次元插画/立绘，动漫粗体，高饱和动漫色，局部角标。
  - **鬼畜/玩梗**：抽象拼贴搞笑，表情包大集合，高饱和撞色分离。
  - **生活/Vlog**：二次元过滤后的生活感，暖色系糖果色，手书体+弹幕。
  - **影视解说**：电影海报感，暗色基调+单点高光，关键瞬间定格+反转钩子。

#### 2. 抖音 - 默认 9:16
- **平台共性**：破屏透视构图、极致焦点放大（主体占70%）、高反差光影（戏剧舞台感）、情绪化夸张表情、动态综艺特效、3D综艺体大字报。
- **核心赛道**：
  - **带货/电商**：超粗3D综艺体红黄撞色，产品破屏怼脸，爆炸贴，紧迫感拉满。
  - **知识口播**：大字报痛点直击，黄黑高对比，UP主夸张表情带白描边，红圈警示。
  - **剧情/短剧**：戏剧性张力悬念，电影感粗衬线，暗色基底+单点高光，冲突瞬间。
  - **直播预告**：强提示倒计时，红紫霓虹色，主播夸张姿势或C位产品。
  - **探店/美食**：沉浸感食物特写拉丝流心，食物破框，食物色+暖光氛围。
  - **颜值/才艺**：高光时刻，简短标题，面部精修+霓虹冷暖对比光效。

#### 3. 快手 - 默认 9:16
- **平台共性**：原生态实景（拒绝棚拍悬浮）、红黄高对比大字报（极粗大黑体）、拒绝高级克制、手部/实物特写传递真实感、直白引导符号（粗大红箭头）。
- **核心赛道**：
  - **源头好货**：产地信任感，带泥土的手捧产品，真实农田车间背景，红黄撞色+爆炸贴。
  - **草根干货**：民间智慧大字报，杂乱但真实的客厅/车间背景，红黄蓝刺眼高对比。
  - **直播预告**：老铁今晚不见不散，实景货架/灶台前的主播，红黄+红色霓虹。
  - **才艺/段子**：滑稽真情瞬间，农村小院街边实景，哈哈表情定格。
  - **探店/家常**：热气腾腾大份量食物，小店/大排档内景，口语化感叹标题。
  - **三农/手艺人**：朴实劳作，老屋山野，大地色系土黄木棕，传统印章。

#### 4. 微信朋友圈 - 默认 1:1 或 3:4
- **平台共性**：原生感优先（像普通用户生活分享，拒绝大字报/抠图合成）、第一人称视角（俯拍/手持）、真实场景为底、轻量留白排版、手写感标注、低饱和真实色温。
- **核心赛道**：
  - **产品安利**：熟人安利，手部/桌面道具不占满画面，自然暖光低饱和滤镜。
  - **上新海报**：新品一角+开箱瞬间，季节性低饱和色，便签手写小字。
  - **开工通知**：整洁工位/门店实景，细黑体直白说明，中性色米白暖灰。
  - **品牌近况**：主理人视角，工作场景大留白，复古胶片色，微噪点。
  - **节日礼盒**：克制温度，应季物品自然散落，压低饱和的节日色，拒绝红包红。
  - **生活记录**：不完美生活感，咬一半的食物特写，暖黄焦糖棕，油烟感痕迹。

#### 5. 微信公众号 - 默认 16:9（头图）或一张长图（长图文内页）
- **平台共性**：克制秩序感、数据化与结构化提炼（数字/提纲）、极简图表化视觉隐喻、品牌色+黑白灰为主、规整字形、金句容器化呈现。
- **核心赛道**：
  - **知识/洞察**：抽象3D隐喻/数据可视化，深邃藏蓝/深灰底色，极粗锐利黑体，左右分栏。
  - **品牌故事**：大片质感，居中单主体+大留白，暖灰米白胶片质感。
  - **产品上新**：产品带轻微环境光，中性场景，结构化卖点（图标+短词）。
  - **节日/活动**：克制节日感，主题色卡纸/剪影底，应季元素秩序环绕。
  - **方法论/培训**：步骤拆解/逻辑树/矩阵图，米白浅灰底，蓝色/绿色系建立理性。
  - **严肃专业(医疗/金融)**：高可信度，专业符号剪影，深蓝/墨绿，大留白严禁花哨。

#### 6. 小红书 - 默认 3:4

### 附表3：T2I设计方案撰写规范
每张图的设计方案必须按以下顺序分别输出，不可缺段：

**结构**：总览 + 分区扫描 + 条件收束 + 画幅比例

##### 总览句

- **第一句**：定位 画面类型 + 载体/构图方向 + 设计风格/氛围关键词。
  - 示例1：这是一张竖版手机屏幕截取的直播间画面，整体呈现出夏日海滩的清新休闲风格。
  - 示例2：这是一幅垂直构图的平面设计海报，融合了复古波普与现代排版风格。
- **第二句（可选）**：交代背景底色/材质 + 画面主体的一句话概括
  - 示例："背景是一片蓝天、碧海和金黄色的沙滩...画面的主体是站在沙滩上的一女一男两位主播。"

##### 主段落

分区扫描式描述（必写，占设计方案 90%+）

- 按 **物理空间顺序** 逐区描述，从上到下/从左到右/从外到内逐个元素展开，每个元素必须满足 **分区描述的颗粒度要求**（见下文）。
- 每个空间分区以 **方位短语开头** 作为段首标记（如"从画面的最顶部开始""画面的左侧中部""屏幕的最底部"）
- 同一分区内的并列元素用"从左到右依次为""左侧...右侧..."句式
- 分区间用 **空间衔接词** 串联（"紧挨着...下方""再往下""继续向下""在...的正下方"）

**分区描述的颗粒度要求**：

1. **基础 9 维（所有元素必填）**
   设计方案中每一个可见元素都填写以下 9 项，以符合高信息密度的要求。

   - **绝对位置**：使用绝对方位词（左上角/正中央/顶部居中/右侧中部/最底部正中央），禁止 旁边/附近/大概 等模糊词。
   - **元素类型**：图标/卡片/文字/按钮/头像/分割线/浮窗/标签/输入框/折线图等。
   - **形态细节**：描述元素由什么视觉部件组成（如 由三个短横线构成的菜单图标/带有风的线条和两片叶子的圆形图标/放大镜形状的搜索图标/蓝色六边形等级牌"17" 等），禁止仅用功能名（菜单图标/搜索图标/等级标志）代替。
   - **内容描述**：图片类：描述图片主体+场景+色调；文字类：用中文双引号""逐字给出所有文案。文字类包括状态栏时间/电量、按钮文字、弹幕内容、价格标签、地址信息、被截断的文字等。
   - **文字视觉样式**：颜色 + 字体粗细 + 大小 + 字色 + 是否有背景色 + 边框 + 特效（删除线/渐变/阴影/倾斜/半透明/模糊/加粗/下划线等）等。公式：\[载体形态] + \[颜色/材质/特效/背景色/边框] + \["文案"]
     - 正确示例：被黑色细线划掉的灰色"原价¥70"。
     - 错误示例："原价70元"。
   - **状态信息**：选中/未选中、亮起/置灰、激活/禁用，是否有角标/徽章/遮罩等。
   - **关联关系**：和相邻元素的对齐方式、间距感 (均匀分布/紧贴/留白/并排/居中)。是否被遮挡或截断。多元素层叠时，必须用"A遮挡了B的X部分，但A的Y部分被C遮挡"句式明确前后关系，例：在...的背景上/在...的正下方还悬浮着/与...产生复杂交叠/覆盖了一层/遮挡了...的X部分。
   - **风格特征**：具体材质/配色/处理手法（高对比度洋红色单色调处理/纯粹平滑且无渐变的孔雀绿色 等）。
   - **视觉处理工艺**：描述后期处理手法（单色调摄影/半透明黑色渐变遮罩/复古粗糙笔触 等）。

2. **量化密度基准**

   - **字数基准（按画面复杂度分档，有下限，无上限）**

     | 画面类型 | 典型场景 | 字数下限 |
     | ------- | ------------------- | -------- |
     | 简洁平面 | 单主体海报、logo、名片 | ≥600 字符 |
     | 中等界面 | 社交瀑布流、营销海报 | ≥1200 字符 |
     | 复杂界面/场景 | 多人场景 | ≥2000 字符 |

   - **元素覆盖率**
     - **每一个** 独立元素都进行描述。
     - **禁止省略**：不得用"等""若干""一些""多个"跳过具体元素；若元素重复，必须写出数量和每个实例的差异（如 四个椭圆，每个椭圆里都写着棕色的”149.75代250元”）。

   - **文案还原率**
     - 所有可见文字 **100% 使用中文双引号“”框定文案**，包括：状态栏时间/电量、按钮文字、弹幕内容、价格标签、地址信息等。
     - 被截断的文字也要如实还原截断状态（如 “COMM…”）。 
     - 禁止只用概括性描述进行文案指代。

   - **颜色词密度**
     - 每个独立元素的描述中必须至少包含 **1 个明确颜色词**（橙色/孔雀绿/半透明深色/灰白色/洋红色等），禁止出现无颜色定义的元素。

   - **方位词密度**
     - 每 50 字符至少出现 1 个绝对方位词或空间关系词

   - **衔接词体系（必须使用）**
     - **分区过渡**：从画面的最顶部开始/画面的中上部/画面的左侧中部/屏幕的最底部/紧接着是/再往下是/继续向下是/倒数第X个模块是... 等。
     - **同层并列（区域内平行元素）**：从左到右依次为/左侧...右侧.../左半部分...右半部分...上行为...下行为... 等。
     - **层叠/遮挡（前后关系）**：在...的背景上/在...的正下方还悬浮着/与...产生复杂交叠/覆盖了一层/遮挡了...的X部分 等。
     - **补充说明**：需要特别注意的是/该图标处于...状态，表明... 等。

##### 条件收束

- 当画面元素较少（独立元素 ≤15）且存在统一的全局视觉特征时，补 1–2 句全局配色/排版/氛围总结。
- 复杂画面（UI界面/多元素场景）不需要收束句，最后一个分区描述结束即终止。

##### 画幅比例

- 每张图的设计方案结尾必须以比值格式写清画幅比例。各平台及专项的默认/推荐画幅如下：
  - **B站**：16:9
  - **抖音 / 快手**：9:16
  - **小红书**：3:4
  - **微信朋友圈**：1:1 或 3:4
  - **微信公众号**：16:9（常规首图） / 2.35:1（电影感长图） / 竖向长图
  - **通用 Banner**：16:9 / 2.35:1 / 3:1
  - **通用海报**：3:4 或 9:16
  - **H5页面**：9:16
- 如多张图共享同一输出规格，需在每个设计方案结尾重复写明当前待生成图对应的比例。

- **极简原生(朋友圈)**：**极简使用**，通常仅需 1 句话。第一人称熟人感叹("终于拿到实物了") + 局部手写标注，严禁官方营销腔与满版大字报。
