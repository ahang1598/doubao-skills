# 证据来源、检索与评价

本文件只管“按什么顺序查、优先查什么、哪些证据可用、如何记录和评价”。问题拆解、检索批次和预算见 `core-workflow.md`；图表见 `synthesis-and-visualization.md`；最终文档结构和交付 QA 见 `output-and-qa.md`。

## 1. 检索顺序

完整检索先创建 UTF-8 with BOM 的 literature-log CSV，再调用检索工具。默认按以下顺序执行；每一步只围绕当前检索批次和子问题，不做无目的的全库扩展。

1. `scholar_search`：优先检索近 5 年学术文献以及最新版指南/共识，覆盖中国和国际指南、系统综述、Meta 分析、RCT、大型队列、诊断研究、真实世界研究、里程碑研究和高影响力期刊论文。
   - 临床实践相关主题先定位最新版中国临床指南、本土专家共识、国家卫健委、中华医学会、中国医师协会或专科学会文件，再补国际权威指南和关键研究。
   - 指南/共识任务用于寻找推荐所依据的关键研究和更新证据。
   - 综述、课题、研究进展或论文写作任务以本步骤为主要检索阶段。
2. `general_search`：其次用于前述工具未返回的学会官网、政府/监管页面、期刊原文入口、DOI 页面、临床试验注册或公开 PDF。需要核对药品适应证、禁忌、剂量、特殊人群、相互作用或说明书警示时，先按“`小荷医典 + 药品通用名 + 药品说明书`”检索可访问说明书页面，并核对药品名称、剂型和来源；找到可访问且可核验的小荷医典说明书后，正文、安全性分析和参考文献优先引用该页面链接。
3. `medical_search`：搜索工具中的最后补充，只能用于药品说明书。仅当 `general_search` 按上述方式仍找不到、打不开或无法核验说明书时调用；不得用于检索指南、共识、论文、一般医学知识、监管新闻或器械资料。不涉及说明书时跳过。
4. WebFetch/`web.fetch`、PDF 解析或 OCR：候选文献写入 CSV 并初筛后，对 1-3 份最能支撑结论、图表或正式引用的资料定向读取正文。
   - 只读取推荐条文、摘要、研究设计、主要结果、关键表格、安全性、亚组和局限。
   - 搜索结果片段只能用于初筛，不能冒充正文或原文引用片段。

若某个具名工具在当前环境不可用，使用具备同等能力的工具替代，但仍遵循“学术文献与指南 → 权威网页及小荷医典说明书检索（按需）→ 最后补充说明书检索 → 核心全文阅读”的顺序。内部工具名不得出现在报告、PPT、Word、CSV 或最终回复中。

搜索工具优先级固定为 `scholar_search → general_search → medical_search`。可核验的小荷医典说明书是药品说明书的优先引用入口；只有其缺失或不可用时，才引用 `medical_search` 补充获得的说明书。没有药品说明书核对需求时直接跳过。无论说明书来自哪一步，都必须立即将标题、来源和可访问链接写入 literature-log CSV。WebFetch/PDF/OCR 不得在候选写入 CSV 和初筛前用于漫无目的浏览，也不为凑轮次强行调用无关工具。

### 任务起始边界

| 任务重点 | 必须优先覆盖 | 后续重点 | 不应扩展到 |
|---|---|---|---|
| 指南/共识查新 | 最新中国指南/本土共识；再查国际指南 | 指南更新依据、关键研究和差异点 | 无关疾病、旧版替代新版、非权威解读 |
| 疗效、诊断或干预证据 | 权威指南用于界定人群和结局 | 系统综述、RCT、队列、诊断研究 | 与目标人群或结局无关的机制研究 |
| 药品、器械和安全性 | NMPA/说明书/国家或学会规范；必要时 FDA、EMA | 高质量安全研究、特殊人群证据 | 自媒体药品解读、商业宣传页 |
| 研究进展、热点或综述 | 近 1-5 年高质量综述和关键研究 | 指南更新、注册研究、争议与空白 | 只凭标题词频推断热点 |
| 里程碑与证据脉络 | 奠基性研究和经典高被引研究 | 后续验证、指南采纳和最新进展 | 用“近 5 年”机械排除经典研究 |
| 课题或论文写作 | 与论点直接相关的综述、原始研究和方法来源 | 争议证据、研究空白和可引用段落 | 为凑引用加入不支持论点的文献 |
| 教学或学习资料 | 最新指南、高质量综述和权威教材 | 经典研究和实践资料 | 用教材旧观点替代最新证据 |

中国来源优先适用于中国临床实践、药品可及性、本土人群和国内规范相关问题；全球研究进展、基础机制或国际课题不机械要求中国证据占主导，但用户涉及中国实践时必须补充本土证据。经典里程碑研究不受年份限制；“最新进展”默认优先近 1-5 年。

## 2. 证据准入与覆盖

### 核心证据准入

核心证据必须同时满足：

1. 有可点击原文、官方指南页、监管页、期刊页、PubMed/PMC、PDF 或 DOI 解析入口。
2. 已打开正文、指南原文、期刊页或官方入口，并如实记录阅读状态。
3. 能定位到支持结论的推荐、结果、表格、章节、摘要结果或研究数据。
4. 与用户的主题、人群、干预/暴露、对照、结局或论文论点直接相关。

只有题名、PMID/DOI 字符串、搜索结果片段、二次转载或无可点击入口时，不得作为核心证据。只能读到摘要的高价值文献可列为 `仅摘要/辅助参考`，前提是有可点击摘要或期刊入口；不得单独支撑确定性疗效、安全性或指南更新结论。

### 默认证据覆盖量

引用数量以覆盖问题为目标，不机械凑数。完整报告通常覆盖：

| 证据类型 | 建议范围 | 主要用途 |
|---|---:|---|
| 指南/共识 | 2-6 条 | 推荐、适用人群、更新点和实践边界 |
| 系统综述/Meta | 2-6 条 | 总体方向、异质性和确定性 |
| 关键研究 | 4-12 条 | 里程碑、核心 RCT、队列、诊断或真实世界证据 |
| 最新进展 | 3-10 条 | 近年新证据、新技术、新药物或争议 |
| 安全/监管 | 1-4 条 | 禁忌、警示、特殊人群和监管变化 |
| 本土/专科证据 | 按需 | 中国规范、人群差异和实践背景 |
| 教材/实践资料 | 按需 | 学习框架、概念和实践解释 |

轻量回答只需 1-2 个高价值批次，但每个关键判断至少有一个可点击权威来源。完整报告定向精读 1-3 份核心资料；其余纳入文献可以按实际访问层级保留为全文、摘要或辅助参考。某类证据缺失时明确披露，不用低质量来源补数。

### 来源等级

`source_level_initial` 是本 Skill 的初筛来源等级，不等同于原文 GRADE 或正式证据等级。

| 等级 | 来源 | 使用边界 |
|---|---|---|
| A | 最新中国或国际权威指南/共识、监管机构和说明书、Cochrane/高质量系统综述、高质量 RCT、大型队列、顶级或核心期刊关键研究 | 支撑核心结论 |
| B | 质量较好的观察性、真实世界或诊断研究，卫生技术评估、权威教材/专著、临床路径和可靠专科资料 | 补充证据与实践背景 |
| C | 小样本单中心研究、病例系列/报告、会议摘要、预印本、低质量叙述性综述、机制/动物/体外研究 | 前沿线索或低确定性补充 |
| D | 自媒体、营销软文、问答社区、搜索结果页、来源不明转载、疑似垃圾期刊或无法核验页面 | 禁止作为依据 |

## 3. 文献台账 CSV

完整检索开始时创建：

`literature-logs/{{YYYYMMDD}}-{{topic_slug}}-literature-log.csv`

从 `assets/literature-log-template.csv` 复制 UTF-8 with BOM 表头。文件名和内容不得包含患者姓名、身份证、手机号、住院号等敏感信息。CSV 不记录 `tool`、`retrieved_at`、`pmid`、`scholar_level` 或 `screening_category`；证据类型与筛选类别分别写入 `publication_type` 和 `screening_decision`。

| 字段组 | 字段 | 用途 |
|---|---|---|
| 检索标识 | `record_id`、`search_batch`、`query` | 唯一编号、批次和实际检索式 |
| 文献基础信息 | `title`、`journal_or_source`、`year`、`publication_type`、`abstract_or_summary`、`url`、`doi` | 题名、来源、类型、摘要和入口 |
| 初筛与阅读 | `source_level_initial`、`full_text_status`、`screening_decision`、`decision_reason`、`topic_relevance` | 权威性、访问层级、纳排和相关性 |
| 综合用途 | `theme_or_use`、`citation_role`、`supports_conclusion`、`diagram_use`、`used_in_report` | 章节、论点、图表和交付用途 |
| 引用映射 | `evidence_id`、`anchor_text` | 正文、图表、证据表与参考文献统一引用 |

写入规则：

- 每轮检索后先把高相关候选追加到 CSV，再筛选、精读、写正文或绘图；若工具返回候选而 CSV 行数未增加，暂停后续步骤并补录。
- 默认每批次保留 4-10 条候选。更新 `screening_decision` 和 `decision_reason`，不要删除已经记录的排除或待核验条目。
- `url` 必须是可点击 HTTP(S) 原文、摘要或官方入口；只有 DOI/PMID 线索时继续补充 URL。
- 使用标准 CSV 转义。含逗号、换行、双引号、作者列表、卷期页、长摘要或复杂检索式的字段用英文双引号包裹；字段内双引号写成两个双引号。
- 写入后抽查 1-3 行的列数和 `title`、`journal_or_source`、`abstract_or_summary`、`url`、`decision_reason` 是否错列。
- 写正文前，为核心纳入证据补齐 `evidence_id`、`anchor_text`、`url`、`journal_or_source`、`year`、`source_level_initial`、`citation_role` 和 `supports_conclusion`。
- 交付前运行 `scripts/validate-literature-log.js <csv>`；CSV 为空、只有表头或核心证据缺少链接/引用映射时不得继续交付。

## 4. 初筛与原文阅读

### 初筛顺序

1. 是否直接回答当前子问题或论文论点。
2. 人群、干预/暴露、对照、结局和研究场景是否匹配。
3. 来源机构、年份、期刊和研究设计是否可靠。
4. 摘要或简介是否提供可用结果，而非只讨论背景。
5. 是否有可点击原文、摘要、官方入口、期刊页或 DOI 页面。

### 原文阅读边界

只对会进入核心结论、图表或正式参考文献的高价值证据定向精读，不重复 Fetch 同一页面，不把整页网页或整篇 PDF 塞入上下文。

- 指南/共识：推荐条文、推荐强度、证据等级、适用人群、更新日期、排除范围、推荐理由和指南差异。
- 系统综述/Meta：PICO、检索范围、纳入研究、偏倚、异质性、主要结果、敏感性分析和作者结论边界。
- RCT/队列/真实世界研究：样本量、入排标准、干预/暴露、对照、主要终点、效应量、置信区间、安全性、亚组和局限。
- 诊断研究：目标人群、指标检查、参考标准、敏感度、特异度、LR、AUC、PPV/NPV 和误判后果。
- 热点/趋势：主题簇、年份和研究类型变化、代表性研究；没有完整文献计量数据时，不声称完成正式 bibliometric 分析。
- 论文写作：逐句核对文献是否真的支持拟写论点，不为补引用扩大原文结论。

## 5. 证据评价

| 证据类型 | 重点评价 |
|---|---|
| 指南/共识 | 机构权威性、系统检索、证据等级、推荐强度、利益冲突、更新日期和适用范围 |
| 系统综述/Meta | PICO、检索充分性、纳入研究质量、异质性、发表偏倚、敏感性分析和稳健性 |
| RCT | 随机、分配隐藏、盲法、样本量、主要终点、ITT、安全性和失访 |
| 观察性研究 | 混杂控制、暴露/结局定义、选择偏倚、随访和因果推断边界 |
| 诊断研究 | 参考标准、代表性、盲法、诊断性能和误判后果 |
| 真实世界研究 | 数据源、缺失数据、混杂校正、外推性和可重复性 |
| 机制/基础研究 | 模型与临床人群差异，不直接外推临床疗效 |
| 教材/实践资料 | 仅用于概念和路径解释，并与最新版指南和研究互证 |

不要声称完成“正式 GRADE 评级”，除非原文采用 GRADE 或等价体系；无原文分级时只能写“按研究设计和偏倚风险初步判断”。

## 6. 引用映射与诚信

证据 ID 统一使用：`G` 指南/共识、`SR` 系统综述/Meta、`R` 原始研究、`N` 最新进展、`S` 安全/监管、`T` 教材、`C` 中国/本土证据。同一资料在 CSV、正文、图表、证据表和参考文献中使用同一 `evidence_id`。

正文首次出现具体来源时，使用 CSV 的 `anchor_text + url` 生成可点击标题或简称，并标注来源/期刊、年份和来源级别，例如：

`[R1] [DAPA-CKD]({{url}})（NEJM，2020，A 级来源/高质量 RCT）`

不得只写编号、PMID、DOI、纯文本题名或裸 URL。只能读到摘要时明确标注 `仅摘要/辅助参考`。飞书 XML 不使用 HTML 上标标签；可用普通 `[G1]`、`[R1]` 或 citation 样式组件，但文献名本身仍须可点击。

引用诚信要求：

- 每个引用只支撑原文实际能支撑的句子，区分背景、核心结论、安全、争议、方法和研究空白用途。
- 不根据摘要扩展未报告的数据，不把来源类型写错，不把方向不一致的研究笼统写成“研究均显示”。
- 直接引用只保留必要短片段并标注来源；不复制长段原文。
- 正文出现的每个 `evidence_id` 都能回到同一 CSV 记录、可点击来源和参考文献条目。
- 无论用户要求 Vancouver、AMA、APA 或 GB/T 7714，最终展示的每条文献仍必须有可点击原文/摘要入口；具体参考文献表结构见 `output-and-qa.md`。

## 7. 证据表字段

完整证据表至少包含：

| 字段 | 填写规则 |
|---|---|
| evidence_id | G/SR/R/N/S/T/C 编号 |
| 类型 | 指南、共识、系统综述、RCT、队列、诊断、安全、教材等 |
| 标题 | 原文标题，超链接到原文或入口 |
| 来源/期刊与年份 | 学会、机构、期刊、数据库或教材及年份 |
| 来源级别与阅读状态 | A/B/C/D；全文、指南正文、PDF 正文、摘要或无法访问 |
| 证据级别/推荐强度 | 原文分级；未报告或不适用时如实填写 |
| 关键结果/推荐 | 与本次问题直接相关的结果或推荐 |
| 主题相关性与引用用途 | 高/中/低；背景、核心结论、争议、空白或论文段落 |
| 局限性 | 偏倚、样本量、终点、外推、年份或可及性 |

可用 `scripts/validate-evidence-table.js` 检查结构化 JSON 证据表。
