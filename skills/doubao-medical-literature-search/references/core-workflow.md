# 核心流程

本文件只管“如何理解用户意图、如何拆解检索任务、如何组织综合”。证据来源和质量评价见 `evidence-sources-and-appraisal.md`，图表见 `synthesis-and-visualization.md`，交付见 `output-and-qa.md`。

## 1. 判断任务类型

先用一句话复述用户真正要解决的问题，例如：

`我理解本次要回答的是：围绕 {{主题/疾病/药物/问题}}，系统检索并综合 {{指南/关键研究/最新进展/热点/论文引用}}，用于 {{学习/课题/论文/汇报/临床实践参考}}。`

将任务归入一个主场景，可有 1-2 个辅场景：

| 场景 | 用户常见说法 | 核心产出 |
|---|---|---|
| 主题循证综述 | 查某疾病、症状、检查、治疗、科室问题的文献 | 结论、指南、关键研究、证据等级、临床意义 |
| 指南/共识查新 | 查最新指南、专家共识、推荐变化 | 指南要点解读、推荐差异、更新点、适用边界 |
| 研究热点/前沿 | 分析某领域研究热点、趋势、最新进展 | 主题簇、时间线、里程碑研究、前沿问题 |
| 课题论证 | 分析某课题有没有价值、研究基础和空白 | 研究现状、争议、空白、可行研究方向 |
| 论文写作支持 | 为论文引言、讨论、综述补文献或引用 | 引用逻辑、可用文献表、段落草稿、参考文献 |
| 学习资料 | 整理某病种/药物/指南用于进阶学习 | 知识框架、指南要点、关键研究、学习路径 |
| 文献支持型研究/论文问题 | 用文献支撑研究设想、论文段落、综述观点或课题论证 | 文献证据链、引用来源、引用用途和局限 |

若用户要某个具体患者/临床问题的诊断、检查、用药或治疗答案，转向 `doubao-clinical-decision-support`；若患者上传报告并主要询问异常含义、复查或就医沟通，转向 `doubao-medical-report`。若用户只是借病例限定文献检索人群，最终要的是综述、研究进展、课题或引用清单，继续本 Skill。

若用户明确只要“精简回答”“概括一下”“300 字以内”“小结”或“一段话”，不要进入完整文献检索交付；仅做 1-2 个高价值批次的轻量检索，以普通聊天简短回答，并为关键判断保留可点击来源链接。不创建飞书文档、PPT、Word 或 CSV。用户明确要求 Markdown、表格或不要创建文档，但仍要求完整主题检索、综述或调研时，继续走完整检索流程，只是将该格式作为目标产物。

## 2. 界定检索边界

优先从用户原话中抽取边界；信息不足时使用透明默认值，不要反复追问。

| 边界 | 默认做法 |
|---|---|
| 人群/疾病 | 使用用户给出的疾病、症状、科室、药物或研究对象；缺失时按通用成人/常见临床场景处理并说明 |
| 干预/暴露 | 药物、手术、检查、危险因素、护理措施、管理策略、机制或研究变量 |
| 对照/参照 | 标准治疗、安慰剂、无暴露、其他方案、健康对照或历史研究 |
| 结局 | 疗效、安全性、死亡、复发、住院、诊断性能、生活质量、指南推荐、研究趋势 |
| 时间范围 | 最新进展默认近 5 年；指南查新默认近 5-10 年并优先最新版；里程碑研究不受年份限制 |
| 语言/地域 | 中英文优先；涉及中国实践时加入中文指南、专家共识和本土研究 |
| 文献类型 | 按任务选择指南、共识、系统综述、RCT、队列、真实世界研究、诊断研究、机制研究、教材/专著、安全来源 |
| 输出深度 | 未说明时输出专业完整报告；用户要“快速”时仍保留原文链接和证据等级 |

复杂问题拆成 2-5 个可检索子问题，避免一个检索式包打天下。

## 2.1 上下文预算与效率

先估算任务深度和可用上下文，再决定检索轮次与 Fetch 深度。目标是覆盖关键证据，而不是穷尽所有搜索结果。

| 任务深度 | 检索预算 | Fetch/精读预算 |
|---|---|---|
| 轻量回答 | 1-2 个高价值批次；每批 3-6 条候选 | 通常只打开摘要页/官方入口，确认题名、年份、来源和链接；不创建 CSV 或文档 |
| 默认完整报告 | 2-4 个高价值批次；初筛候选总量约 12-36 条 | 定向精读 1-3 份核心指南/综述/关键研究；其他资料按需阅读摘要、推荐条文、主要结果、关键表格和局限 |
| 复杂课题/查新 | 一般不超过 5 个批次；需要更多时说明原因 | 只对会进入核心结论、图表或正式参考文献的文献做深读 |

除非用户明确要求系统综述级别穷尽检索、文献计量或专项查新，不要反复扩展同义词、连续搜索很多轮，或把低相关网页内容读进上下文。若检索结果已经足以回答问题，停止扩大检索，进入筛选、阅读和综合。

## 3. 选择问题框架

- 治疗、管理、预防问题：用 PICO/PICOTS。
- 病因、危险因素、暴露问题：用 PECO。
- 诊断问题：写清目标疾病、待评价检查、参考标准和诊断性能指标。
- 领域/热点/范围综述：用 PCC（Population, Concept, Context）。
- 论文写作：用“研究问题 + 论文段落用途 + 需要支撑的论点”。
- 学习资料：用“学习对象 + 临床场景 + 知识层级 + 必须覆盖的指南/教材/实践问题”。

输出检索问题时尽量写成 1 句话和一张小表，便于后续 CSV、证据表和图表引用。该小表后续必须进入“检索问题与范围”章节，并在同章精简展示检索来源、核心关键词、检索效率和证据覆盖局限。

## 4. 关键词扩展

每个子问题都做中英文扩展：

- 疾病：中文名、英文名、缩写、旧称、分型、严重程度、MeSH/Emtree 近似词。
- 症状/体征：同义词、常见描述、鉴别诊断方向。
- 药物：通用名、商品名、靶点、药物类别、中英文名、适应证。
- 干预/检查：技术名、别名、设备/方法名、路径或策略名。
- 结局：临床终点、替代终点、安全事件、患者报告结局、诊断指标。
- 研究类型：guideline, consensus, systematic review, meta-analysis, randomized trial, cohort, real-world, registry, diagnostic accuracy, safety。

关键词扩展应提高召回，不要把检索边界扩到无关领域。每个检索批次记录实际 query；最终文档只在“检索问题与范围”章节精简展示 3-8 个核心中英文关键词、缩写或 MeSH/Emtree 近似词，不铺开完整检索流水。

## 5. 检索批次

按任务需要组合批次；工具调用和优先来源边界遵循 `evidence-sources-and-appraisal.md` 的“检索顺序”。默认 2-4 个高价值批次，轻量回答 1-2 个批次即可，复杂课题一般不超过 5 个批次：

1. 指南/共识批次：国际/国内学会、政府、权威机构、最新指南和共识。
2. 综述证据批次：Cochrane、系统综述、Meta 分析、证据总结。
3. 关键研究批次：RCT、大型队列、真实世界研究、诊断研究、里程碑研究。
4. 最新进展批次：近 1-5 年高相关研究、新药/新技术、指南更新。
5. 安全/监管批次：药品说明书、FDA/EMA/NMPA、禁忌、警示、特殊人群。
6. 本土/专科实践批次：中文指南、专家共识、本土队列、专科规范。
7. 论文写作批次：围绕引言背景、研究空白、方法依据、讨论比较分别检索。
8. 学习资料批次：教材/权威综述、指南要点、实践流程和经典研究。

完整检索流程中，每轮检索结果先筛掉明显无关、重复、低质量来源，再把高相关候选写入 CSV；默认每批次保留 4-10 条候选，完整报告初筛候选总量通常控制在 12-36 条。不要只保存“看起来有用”的核心证据，也不要等全部报告写完才补 CSV；但也不要把低相关搜索结果全部写入上下文。每个检索批次结束后执行一个短检查：CSV 文件已保存、行数增加、候选记录保留了标题/来源/链接/摘要或摘要占位。若本轮工具返回了高相关候选但 CSV 行数没有增加，立即暂停后续分析并补录。轻量回答不创建 CSV。

## 6. 初筛与精读

初筛顺序：

1. 题名和来源是否相关。
2. 文献类型是否符合任务目的。
3. 年份、机构、期刊和研究设计是否有价值。
4. 摘要或简介是否回答子问题。
5. 是否有可点击原文、官方入口、DOI/PMID 或可追溯页面。

初筛后为所有拟进入正文或图表的核心证据建立引用地图：`evidence_id -> anchor_text -> url -> 来源/年份/级别 -> 支持结论`。正文、图例、证据表和文末参考文献表都使用这张引用地图。没有可点击 `url` 或没有可用 `anchor_text` 的文献，不能作为正文核心证据。完整报告在 1-3 篇最能支撑结论的资料上完成定向精读；其余纳入条目应依据可访问层级明确标注为全文、指南正文、摘要或待核验。

Fetch/精读核心证据时按需记录，不把整页或整篇 PDF 原样塞入上下文。优先读取与问题直接相关的摘要、推荐条文、主要结果、关键表格、局限和链接入口；对背景性或仅摘要证据，只记录题名、来源、摘要要点和链接。

精读核心证据时记录：

- 指南：推荐条文、推荐强度、证据等级、适用人群、更新时间和差异点。
- 系统综述/Meta：PICO、纳入研究、效应量、异质性、偏倚、结论边界。
- RCT/队列：入排标准、样本量、干预/暴露、对照、主要终点、效应量、安全性、亚组和局限。
- 诊断研究：参考标准、敏感度、特异度、AUC、LR、PPV/NPV、误判后果。
- 机制/基础研究：只用于机制解释或研究假设，不直接支撑临床推荐。
- 教材/临床实践资料：用于学习框架和背景，不替代最新指南与原始研究。

## 7. 综合方式

根据任务类型选择主结构：

- 主题循证综述：`结论 -> 指南与专家共识 -> 综述证据 -> 关键研究 -> 最新进展 -> 临床意义 -> 局限`。
- 指南查新：`最新版推荐 -> 与既往/不同指南差异 -> 证据依据 -> 实践影响`。
- 热点分析：`主题簇 -> 时间线 -> 里程碑 -> 最新前沿 -> 争议 -> 研究空白`。
- 课题论证：`研究背景 -> 现有证据 -> 未解决问题 -> 可行研究问题 -> 方法建议`。
- 论文写作：`可支撑论点 -> 引用文献 -> 可写段落 -> 参考文献 -> 引用风险`。
- 学习资料：`知识框架 -> 指南要点 -> 经典研究 -> 实践路径 -> 推荐阅读`。

综合时必须把“研究发现”翻译成用户能用的临床、科研或学习意义；不要停留在文献摘要搬运。

综合完成后必须进入图表设计步骤。只要默认输出飞书文档、Word/docx、正式 Markdown 或 PPT，就必须读取 `synthesis-and-visualization.md` 并生成固定证据图组。不要因为用户说“只要题目”“简单检索”“从 PubMed 找一下”而跳过图表；这类表达只影响正文深度，不取消正式文档的图表要求。

默认飞书长文档先生成一份完整 XML 草稿并通过校验，再按 `section-1`（标题、结论、检索问题与范围、证据脉络与研究关系）、`section-2`（指南与专家共识、系统综述与关键研究）、`section-3`（最新进展/热点/争议/空白、学习/课题/论文写作建议、完整参考文献、免责声明）拆成 3 段，通过 `lark-doc` 先创建再追加/插入。不要一次性写入超长正文；也不要把 3 段拆成 3 个独立文档。若追加或插入失败，保留失败 section 的 XML 文件并从该段重试。默认飞书文档取得文档 ID 后，必须按 `output-and-qa.md` 从 CSV 所在目录用相对路径把同一 CSV 作为文件附件插入文档末尾；插入类型必须为 file，结构化参数必须等价于 `"type": "file"`；不要把 CSV 全量内容展开为正文表格。

## 8. 完成条件

默认任务只有在以下条件满足时才算完成：

- 已生成目标产物：未明确格式时必须是通过 `lark-doc` Skill 创建或更新的飞书文档，并已拿到文档链接、文档标识或可打开入口；用户明确要求时才可改为通过 `lark-ppt` Skill 创建的 PPT/幻灯片、通过 `lark-doc` Skill 的 `office-word` 子模块创建的 Word/docx，或用户指定的轻量格式。
- 默认飞书文档已将 CSV 作为文件附件插入文档末尾；若附件插入失败，最终回复已简短说明，且未列出本地 CSV 或台账目录。
- CSV 只是文献台账和附录来源，不能作为默认主产物。只生成 CSV、只给 PMID/题名清单、只在聊天窗口输出文献列表，都不满足完成条件。
- 若默认飞书文档创建失败，必须明确标注 `飞书文档创建失败`，说明 `lark-doc` 不可用或具体错误，并提供可重试飞书 XML 草稿文件路径或完整 XML 草稿；不能静默降级为 CSV 或聊天正文。
- 已创建并校验本地 literature-log CSV，且 CSV 至少包含 1 条候选文献/来源记录；只有表头的空 CSV 不能交付。
- 关键结论和图表节点能追溯到 CSV、正文链接和参考文献表。
- 默认飞书文档或正式文档包含论文关系/核心逻辑梳理图和关键研究里程碑时间线图；候选/纳入文献达到 5 条以上时，缺少这两张主图视为未完成。系统综述、关键研究、最新进展、热点、争议与空白必须在后续章节用逐篇或逐主题段落解读，不能只给题名清单或参考文献表。
- 完整产物在“检索问题与范围”中精简说明检索来源/数据库、核心关键词、检索批次/效率、纳入排除标准和证据覆盖局限；只展示外部数据库、网站或来源渠道，不展示内部工具名称。不再单独设置检索方法/关键词/证据覆盖类一级章节。
- 正文、图例、证据表中提及的具体文献标题或简称均就地带超链接；不能只靠文末参考文献表提供链接。
- 文末参考文献表每条正式参考文献都有可点击的原文、摘要页、DOI、PubMed、期刊页、指南官网或官方入口链接之一，无论采用哪种参考文献格式；无任何可点击入口的资料不能进入最终参考文献表。
- 证据不足、无法访问原文或存在争议时已明确标注。
