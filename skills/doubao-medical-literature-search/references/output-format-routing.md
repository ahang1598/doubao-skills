# 输出格式路由

本文件只管“用户要哪种产物时，选择什么模板、怎么创建、怎么校验”。默认飞书文档 QA 见 `output-and-qa.md`。

## 选择规则

| 用户意图 | 默认产物 | 模板 | 适用场景 |
|---|---|---|---|
| 未指定格式、格式含糊、只提出分析/检索/整理需求 | 飞书文档 | `assets/feishu-doc-template.xml` | 文献综述、指南查新、热点分析、课题论证、学习资料 |
| 说“PPT、汇报、组会、教学、幻灯片” | PPT/幻灯片（使用 `lark-ppt` Skill） | `assets/ppt-deck-template.xml` | 科室汇报、研究生组会、教学课件、项目汇报 |
| 明确要求“Word、docx、正式报告、投稿附件、归档” | 可打开的 Word/docx（通过 `lark-doc` Skill 的 `office-word` 子模块） | `assets/word-report-template.md` | 正式文献综述、课题附件、论文写作支持、可打印归档 |
| 明确要求 Markdown、表格或不要创建文档，且仍要求完整主题检索/综述/调研 | 按用户要求 | 保留链接和证据等级 | 临时检索、轻量调研 |
| 明确要求精简回答、概括、小结、300 字以内或一段话 | 普通聊天回答 | 不加载文档模板 | 仅做 1-2 个高价值批次的轻量检索；保留关键来源链接，不建文档或 CSV |

不确定时按飞书文档执行。只要用户没有明确表达预期格式，就默认创建飞书文档；不要为了方便把默认专业交付降级为聊天窗口正文、Markdown、表格、Word 或 PPT。大量文献列举、候选清单、筛选记录和完整参考文献展开必须放入文档/附录/CSV，不在聊天窗口铺开。

默认飞书文档不是可选增强，而是未指定格式时的主交付物。CSV 是配套文献台账文件：默认飞书文档创建后直接作为文件附件插入文档末尾，最终回复不单独列出本地 CSV、文献台账或台账目录；只输出 CSV、PMID/题名列表或聊天摘要时，必须视为未完成并返回飞书创建步骤。该完成条件不适用于明确的轻量短答。

## 渐进式披露

完整检索流程先完成共通步骤：任务分类、检索边界、关键词扩展、CSV 台账、预算内检索、按需原文阅读、证据评价、综合和图表设计。每轮检索后先写入高相关候选 CSV 并确认行数增加，再进入筛选和阅读。轻量短答只做 1-2 个高价值批次并保留来源链接，不创建 CSV 或加载模板。只有准备完整交付时才加载对应模板：

- 飞书文档：加载 `assets/feishu-doc-template.xml`。
- PPT/幻灯片：加载 `assets/ppt-deck-template.xml`，并按 `lark-ppt` Skill 要求先做页面规划。
- Word/docx：加载 `assets/word-report-template.md`，然后使用 `lark-doc` Skill，并按其顶层路由进入 `office-word/office-word-skill.md` 子模块生成真正 `.docx`。

完整交付格式必须包含：直接回答用户问题的结论、检索范围、精简固定可视化组（论文关系/核心逻辑梳理图、关键研究里程碑时间线图）、指南与专家共识解读、系统综述/关键研究逐篇解读、最新进展/热点/争议/空白解读和参考文献。文末参考文献默认使用 `标题｜来源｜类型｜年份｜链接` 五列表格，且每条有可点击原文/摘要入口。默认飞书文档不单独设置检索方法/关键词/证据覆盖类一级章节，检索来源、核心关键词、检索效率和证据覆盖局限精简并入“检索问题与范围”。最终回复不单独列出文献台账 CSV、本地路径或台账目录；默认飞书文档需把该 CSV 作为文件附件插入文档末尾。

## 飞书文档

飞书文档适合完整、可追溯、图文并茂的文献分析。

创建规则：

- 使用已有 `lark-doc` Skill 创建或更新文档，并取得飞书文档链接、文档标识或可打开入口。
- 默认 XML；用户明确要 Markdown 时才用 Markdown。
- 长报告先建章节骨架，再按章节插入或更新。
- 使用结论段/结论 callout、论文关系/核心逻辑梳理图、里程碑时间线图、逐篇文献解读和参考文献分组；第一章 callout 标题用“总结”，并优先用无序列表呈现检索结论、实践启示、研究局限和空白。
- 飞书文档 XML 禁止使用 HTML 上标标签；引用编号和脚注改用正文括号、可点击文献名或参考文献表。生成 XML 后先运行上标标签清理和校验，再调用 `lark-doc`。
- 正文、证据表、图例和建议段落提到具体文献/指南/试验时必须就地给出可点击标题或简称，例如 `[G1] <a href="{{url}}">{{文献名}}</a>`；文末参考文献链接不能替代正文链接。
- 正文证据章节必须插入精简固定可视化组：论文关系/核心逻辑梳理 Mermaid 白板图和关键研究里程碑时间线图。关系图可用多样十六进制语义色；时间线图优先按“横向年份/阶段轴 + 时间节点下方纵向研究卡片列”呈现；若年份过多或大量年份各只挂 1 篇文献，合并相邻年份为时间段/阶段节点，让每个时间节点尽量挂 2-5 个代表文献。
- 图中每个文献节点必须包含研究简称或 evidence_id，如 `[R1] 研究简称`、`[G1] 指南简称`，并通过图例表或同页注释提供链接。
- 指南与专家共识、系统综述、关键研究和最新进展章节优先用逐篇/逐主题段落解读。复杂任务可补充争议矩阵表、研究空白表、指南对比表或引用逻辑表；只有用户明确要求或能稳定映射为飞书标准图型时，才额外画图。
- 产物正文中不展示本地 literature-log CSV 路径；最终回复中也不单独列出本地 CSV、文献台账或台账目录。
- 在“检索问题与范围”中精简介绍检索方法和主要关键词，至少包含检索来源/数据库、核心关键词、纳入排除标准、证据覆盖、检索批次数、Fetch/原文阅读取材范围和停止扩展理由；只展示外部数据库/网站/来源渠道，不展示内部工具名称。
- 默认飞书文档创建成功并取得文档 ID 后，先进入 CSV 所在目录，用相对路径把同一 CSV 作为文件附件插入文档末尾；插入类型必须为 file，结构化参数必须等价于 `"type": "file"`；不要把 CSV 全量内容、CSV 预览表或 CSV 交付检查表展开到正文。
- 若 `lark-doc` 不可用或创建失败，最终回复必须写 `飞书文档创建失败`，给出原因和可重试飞书 XML 草稿入口；不得改成 CSV-only。

## PPT/幻灯片

PPT/幻灯片必须通过 `lark-ppt` Skill 创建。它是汇报材料，不是把报告压缩成标题和项目符号。创建前按 `lark-ppt` Skill 要求写页面规划，每页明确 `layout_type`、`visual_focus`、`text_density`、核心信息和引用来源。

推荐 10-14 页结构：

1. 封面：主题、用途、日期。
2. 结论先行：3 条以内主结论和证据确定性。
3. 检索问题与范围：PICO/PCC/论文论点、纳入排除，并精简展示检索来源、核心关键词、检索效率和覆盖局限。
4. 论文关系/核心逻辑梳理图：指南、综述、关键研究、最新进展如何汇聚到核心逻辑和结论。
5. 指南/共识要点：逐条解读推荐方向、等级、更新点和差异。
6. 关键研究里程碑时间线图：优先做横向年份/阶段轴，时间节点下方纵向排列研究节点；年份很多时合并为时间段，节点写 evidence_id 或研究简称，避免一堆年份各只挂 1 篇文献。
7. 系统综述与关键研究：逐篇解释研究设计、主要结论、局限和引用用途。
8. 研究热点或主题簇：逐主题解释方向、代表文献、变化和证据边界。
9. 学习/课题/论文写作启示：优先用分点无序列表，突出学习路径、课题机会、引用用途和引用风险。
10. 参考文献：核心证据；每条正式参考文献必须有原文/摘要页/DOI/PubMed/期刊页/官方入口之一的可点击链接。

设计约束：

- 画布按 960 x 540 规划。
- 每页一个主信息；正文不超过 5 条要点。
- 4 页以上至少使用 4 种版式，如封面、结论页、逻辑关系页、时间线、矩阵表、参考文献页。
- 关键图表优先使用原生 `shape`、`line`、`table` 或原生图表组合绘制。
- Mermaid whiteboard 只作增强，不能作为唯一主图。
- 每页引用 evidence_id 或文献简称；正文页提到具体文献时，文献标题/简称优先做成可点击链接，至少同页脚注或末页参考文献必须有可点击链接，不能只给编号。
- 时间线页和论文关系/核心逻辑图页必须在节点中显示研究简称或 evidence_id，而不是只显示概念词。
- 生成后回读/校验页数、主体元素、空白页、重叠、溢出和引用入口。

## Word/docx

Word 适合正式、可打印、可归档的文献综述或论文支持材料。用户要求 Word/docx 时，必须使用 `lark-doc` Skill，并按 `lark-doc` 顶层路由进入其 `office-word/office-word-skill.md` 子模块，默认交付真正可打开的 `.docx` 文件；不要改用独立的 office-word 工具名、泛泛的 Word 工具或把 Markdown 改名成 docx。Markdown 只作为 `lark-doc` / `office-word` 不可用时的可重试草稿。

推荐结构：

1. 标题页与文档信息。
2. 结论。
3. 检索问题与范围（含精简检索方法与关键词）。
4. 论文关系/核心逻辑图说明。
5. 关键研究里程碑时间线图。
6. 指南与专家共识。
7. 系统综述与关键研究。
8. 最新进展、热点、争议和空白。
9. 论文写作/课题/学习建议。
10. 参考文献。

版式要求：

- A4，页边距约 2.5 cm。
- 正文宋体或等效中文正文字体，小四或 11-12 pt，1.5 倍行距。
- 标题层级清晰，自动编号或稳定编号。
- 表格列宽稳定，不横向溢出。
- 图表有编号、题注和图例表；打印后仍能追溯 evidence_id。
- 参考文献格式按用户要求；未指定时默认使用 `标题｜来源｜类型｜年份｜链接` 五列表格。无论 Vancouver、AMA、APA、GB/T 7714 或其他格式，文末每条正式参考文献都必须带原文/摘要页/DOI/PubMed/期刊页/官方入口之一的可点击链接。正文中被引用的文献标题或简称也必须就地带超链接，不只在文末给链接；没有任何可点击入口的资料不能进入最终参考文献表。
- 生成后检查 `.docx` 文件可打开、标题层级正常、表格不溢出、链接可见、图表有图例；最终回复给出 Word 文件入口和关键说明，不单独列出文献台账 CSV。

## Markdown/表格

仅在用户明确要求且仍需要完整主题检索/调研时使用，保留：

- 结论和证据确定性。
- 核心文献表，含链接、类型、来源、年份和证据级别。
- 检索范围和局限。
- 检索方法和主要关键词。
- 参考文献默认按 `标题｜来源｜类型｜年份｜链接` 呈现，且每条都有链接。

不要把轻量格式变成无来源的观点回答。若文献数量很多，聊天窗口只给摘要和入口；完整文献清单仍放入目标文档。明确的精简回答、概括、小结、300 字以内或一段话请求不进入本节，而走普通聊天的 1-2 批轻量检索路径。

## 失败兜底

如果目标格式工具不可用：

- 明确说明失败原因。
- 不要以“稍后生成”结束。
- 提供对应模板的完整可重试草稿：飞书 XML、PPT XML，或供 `lark-doc` Skill 的 `office-word` 子模块重试的 Word/docx Markdown 草稿。
- 最终回复仍保持简短，给出草稿路径和关键说明，不单独列出文献台账 CSV。
