# 综合与图表规范

本文件用于决定“如何把文献变成证据逻辑、研究脉络和专业可视化”。证据节点必须来自文献台账 CSV 或已说明的综合主题，不得凭空生成。图表策略采用飞书标准 diagram 思路：先判断图要回答什么问题，再选择最小可行图型；关系图节点和边必须有语义，边要有标签；不稳定、难渲染或非标准的复杂图，默认改为正文段落或结构化表格。

## 触发规则

满足任一条件时必须提供图表：

- 核心纳入证据达到 5 条以上。
- 用户要求研究热点、前沿进展、课题设计、论文写作、教学材料或汇报。
- 存在多条指南、多个治疗/诊断/研究方向或多个研究类型。
- 文献结论不一致、指南更新与旧实践存在张力、证据不足或研究空白明显。
- 用户需要 PPT、飞书文档或 Word 正式材料；其中 PPT/幻灯片通过 `lark-ppt` Skill，Word/docx 通过 `lark-doc` Skill 的 `office-word` 子模块。

文档类产物（默认飞书文档、Word/docx、正式 Markdown 文档）默认包含一个精简固定可视化组，放在“证据脉络与研究关系”附近，而不是只放在附录：

1. `论文关系/核心逻辑梳理图`：用 Mermaid 白板展示指南、综述、关键研究、最新进展、安全/监管证据如何围绕核心机制、变量、主题或科学问题形成证据脉络；可用多样十六进制语义色区分节点类型。
2. `关键研究里程碑时间线图`：按年份或研究阶段横向排列里程碑，并在时间节点下方纵向列出研究/指南/综述/监管节点；年份多时必须合并为时间段，避免一堆年份各只挂一篇文献。

论文关系/核心逻辑梳理图和时间线图是文献报告的默认主图，除非用户明确要求极简输出或证据少到无法构成时间线。图中的文献节点必须写研究简称或 evidence_id，例如 `[R1] DAPA-CKD`、`[SR2] Cochrane 2023`、`[G1] KDIGO 2024`；概念节点可以只写机制/主题名，但相邻证据节点必须能追溯到 CSV。系统综述、关键研究、最新进展、热点、争议与空白默认放在后续章节逐篇或逐主题解读，不在“证据脉络与研究关系”中重复放大表。除这组主图外，不默认再生成额外白板图。

反模式：检索到 5 条以上文献后，以“用户只要完整题目”“需求简单”“先做简洁文档”为理由，只生成文献题名表、CSV 或参考文献页而不生成论文关系图和时间线图。这种输出不合格，必须回到图表设计步骤补图。

若证据少于 3 条，也要给简化版论文关系图；时间线无法成立时用简短里程碑段落说明，并保留逐篇解读，不要完全省略证据脉络。

## 飞书标准图型取舍

默认只把以下内容做成图：

| 需求 | 首选表达 | 说明 |
|---|---|---|
| 领域如何演进 | 关键研究里程碑时间线图 | 标准时间线；年份多时合并阶段 |
| 文献/机制如何构成证据脉络 | 论文关系/核心逻辑梳理图 | Mermaid 白板；节点和边都要有语义，可用十六进制语义色 |

其它内容默认不画复杂图，改为正文段落或必要的标准表格：

| 需求 | 首选表达 |
|---|---|
| 核心研究怎么解读 | 每篇核心文献一个小节 |
| 指南推荐有何差异 | 逐份指南/共识段落解读，必要时补指南对比表 |
| 热点方向有哪些 | 逐主题段落，必要时补热点主题表 |
| 证据为何冲突 | 争议矩阵表 |
| 还缺哪些研究 | 研究空白表 |
| 论文哪句话引用哪篇文献 | 引用逻辑解读，必要时补对照表 |
| 学习材料怎么组织 | 学习路径段落或知识框架，必要时补简表 |

只有在用户明确要求“画图”、或内容天然属于飞书标准图型（流程图、决策树、时间线、矩阵、思维导图等）且能稳定显示时，才额外生成图。额外图必须说明图型选择理由，并保持节点少、标签短、图例完整。

默认组合：

- 主题循证综述：`论文关系/核心逻辑梳理图 + 关键研究里程碑时间线图 + 系统综述/关键研究逐篇解读`。
- 指南查新：`论文关系/核心逻辑梳理图 + 关键研究里程碑时间线图 + 指南/共识逐份解读`；只有多份指南需要横向比较时再补指南对比表。
- 热点/前沿：`论文关系/核心逻辑梳理图 + 关键研究里程碑时间线图 + 逐主题进展/争议/空白解读`；只有用户要求图形化热点时再补主题簇图。
- 课题论证：`论文关系/核心逻辑梳理图 + 研究空白解读 + 可行研究问题建议`；必要时补简表。
- 论文写作：`引用逻辑解读 + 参考文献用途建议`；必要时补引用对照表，不默认做额外白板图。
- 学习资料：`知识框架 + 学习路径建议`；必要时补简表，不默认画复杂知识图谱。

## 图表选择

| 表达 | 回答的问题 | 适用任务 | 默认形式 |
|---|---|---|
| 关键研究里程碑时间线图 | 领域证据如何演进，哪些研究是里程碑，最新研究改变了什么 | 所有文档类产物、热点分析、指南查新 | 飞书标准图 |
| 论文关系/核心逻辑梳理图 | 关键机制、变量、主题和文献证据如何汇聚为核心科学假设或结论 | 所有文档类产物、课题论证、学习资料 | Mermaid 白板 |
| 关键研究逐篇解读 | 哪些研究最值得读，它们各自贡献、局限和引用用途是什么 | 所有文档类产物、论文写作、课题论证 | 正文小节 |
| 指南差异解读/可选对比表 | 不同指南推荐是否一致，推荐强度和适用人群如何不同 | 指南查新、临床实践学习 | 正文小节，必要时表格 |
| 热点主题解读 | 文献集中在哪些方向，近年有哪些新主题 | 研究热点、课题论证 | 正文段落，必要时表格 |
| 争议矩阵 | 证据为何不一致，争议来自人群、干预、终点还是研究质量 | 课题设计、讨论章节 | 表格 |
| 研究空白表 | 哪些人群、终点、场景、方法或随访证据不足 | 课题论证、基金/论文选题 | 表格 |
| 引用逻辑解读/可选对照表 | 每组文献支撑论文哪个论点或段落 | 论文写作支持 | 正文建议，必要时表格 |
| 学习路径解读/可选简表 | 先学什么指南、再读哪些经典研究、最后看哪些前沿 | 进阶学习资料 | 正文小节，必要时表格 |

不要把所有内容塞进一张图。节点超过 15 个、证据节点超过 10 个、分支超过 6 个或连线交叉明显时拆成多图。

## 可选证据地图

证据地图回答：“本次结论由哪些证据构成，各证据的强弱和作用是什么？”

证据地图不作为默认额外白板图。若论文关系/核心逻辑梳理图已经覆盖“证据来源 -> 结论”的关系，优先用逐篇解读或简要证据表承载证据地图信息；只有用户明确要求证据地图，或需要展示多类证据共同支持/限制多个结论时，才额外生成标准关系图。

必须包含：

- 中心节点：用户问题或主题。
- 证据节点：CSV 中纳入的 `G/SR/R/N/S/T/C` evidence_id。
- 结论节点：`相对确定`、`倾向支持`、`存在争议`、`证据不足`、`不支持` 或任务化结论。
- 连线标签：`指南推荐`、`系统综述支持`、`RCT 支持`、`观察性补充`、`安全限制`、`证据冲突`、`仅摘要辅助`、`研究空白`。
- 图例表：列出 evidence_id、标题、链接、来源/期刊、年份、来源级别和阅读状态。

证据地图不能只画“文献 A -> 文献 B -> 结论”的链条；必须表达证据角色。

## 论文关系/核心逻辑梳理图

论文关系/核心逻辑梳理图回答：“关键文献、机制、变量或主题如何共同形成核心科学逻辑，并怎样支持、限制或补充本次结论？”

必须包含：

- 中心主题或研究问题。
- 指南/共识、系统综述、关键原始研究、最新进展、安全/监管或机制证据节点。
- 关系标签：`奠定背景`、`汇总证据`、`直接支持`、`扩展人群`、`提出冲突`、`安全限制`、`提示空白`、`推动指南更新`。
- 图例表：evidence_id、标题、链接、来源/期刊、年份、类型、用途。

优先使用“中心机制/主题汇聚图”样式：上方放 2-4 个关键驱动因素或证据来源，中间放核心通路/主题/科学假设，下方放主要结果、结局或研究空白；箭头显示方向和关系。文献节点必须带 evidence_id 或研究简称，概念节点旁边至少有一个证据节点支撑。

节点标签格式：

- 文献节点：`[R1] DAPA-CKD`、`[G1] KDIGO 2024`、`[SR2] Cochrane 2023`。
- 概念节点：`Wnt/beta-catenin 通路`、`免疫微环境`、`诊断性能`。
- 结果节点：`复发风险下降`、`骨破坏加重`、`研究空白`。

禁止只按年份把文献串起来；那是时间线，不是关系图。关系图要体现文献之间的证据作用、机制汇聚、变量关系和逻辑递进。

Mermaid 白板模板：

```xml
<whiteboard type="mermaid">
flowchart TD
    A1[[G1 指南简称]]:::guideline
    A2[[SR1 综述简称]]:::review
    A3[[R1 关键研究简称]]:::study
    A4[[N1 最新研究简称]]:::new
    Core[核心机制或研究问题]:::topic
    O1[主要结局或主题 1]:::outcome
    O2[主要结局或主题 2]:::outcome
    Gap[研究局限和空白]:::gap
    A1 -->|推荐背景| Core
    A2 -->|汇总证据| Core
    A3 -->|直接支持| Core
    A4 -->|新变化| Core
    Core -->|影响| O1
    Core -->|影响| O2
    O1 -->|提示| Gap
    O2 -->|争议| Gap
    classDef topic fill:#DBEAFE,stroke:#2563EB,color:#1E3A8A;
    classDef guideline fill:#EDE9FE,stroke:#7C3AED,color:#4C1D95;
    classDef review fill:#DCFCE7,stroke:#16A34A,color:#14532D;
    classDef study fill:#CCFBF1,stroke:#0F766E,color:#134E4A;
    classDef new fill:#FEF3C7,stroke:#D97706,color:#78350F;
    classDef outcome fill:#E0F2FE,stroke:#0284C7,color:#0C4A6E;
    classDef gap fill:#FEE2E2,stroke:#DC2626,color:#7F1D1D;
</whiteboard>
```

## 关键研究里程碑时间线图

关键研究里程碑时间线图用于里程碑和最新进展，优先做成类似“上方年份/阶段色块 + 中间横向时间轴 + 下方按时间节点纵向排列研究卡片”的标准时间线，而不是阶段演进流程图或只有表格：

- 按年份或研究阶段排列关键指南、经典研究、重要综述、监管事件和最新研究。
- 若总年份节点超过 7 个、年份跨度超过 8-10 年，或多个年份各只有 1 篇文献，优先把相邻年份合并为时间段/阶段节点；若 5-7 个年份节点每个都能挂 2 条以上代表证据，可以保留单年节点以形成清晰横向年份轴。常用分组如 `早期基础研究`、`机制验证阶段`、`关键临床研究阶段`、`指南更新阶段`、`近 5 年最新进展`。
- 每个时间段下方尽量纵向列出 2-5 个代表节点；只有真正改变指南、监管、安全警示或研究范式的事件，才保留为单一年份节点。
- 节点文本必须包含 evidence_id 或研究简称，如 `[R1] DAPA-CKD`、`[G2] ESC 2025`。
- 标记每个节点的意义：`奠定标准`、`改变指南`、`扩展人群`、`提出争议`、`安全警示`、`最新进展`。
- 经典里程碑不受年份限制；最新进展默认近 1-5 年。
- 不要把普通低质量研究放成里程碑。
- 反模式：`2020 -> 2021 -> 2022 -> 2023 -> 2024 -> 2025` 且每个年份只挂 1 篇文献。应改为 `2020-2022 机制与观察性证据`、`2023-2024 系统综述与关键研究`、`2025 最新指南/进展` 等阶段节点。
- 反模式：用 `flowchart LR` 画成“2021 基础证据 -> 2022 核心综述 -> 2023 负担数据 -> 2024-2025 指南更新”并把文献节点散在右侧或斜向连线。这会变成流程/关系图，不是里程碑时间线。`flowchart LR` 只作为 Mermaid `timeline` 不可用时的兜底，而且必须保持横向年份/阶段轴和每个时间节点下方的纵向文献卡片列。

时间线节点必须能在 CSV 中反查到来源，或明确标注为教材/背景节点。

推荐视觉结构：

- 上方是标题，如 `{{主题}}关键研究里程碑（{{起止年份}}）`。
- 中间是横向时间轴，年份或时间段节点横向排列；若使用原生形状或 PPT，年份/阶段节点做成彩色短条，横轴贯穿所有节点。
- 每个年份/阶段下方是纵向研究卡片列；卡片第一行写 `[evidence_id] 研究简称`，第二行写核心贡献。年份多时同一时间段下要放多张代表研究卡片，而不是拆成很多单文献年份。
- 图下必须有图例表，列出 evidence_id、研究简称、完整标题、链接、年份/时间段、来源/期刊和意义。

Mermaid 白板模板：

```xml
<whiteboard type="mermaid">
timeline
    title {{主题}}关键研究里程碑（{{起止年份}}）
    {{2018 或阶段 1}} : {{[G1] 指南简称：奠定标准}}
                      : {{[SR1] 综述简称：汇总证据}}
    {{2019-2021 或阶段 2}} : {{[R1] 研究简称：关键发现}}
                         : {{[R2] 研究简称：扩展人群}}
    {{2022-2023 或阶段 3}} : {{[SR2] 综述简称：整合证据}}
                         : {{[R3] 研究简称：提出争议}}
    {{2024-2026 或阶段 4}} : {{[G2] 指南简称：更新推荐}}
                         : {{[N1] 最新研究：提示新方向}}
</whiteboard>
```

## 关键研究逐篇解读

关键研究逐篇解读回答：“哪些研究最值得读，它们各自贡献了什么，应该如何引用？”默认用正文小节，不再把所有研究压成一张大表，也不额外做白板图。

推荐小节结构：

- 小节标题：`[R1] 研究简称或完整题名`，标题必须带可点击链接。
- 类型与来源：写清系统综述、Meta、RCT、队列、诊断研究、机制研究、指南或共识，以及年份和期刊/机构。
- 研究设计与对象：用 1-2 句说明人群、样本、干预/暴露、对照和主要结局。
- 主要结论：解释这篇文献对用户问题的实际贡献。
- 局限与适用边界：说明人群、方法、随访、终点或证据等级限制。
- 引用用途：说明适合放在背景、指南依据、机制解释、结果比较、争议或研究空白哪个位置。

若用户明确要求横向比较，或论文写作需要快速比较多个研究，可在逐篇解读后补一个简短比较表；表中的研究标题仍必须有链接。

## 热点主题表

热点分析默认用逐主题段落解读，而不是复杂白板图或大表。只有用户明确要求图形化热点，且主题数量少、可稳定显示时，才补充主题簇图；只有需要横向比较主题时才补充热点主题表。

若没有完整文献计量数据库，只能写：

`本热点归纳基于本次检索纳入文献的标题、摘要和关键词归纳，不等同于完整 bibliometric 共现分析。`

主题簇建议维度：

- 机制/病理生理。
- 诊断/预测模型/生物标志物。
- 药物/干预/手术或管理策略。
- 特殊人群。
- 真实世界与长期结局。
- 安全性与不良事件。
- 指南实施与实践差距。
- 数字医疗、AI 或新技术。

必要时使用以下字段做简表：

| 主题簇 | 代表证据 | 近年变化 | 证据强度 | 研究空白 |
|---|---|---|---|---|

每个主题簇至少给出 1-3 条代表文献或说明“证据不足”。

## 指南差异解读或可选对比表

指南查新或学习材料默认先逐份解读指南/共识；只有确实需要横向比较多份指南差异时，才补充以下对比表：

| 维度 | 填写内容 |
|---|---|
| 指南/机构/年份 | 必须带链接 |
| 适用人群 | 年龄、疾病阶段、特殊人群 |
| 核心推荐 | 推荐条文的准确概括 |
| 推荐强度/证据等级 | 原文报告则填写；没有写 `未报告` |
| 与其他指南差异 | 推荐方向、阈值、药物/检查顺序、证据解释差异 |
| 更新点 | 与旧版或旧实践相比的新变化 |
| 实践影响 | 对学习、课题、论文或临床实践的意义 |

不要把不同年份或不同疾病范围的指南混为同一推荐；不要把这张表作为“指南与专家共识”章节的默认结构。

## 争议矩阵

当证据不一致时，用争议矩阵替代单一结论：

| 争议点 | 支持证据 | 反对/限制证据 | 争议原因 | 当前判断 | 后续研究方向 |
|---|---|---|---|---|---|

争议原因通常来自：人群不同、干预剂量/时机不同、结局不同、随访时间不同、研究质量不同、指南更新时间不同、真实世界可及性不同。

## 研究空白表

课题和论文写作支持必须识别空白，但不要夸大。

常见空白：

- 人群：儿童、老年、妊娠、合并症、不同种族/地区、本土数据不足。
- 干预：剂量、疗程、联合策略、序贯策略、真实世界实施。
- 结局：长期结局、生活质量、成本效果、安全性、患者报告结局。
- 方法：缺少 RCT、样本小、回顾性多、混杂控制不足、诊断标准不一致。
- 场景：基层、急诊、围手术期、ICU、慢病管理、科室协作。

输出研究空白时优先用段落或短列表解释，同时给出可行研究问题；若空白类型较多或需要横向比较，再补充简表。不要只写“需要更多研究”。

可选简表字段：

| 空白类型 | 当前证据不足之处 | 代表证据 | 可行研究问题 | 方法建议 |
|---|---|---|---|---|

## 论文引用逻辑解读或可选对照表

为论文添加文献时，优先用正文说明“哪个论点应引用哪类文献、各自引用角色和风险”；只有用户需要写作引用安排、段落级对应或多文献横向比较时，才补充以下对照表。不默认做额外白板图：

| 论文段落 | 需要支撑的论点 | 推荐引用 | 引用角色 | 风险 |
|---|---|---|---|---|
| 引言背景 | 疾病负担/临床重要性 | G1/T1/SR1 | 背景 | 数据年份可能较旧 |
| 研究空白 | 现有研究不足 | SR2/R3 | 空白 | 需避免夸大 |
| 讨论比较 | 与既往研究一致/不一致 | R1/R2/N1 | 比较 | 人群或终点不同 |

草拟段落时每一句重要事实后都要能定位到引用。不要把一个引用放在段落末尾覆盖多个无关事实。

## 学习资料组织

学习资料应体现“从框架到实践”的层次，默认用小标题、短段落或短列表组织；只有信息密度较高或用户需要课件/讲义结构时，才补充简表：

- 知识框架：疾病定义、分型、诊断、治疗、随访、安全。
- 指南路径：推荐流程、检查/治疗顺序、特殊人群。
- 经典研究阅读路径：先指南，再综述，再里程碑研究，再最新前沿。
- 临床实践提示：常见问题、证据依据、实践提示、风险提醒。

学习资料可以使用教材/专著，但结论仍应与最新指南和研究校对。

## Mermaid 和飞书白板规则

飞书白板图只用于默认固定图组或用户明确要求的标准图。生成前先确认图型、目的、节点和边；不要把任意概要表强行转成 Mermaid。

若使用 `<whiteboard type="mermaid">`：

- 白板块内只能放纯 Mermaid 文本。
- 不要放 Markdown 代码围栏、HTML/XML 标签、`<br/>`、`<span>`、`<b>`。
- 不要放 HTML 上标标签；上标包裹的引用编号必须改成普通文本 `[12]`、`[G1]` 或可点击文献名。
- 不要使用 `light-yellow`、`light-green` 等飞书 callout 色名；要么省略样式，要么使用合法十六进制颜色。
- 节点标签保持短句；复杂信息放图例表。
- 图中每条边必须有标签。
- 每个节点必须有语义角色，例如指南、系统综述、关键研究、最新研究、机制、结论、研究空白；不要只用视觉样式区分节点。
- 关系图必须表达作用关系，如 `推荐背景`、`汇总证据`、`直接支持`、`提出冲突`、`提示空白`；不要只把标题用箭头串起来。
- 不承诺飞书白板连线一定具备对象绑定；若需要严格可编辑关系图，应按当前环境支持的标准飞书 diagram 能力验证后再说明。

安全示例：

```xml
<whiteboard type="mermaid">
flowchart LR
    Topic[主题问题] --> G1[G1 指南]
    Topic --> SR1[SR1 系统综述]
    Topic --> R1[R1 关键研究]
    G1 -->|推荐方向| C[倾向支持]
    SR1 -->|总体证据| C
    R1 -->|关键结局| C
    C -->|仍需研究| Gap[研究空白]
</whiteboard>
```

生成报告 XML 后，若包含 Mermaid 白板，运行 `scripts/validate-report-whiteboards.js <report.xml>`；失败时先修复或改为表格/形状图。

## PPT 图表规则

通过 `lark-ppt` Skill 创建 PPT/幻灯片时，关键图表优先用原生 `shape`、`line`、`table` 或原生图表组合绘制。默认只做时间线页和逻辑关系页；系统综述、关键研究、最新进展、热点、争议和空白优先做逐篇/逐主题解读页，只有横向比较确有必要时才做表格页。Mermaid whiteboard 只能作为增强，不能作为唯一主图，因为回读 XML 不一定能证明 whiteboard 内容实际显示。

每页只讲一个主信息。逻辑图、时间线、逐篇解读页和必要表格必须显示 evidence_id 或文献简称，并在同页脚注或末页参考文献提供链接。

## 交付前检查

- 图表是否回答明确问题，而不是装饰。
- 文档类产物是否包含 `论文关系梳理图 + 关键里程碑时间线`，并插入正文证据章节；系统综述、关键研究、最新进展、热点、争议和空白是否已在后续章节逐篇或逐主题解读。
- 图中证据节点是否来自 CSV 或参考文献表。
- 图例是否提供链接、来源/期刊、年份、来源级别和阅读状态。
- 每条边是否有关系标签。
- 是否区分高质量证据、辅助证据、争议证据和研究空白。
- 热点主题解读是否说明数据来源和局限；若使用热点主题表，是否避免伪装成完整文献计量分析。
- 论文引用逻辑是否把论点和引用一一对应；若使用对照表，表格是否只作为补充。
- 是否避免把概要表、热点、争议、空白、学习路径强行画成非标准复杂图。
- 图表是否过载；过载则拆成两张或改为表格。
