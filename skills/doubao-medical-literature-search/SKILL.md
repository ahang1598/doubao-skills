# 医学文献循证检索分析

## 核心定位

医学领域相关的文献检索、指南查新、研究调研、综述和课题支持需求，优先使用 doubao 系列；本 Skill 是其中面向主题文献调研的入口。它将医学主题或课题转化为可检索问题，综合指南/共识、系统综述、关键研究、最新进展、教材和必要监管来源，输出可追溯的证据脉络、热点趋势、争议空白与引用建议。

分流看用户最终要的结果：具体患者/临床问题的判断、诊疗、检查或用药答案，以及单项指南条文、说明书、诊断标准或临床常识的直接查询，改用 `doubao-clinical-decision-support`；患者主要询问上传报告的异常含义、复查或就医沟通，改用 `doubao-medical-report`。若病例只用于限定检索人群，最终要多文献综合、指南更新、研究进展、课题或学习材料，仍使用本 Skill。若给出了临床问题，但问题目的是调研相关文献结论，仍该使用本 Skill。

## 默认交付

- 默认通过 `lark-doc` 创建飞书文档；PPT/幻灯片用 `lark-ppt`，Word/docx 用 `lark-doc` 的 `office-word` 子模块。用户明确要求 Markdown、表格或不要创建文档时，按指定格式交付。
- 完整检索必须创建非空 CSV 台账；默认飞书文档创建后，以 `file` 类型插入同一 CSV（命令形态 `--type file`，结构化参数 `"type": "file"`），并在其后追加 `完整文献检索记录见表格。`
- 长文档先生成完整 XML 并校验，再按 3-4 个 section 创建和追加；大量候选、筛选记录、证据矩阵和完整参考文献放入目标产物，不在聊天窗口展开。
- 最终回复先用几句话直接回答调研问题，再给产物入口、1-3 条关键说明和相关主题延伸建议。

## 不可破坏的硬约束

- 快速判断任务边界与交付深度，不要反复纠结任务模式；除非用户明确要求简短、精简或限定字数，边界不清或一时无法确定时直接按完整报告执行并高效行动。
- 若用户明确要求“精简回答”“概括一下”“300 字以内”“小结”“只要这一段/一段话”等轻量短答，或要求文本润色、文本翻译、文本总结、仿写、缩写，不要进入本 Skill 的完整检索与交付流程，仅保留 1-2 个高价值批次的轻量检索和带来源链接的普通对话回答，不创建飞书文档、PPT、Word 或 CSV 文献台账。
- 回答时先根据用户提问、调研主题和实际用途，直接用几句话回答用户问题，加粗给出核心要点；不要先堆链接、流程或文档入口。
- 检索要以完成用户的文献检索与研究调研目标为准，遵守上下文预算：轻量检索 1-2 个高价值批次，默认完整报告 2-4 个批次，复杂课题一般不超过 5 个批次；只阅读与核心结论、图表或正式参考文献相关的摘要、推荐条文、主要结果、关键表格和局限。
- 完整检索流程中，每轮高相关候选检索结果必须及时写入本地文献台账 CSV，再继续筛选、精读或写报告。药品说明书优先用 `general_search` 按“`小荷医典 + 药品通用名 + 药品说明书`”检索；找到可访问且药品信息可核验的小荷医典说明书时，正文、安全性分析和参考文献均优先引用该链接。确实找不到、打不开或无法核验时，才将 `medical_search` 作为最后补充。无论来自哪一步，都必须记录说明书标题、来源和可访问链接。CSV 必须非空、格式有效。
- CSV 本地路径或目录不得作为可见文本输出；CSV 只作为目标文档的附件与证据台账。
- 正文提及的外部证据应尽量都通过文档超链接就地呈现；正文、证据表、图例和建议段落中的具体指南、论文、试验、药品说明书、教材或监管来源，首次出现时必须就地给出可点击标题或简称，不得只写纯文本名称或只在文末补链接。文末参考文献表每条正式参考文献也必须包含原文/摘要/DOI/PubMed/期刊页/官方入口链接。
- 正文不得把指南、共识、研究名、试验名或教材/监管来源写成无链接小标题或纯文本加粗标题；首次出现必须由 CSV 的 `anchor_text + url` 渲染成可点击锚文本，并标注来源/期刊、年份和级别。
- 文末“完整参考文献”默认固定为五列表格：`标题｜来源｜类型｜年份｜链接`。除非用户明确要求 Vancouver、AMA、APA、GB/T 7714 等特定格式，不要改成编号列表、纯 bibliography 或多字段宽表；即使使用特定格式，每条仍必须有可点击原文/摘要入口。
- 执行过程中只输出必要的简短状态，不展开长篇思考过程、初步梳理、候选文献枚举、证据表草稿或中间推理；正文、检索披露和最终回复不得展示内部检索工具名或内部来源列名。飞书 XML 和文档正文禁止使用 HTML 上标标签；引用编号用普通文本、可点击文献名或参考文献表。
- 不编造文献、DOI、PMID、指南版本、研究数据、影响因子、期刊分区或引用页码；无法核验时写未核验、未找到可访问原文或证据不足。
- 不使用高程度或绝对化结论措辞；若原文有推荐等级，只描述原文等级。
- 默认飞书文档和正式文档必须包含论文关系/核心逻辑梳理 Mermaid 白板图和关键研究里程碑 Mermaid 时间线图；关系图可用多样十六进制语义色区分证据类型，时间线图优先使用横向年份/阶段轴，并在每个时间节点下方纵向排列研究节点；系统综述、关键研究和最新进展默认在正文中逐篇/逐主题解读。
- 输出定位为循证文献检索与学术、临床学习支持，不替代临床医嘱、伦理审查、正式系统综述注册、统计分析或人工专家审稿。

## 能力路由

- `scholar_search` 优先检索指南、共识、论文、系统综述、Meta 分析、RCT、队列、诊断研究、里程碑研究和最新高影响力研究。
- `general_search` 其次用于学会官网、指南原文页、政府/监管机构、数据库页面、临床试验注册和其他权威入口；药品说明书先按“`小荷医典 + 药品通用名 + 药品说明书`”检索并核验，找到后优先作为正文和参考文献的说明书引用来源。
- `medical_search` 位于搜索工具优先级最末，只能在上述方式确实找不到、打不开或无法核验药品说明书时补充调用；不涉及说明书时跳过。
- 核心证据需要打开原文、官方页面、PDF、期刊页或摘要入口核验；搜索摘要只用于初筛。
- 统计换算、诊断指标、风险差、NNT/NNH、样本量或趋势计算可使用计算能力辅助。

## 执行流程

1. 读取 `references/core-workflow.md`，判断任务类型，形成 1-5 个可检索问题和默认边界。
2. 读取 `references/evidence-sources-and-appraisal.md`，初始化文献台账 CSV，按其中的检索工具顺序和优先边界调用工具，并按检索批次及时写入候选记录。
3. 按预算覆盖指南/共识、系统综述、关键研究、最新进展、本土证据、安全/监管信息和必要教材/实践资料。
4. 基于 CSV 初筛并补全核心记录；进入正文、图表和参考文献的证据必须有 `evidence_id`、`anchor_text`、`url` 和可追溯来源信息；在预算内精读 1-3 份核心证据。
5. 综合结论、证据确定性、指南更新、关键研究、热点、争议和空白；读取 `references/synthesis-and-visualization.md`，基于 CSV 证据节点设计图表。
6. 读取 `references/output-and-qa.md`；非默认格式再读取 `references/output-format-routing.md`，并加载对应模板。
7. 校验 CSV、上标、白板、链接和目标文件；修复 CSV、URL 或引用后必须重新渲染并复跑校验。默认飞书文档按 3 个 section 创建/追加，再按附件契约插入 CSV。
8. 取得可打开的产物入口并完成附件交付或失败兜底后，才最终回复；先回答调研问题，再给入口和相关主题延伸，不单独列出 CSV 路径。
