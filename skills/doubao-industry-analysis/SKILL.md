# 行业研究

本 Skill 用于交付一份行业中长期深度研究报告。执行者应以深度行业研究员的方式工作：先判断这个行业未来中长期最关键的变化是什么，核心矛盾如何演化，价值会在产业链哪些环节重新分配，哪些变量足以验证或推翻主线判断；再围绕这些问题组织资料、拆解产业链、分析竞争格局、评估商业化进程与风险，最终形成可供决策者理解的深度报告。正式展开报告前，先形成一份几百字的判断主线底稿。后续报告写作应围绕这份判断主线底稿展开，按固定五大板块组织内容，把行业写透，并最终渲染成一份飞书文档交付给读者。完整公司深度报告链路长、过程状态多，主流程应委派给 OrganizerAgent 承担统筹OrganizerAgent 可按需分配给subagent执行。

## 质量标准（Quality Bar，最高优先级）

这是本 Skill 唯一的最高标准。章节顺序、表格选择、飞书样式、写作规则都为它服务；**任何排版或格式规则若让报告变浅、变得不可溯源、丢掉判断或更像填空模板，一律以质量标准为准、把格式简化掉。**

一份合格的行业深度报告必须做到五件事：

1. **数据真实可溯源（底线）**。进入正文或表格的每个数字都要能追溯到清晰来源，每个关键判断都要说清依据来自一手披露、具名机构、行业数据还是本研究测算。**绝不用印象、常识或模糊出处补数字。** 报告的深度不来自篇幅或形容词，而来自把每个数字追溯到多深的因果链、对自己判断的置信度有多诚实。这一条优先于排版、风格与篇幅。
2. **判断贯穿一条主线**。报告要有一个**可验证、可推翻**的核心判断，五大板块各自认领一个子命题去印证或限定它，而不是"什么都讲一点的资料汇编"。读者读完应清楚：这个行业的钱赚在哪个环节、格局会怎么变、哪些信号能验证或推翻你的判断。
3. **信息详实、有量化**。讲清行业真实的经营图景：定义与边界、规模与增速、产业链与利润池、竞争格局与集中度、政策与驱动、盈利质量与现金流、关键风险与落地路径，每处尽量给具体数字、做一步量化测算或对比，再点出"这对决策意味着什么"。
4. **事实支撑判断，而非替代判断**。分析性正文应围绕判断展开，而不是仅仅罗列信息项。读者应看到判断如何由事实推出，而不是仅仅看到"核心证据 / 触发机制 / 受益方 / 时间表"式事实清单。列表只用于风险提示、参考文献、检查清单、T 信号或真正并列且需要扫描的信息。
5. **克制专业**。语气像懂行的研究员在解释：直接、克制、重因果、少口号。区分确定事实、机构估算、媒体观点与作者推断，措辞与证据档位相称。

## 核心思路（Core Idea）

报告应当读起来像一位懂行的研究员做完功课后**自然写出来的成品**，而不是被一张清单填满。

- **先有判断、再写正文**：检索取证结束、动笔之前，先落一张判断主线底稿（七格表 + 三情景 + 各板块子命题）。证据推翻判断时，先改这张表，再动正文。
- **五大板块写透**：固定结构——行业定义与规模、行业结构与竞争格局、核心驱动与制约、趋势研判与结构性机会、商业化路径与盈利质量；外加执行摘要、风险提示、结语、参考文献。每个板块开头用一句"亮明判断"的主题句，结尾用一个 `回扣主线` 小节收回到核心判断。
- **三级数据分级守住真实性**：每条数据先归入一/二/三级可信度，据此决定能否当展示数值、措辞要多克制。一级可直接陈述、二级须显名归因、三级仅作线索不入展示。
- **图全部改为表**：本版**不出折线图、柱状图、饼图等任何配图**，所有原本用图承载的并列信息（规模趋势、产业链全景、各环节毛利率、市场份额、三情景对比等）一律用**结构化表格**呈现在报告正文中。
- **正文不暴露内部机制**：不写"本板块要证明的子命题"，不出现 lane/tier/门禁/子命题标签等工程术语；不提及本 Skill 名称、检索轮次或内部流程。

## 何时使用（When To Use）

仅当用户想要一份**针对某一行业或赛道的中长期完整深度报告**时使用：行业定义与规模、产业链与竞争格局、政策与驱动、景气周期、趋势研判、盈利质量与落地。

用户可能强调一个侧重点（如规模测算、竞争格局、政策影响、估值、出海、某条技术路线）。把它当作完整报告内部的一个**焦点**，而不是只产出一个窄答案的理由。

**不要用于**（转对应场景）：

- 问"该行业今天涨跌/热度/资金流/龙头股" → 走板块热度分析。
- 问某个**具体公司**怎么样、它的竞对、估值贵不贵 → 走公司分析/个股日报。
- 问某个**宏观事件**对市场的影响 → 走市场信息/事件解读。
- 问行业内某公司的**财报/业绩** → 走财报与业绩分析。
- 一句话能答的窄问题、普通行业科普、翻译 → 不走本 Skill。

目标对象的判定细则见 `references/task-router.md`。

## 工作流程（Workflow）

### 1. 定锚与策略

用系统时间解算**基准年**（执行当年，如 2026）与**预测窗口**（当前年 +5）。承认训练数据可能过期：规模/政策/融资/技术以联网检索结果为准，与记忆冲突时以可溯源的检索结果为准。若环境有更专业的数据 Skill（财经数据、工商数据、专利等），对应维度优先调用。先用 `task-router.md` 确认任务、记录用户焦点、判定行业结构类型。

### 2. 检索取证与三级分级

先读 `data-collection-protocol.md` 与 `data-grading-and-citation.md`。取数只走两条通道：

- **`general_search`**：公开事实、政策原文、催化、需求、行业事件、可点击的原文 URL。
- **`seed_finance_search`**：市场规模/增速/份额、价格、毛利率、产能利用率、融资、估值、财务等市场与财务数字（金融数据库/官方统计/一手披露口径）。

以**覆盖度**而非轮数停机，围绕每个维度成组铺开检索、同步推进、不逐条串行干等：核心维度（规模、增速、格局、价格、盈利、估值、政策、需求）各 ≥2 条证据，政策梳理 ≥5 行，建议累计 ≥18 条。边查边把关键数字、关键判断**记在对话上下文/工作笔记**、登记清楚它的**取数通道、来源级别、正文角色、数据基期、源值（原文数字＋单位＋口径，照抄准）、临时来源标识**（即**来源台账**，用于记录与核对，不落本地盘）；最终引用编号不要按检索顺序预分配，而是在正文成稿时按**读者从上到下首次遇到该来源的顺序**分配，同一来源全文复用同一编号。估算、产能、份额等易被拔高的同步想好安全措辞与必须保留的限定语；两条通道都拿不到的，作为**数据缺口**如实记录。**缺数据本身是一个结论**——正文标"公开信息有限"，绝不切换隐藏来源、绝不补造。

### 3. 定判断主线（写正文前必做）

照 `insight-spine.md` 的模板把判断主线**写在对话上下文里**（不落本地盘），含：一句话主线（可验证可推翻）、七格表（决策问题/市场共识/非共识判断/核心数据 3–5 个/因果链/利润池迁移/验证证伪信号 T1…≥5 个）、三情景速览（**先想好结果**）、各板块子命题。**先有判断再写正文**：证据推翻判断时先改这张表再动正文。

### 4. 分批写入 Markdown 分片

打开 `report-finalization.md`（批次表、各板块展开重点、回扣主线写法）、`analysis-framework.md`（分析镜头）与 `lark-doc-report-standard.md`（Markdown 规范、拼接与交付门禁），判断主线当尺子。核心机制：**正文按批次表逐批写入独立 Markdown 分片**，默认落在 `parts/` 目录，由脚本按固定顺序拼接成整篇 `output/report.md`。推荐文件名：批1 `parts/01-executive-summary.md`，批2 `parts/02-industry-definition-scale.md`，批3 `parts/03-structure-competition.md`，批4 `parts/04-drivers-constraints.md`，批5 `parts/05-trends-opportunities.md`，批6 `parts/06-commercialization-risk-refs.md`。每批只写当前板块，写完立即做**单批校验**：论证完整、信息充分、与主线一致、标题层级正确、必需表格和 `回扣主线` 小节齐备。批3 写完后额外做一次**阶段校验**，检查前 3 片结构、顺序与引用连续性，但不上传飞书。数字引用**按正文首次出现顺序递增编号、同一来源全文复用、句末标注**；如果后续改稿把某个来源提前或延后，要同步调整编号和文末参考文献顺序，避免出现 `[1][4][2][3]` 这类阅读顺序错乱；**每写一个数字即对照台账源值核对准确性**（句末规则与核对六维见 `data-grading-and-citation.md` 第五、六节），与主线核心数据前后一致、措辞与来源档位相称。

### 5. 交付前自检

对照 `report-finalization.md` 末尾的"交付前自检清单"逐条核对。最关键四项：判断主线已贯穿且无标签痕迹；各批内容完整且核心数据前后一致；每个市场/财务数字取数通道与出处级别达标、估算带相称限定语、缺口如实标注；**正文每个数字已句末标引用、并对照台账源值完成准确性核验**（见 `data-grading-and-citation.md` 第五、六节）。再对照 `lark-doc-report-standard.md` 的"交付门禁"。

### 6. 收尾与交付

六批写完后，先运行拼接脚本（建议 `python3 scripts/assemble_report.py`）按固定顺序把 6 个 Markdown 分片合并为 `output/report.md`（拼接后脚本会自动按正文首次出现顺序校正引用编号与文末参考文献作为兜底；编号已正确时为无操作，改稿后也可单独运行 `python3 scripts/renumber_references.py --file output/report.md`），再运行整篇校验（建议 `python3 scripts/validate_report.py --stage final --file output/report.md`）。最终核验聚焦四件事：执行摘要核心判断与各章 `回扣主线` 完整保留、图已全改表、正文数字引用置于句末且**首次出现顺序为 `[1]`、`[2]`、`[3]` 递增**、文末参考文献按该最终编号一一勾稽、对照源值完成最终数据准确性核验。整篇校验通过后，将 `output/report.md` 导入为飞书在线文档；上传后做一次交付抽查，确认标题层级、引用块、表格、引用与可点击链接渲染正常。核验通过后返回飞书文档链接（**最终交付物**）。

## 硬约束（Hard Constraints）

1. **无来源数字不入正文、数字必核对**：每个进正文/表格的数字都要在取证阶段有据可查、能在文末参考文献或来源说明中追溯；正文数字以**句末**数字引用 `[n]` 标注、写入前对照台账源值核对准确性（见 `data-grading-and-citation.md` 第五、六节）。
2. **市场数字两维红线**：展示值的取数通道须 `seed_finance_search`（或用户文件/显名测算），出处须一/二级；新闻里的同类数字只作线索；三级不得单独撑数。
3. **取数只有两条通道**（`general_search` + `seed_finance_search`），不得引入第三条隐藏数据源。
4. **结论强度不超出证据档位**：估算写"据券商估算"，媒体观点写"据报道"，推断写"这可能意味着……后续需验证"，不得写成披露事实（措辞表见 `data-grading-and-citation.md`）。
5. **缺数据如实暴露、绝不编造**：查不到写"公开信息有限"或给测算假设（注明），不硬凑 CR_n/投后估值/渗透率等。
6. **时效红线**：趋势预测须以当年为时效基准；正文标注数据基期；年份用真实数字。
7. **结构红线**：9 个板块齐备（核心观点/一至五/风险提示/结语/参考文献）；五大板块每章末尾有 `回扣主线` 小节；免责声明不删改。
8. **主线红线**：写正文前先想清楚判断主线；执行摘要交代共识/非共识、各板块认领子命题并回扣主线、板块四三情景给结果。
9. **数据合规红线**：仅用授权检索工具与用户文件、只取合法公开内容；严禁绕过登录态/付费墙/反爬、严禁抓取未授权数据。
10. **配图红线**：本版不出任何配图（折线/柱状/饼图/产业链结构图等），所有并列信息一律用表格在报告中呈现。
11. **交付红线**：正文分 6 批写入 `parts/` 下的独立 Markdown 分片，由脚本按固定顺序拼接为 `output/report.md`；每批保留单批校验，批3 后做阶段校验，拼接后做整篇校验；飞书文档由完整 Markdown 导入生成并作为最终交付物。
12. **不暴露内部机制**：正文不出现 lane/tier/门禁/子命题标签等工程术语，不提及本 Skill 名称、检索轮次或内部流程。

## 交付标准（Output Standard）

一份合格的行业深度报告应具备三个特征：

- **信息充分**：真实的规模、产业链、利润池、竞争格局、政策、盈利质量与风险事实，每处尽量带具体数字与一步测算。
- **判断清晰**：一条贯穿全文的核心判断、各板块认领的子命题、明确的"什么能验证 / 什么能推翻"，而非"行业空间大、长期看好"这种正确的废话。
- **证据可溯、数据可核**：正文每个数据点有句末数字引用、与台账源值核对一致、文末参考文献可点击且与正文勾稽、估算与二三级来源带相称限定语、数据缺口如实标注。

报告不提及本 Skill 名称、内部检索机制、检索轮次或模板规则。
