# 取数协议（Data Collection Protocol）

本文件规定本 Skill 对外取数的唯一方式，配合工作流程第 2 步使用。取数得到的数字与判断**记在对话上下文/工作笔记**、登记清楚来源、档位与**源值（原文数字＋单位＋口径，照抄准）**（不落本地盘——本地文件跨回合会丢；分级定义见 `data-grading-and-citation.md`）；正文可溯源由**句末数字引用**与文末参考文献承载（源值供交付前核对准确性，见 `data-grading-and-citation.md` 第六节）。目标是避免"一遍搜完就写"，同时保住模型在套用固定结构之前先发现行业特定事实、矛盾与线索的能力。

## 一、数据合规与授权（红线，先读）

- **只用授权可用的通道**：对外取数只通过 `general_search`、`seed_finance_search` 两条工具通道，以及用户主动提供的文件；不得自行调用其它未授权的数据接口或数据源。
- **只取合法公开的内容**：仅使用合法、公开可访问的页面或官方/授权接口返回的信息；引用前确认来源公开可得。
- **严禁绕过任何访问控制**：不得绕过登录态、付费墙、验证码或反爬限制，不得抓取未授权数据，不得规避站点条款（robots/ToS）或访问受限的非公开数据。
- **缺数据如实暴露，不另寻隐藏来源**：两条通道（及用户文件）都拿不到的字段，按缺口降级、如实标注，绝不切换到隐藏的第三方数据源、绝不补造。

## 二、两条取数通道

对外取数只有两条通道（lane），不得引入任何第三条隐藏数据源——无论是在提示词、来源计划、正文还是参考文献里：

- **`general_search`**：公开新闻、政策原文、公司/交易所公告、产品发布、行业事件、需求数据，以及可点击的原文 URL。
  - **取什么**：公开事实、催化、政策/事件时间线、证实/证伪表态，以及参考文献用的可点击链接。
  - **不取什么**：规模/价格/份额/估值/财务等口径敏感的市场数字（这类只能作线索去核验，不得直接当展示值）。
- **`seed_finance_search`**：市场规模/增速/份额、价格、毛利率、产能利用率、资金/融资、估值（PE/PB）、财务指标等市场与财务数字（金融数据库/官方统计/一手披露口径）。
  - **取什么**：报告核心数字与判断主线"核心数据"的唯一取值源。
  - **数据基期**：涉及时点的数字写明基期（YYYY 或 YYYYFY/YYYYQn，行情类写"数据截至 YYYY-MM-DD"）。

事实可能来自公开网站或金融数据库，但工作流层面的通道名只有这两个（外加用户提供的文件）。**两条铁律**：①不得引入第三条隐藏数据源；②市场/财务数字一律以 `seed_finance_search` 为唯一取值源。

## 三、来源计划（动笔前先想清楚）

动笔检索前，先想清楚这篇报告每条通道各要拿什么。核心是把下面几件事定下来（其中主线与口径会进入判断主线底稿）：

- **问题类型**：行业长期趋势 / 景气位置 / 产业链拆解 / 竞争格局 / 政策环境 / 盈利与估值 / 完整深度报告。
- **要从 `general_search` 拿什么**：公开催化、政策与事件、需求事实、原文 URL、风险事件、证实/证伪表态。
- **要从 `seed_finance_search` 拿什么**：市场规模/增速/份额、价格、毛利率、产能利用率、估值、资金/融资、财务指标。
- **缺通道时的降级**：若某条通道缺失，明确写出缺了哪些字段，对相关板块降级或标数据缺口，绝不用另一条通道补造。

正常生产型深度报告必须两条通道齐备。只有 `general_search` 时，可先给催化/需求/政策解释，但规模/价格/份额/估值/财务等市场结论须标"数据不足"、相关板块降级；只有 `seed_finance_search` 时，可先讨论市场与财务数字，但催化、政策原文与可点击来源须标"数据不足"。

## 四、检索轮次与覆盖度

不要"一遍搜完就写"。每一轮围绕一个取证目的，把该目的下相关的几条检索一次性成组发出、同步推进，而不是查一条等一条；用足够宽的覆盖面把这个证据问题快速答清。

**分两遍主取数**：第一遍用 `general_search` 收集公开事实、催化、政策/新闻/事件时间线、需求侧证据与原文 URL；第二遍用 `seed_finance_search` 收集市场印证——规模/增速/份额、价格、毛利率、产能利用率、资金/融资、估值、财务指标。

**以覆盖度而非轮数停机**。动笔写正文前，下列核心维度应尽量覆盖到位（每个 ≥2 条证据）：

1. 行业定义、边界与规模（多源对照 + 一条本研究测算）。
2. 增速 / CAGR、宏观敏感性、生命周期阶段。
3. 产业链结构与各环节价值分布（利润池）。
4. 竞争格局：集中度（CR3/CR5）、头部玩家与阵营。
5. 价格、产能利用率、库存与供需（如适用）。
6. 政策梳理（≥5 行：发布时间/发文机关/文件名称文号/政策层级/核心内容/影响方向）。
7. 核心驱动力与制约因素（需求、技术、政策、成本、竞争五维）。
8. 盈利质量与估值（毛利率、ROE/ROIC、现金流、可比公司 PE/PB 区间与分位）。

建议累计 ≥18 条证据。覆盖到位、最重要的行业变量已清晰、剩余未知项已转为明确的"数据缺口"时，即可停机动笔。

**不要停**，如果：竞争格局还没讲清谁拿走利润；核心判断还依赖一个未经核验的数字；只有正面的、公司/聚合站口径的来源；政策类事实还没取到官方一手源。

## 五、采集规则

1. **公开来源的可点击链接**：只有当 URL 来自 `general_search`（或用户提供）时，才能在参考文献/来源行显示"打开原文"。

2. **市场/行业数字**：只有来自 `seed_finance_search`、用户文件、或显名"本研究测算"时，才能在正文/表格引用。

3. **来源分级与取数优先级**：同一事实优先引用级别更高的来源。三级可信度的完整定义见 `data-grading-and-citation.md`。概要：**一级**（可直接当事实）＝①金融数据库的市场/财务数据；②一手官方与权威机构原文（公司年报/季报/业绩快报/公告/招股书/交易所披露，统计局/部委/央行/海关的官方统计与政策原文，官方通讯社，券商研报原文）。**二级**（须显名归因、限定措辞）＝权威财经/行业媒体原创报道、行业协会与第三方研究机构数据（中汽协/光伏协会/IDC/Gartner/IEA/BNEF 等）、券商或机构对未披露口径的估算。**三级**（仅作线索）＝聚合站（前瞻/智研/中商/华经）、自媒体/UGC（雪球/股吧/公众号）、传闻与二手转载。政策类事实必须取自官方一手源；**三级不得作为任何展示数值或政策事实的唯一来源——必须顺藤摸到一级/二级原文核验后引用更高等级来源，核验不到就不展示**。

4. **行情/财务数字的来源红线**：价格、涨跌幅、成交额、资金流、估值（PE/PB）、历史区间、近一年涨幅，以及市场规模/份额/毛利率/产能利用率等核心数字——这类时效性强或口径敏感的数字，一律以 `seed_finance_search`/金融数据库/官方统计/一手财务披露为唯一取值来源；`general_search`（新闻）里出现的此类数字只能作线索去核验，禁止直接当展示值。正文/表格里每个具体数值都必须可追溯，严禁凭印象或估算给数，拿不到就按缺口降级。

5. **取数时点（数据基期）**：涉及行情/财务时点的数字写明基期——年报/统计年鉴等滞后数据标注其基期（YYYY 或 YYYYFY/YYYYQn），不因发布晚而失真；涉及行情时点的（如估值分位）统一写"数据截至 YYYY-MM-DD"；禁止用盘中实时价或来路不明的估算值充当口径数字。

6. **估值口径**：PE/PB/PS 须标注 TTM/静态/动态(LYR) 与时点。

7. **代表公司选取口径**：板块二"重点公司简析"的样本必须从该行业代表性上市公司中选取，优先权重靠前、有代表性的龙头与典型品种，每家写清其代表性（如"整机龙头""核心零部件标的"），且仅用法定信披来源（年报/公告/交易所披露）。

8. **同一指标多源差异 >30%**：并列展示并解释口径（出厂/零售、含税、销量/销售额、全球/中国、收入/出货/装机口径），不取平均、不只取单源。

## 六、缺通道时的降级

- **某条检索通道不可用**：按本文第二、三节降级——只有受影响板块已明确降级、数据缺口已如实标注时才交付；绝不用另一条通道顶替、绝不补造。
- **两条通道都不可用**：不强行产出含编造数字的报告；向用户说明数据缺口与限制。
- **缺数据本身是一个分析结论**，必须作为来源缺口如实暴露，绝不通过切换隐藏来源或补造来填补。

## 七、品牌、命名与示例数据纪律

- 本 Skill 名称含 `doubao` 前缀。该品牌词的对外使用须经内部合规/法务确认；正文中提及的任何第三方机构、平台或产品名称仅作中性、客观引用，不暗示授权、背书或合作关系。
- 模板、说明或示例中出现的任何示例链接、示例数字、占位机构名，仅用于演示格式，**不得作为真实来源或真实数据写入正式交付物**。
- 正式交付物（正文、表格来源行、参考文献）中的每个数据点都必须可追溯到两条来源通道（`general_search` / `seed_finance_search`）或用户提供的文件；拿不到则按缺口如实标注、绝不补造。
