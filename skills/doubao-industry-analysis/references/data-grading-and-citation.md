# 数据分级与引用纪律（Data Grading & Citation）

本文件是整篇报告的**数据真实性核心**，动笔取数前必读。它讲四件事：数据如何分三级、每个维度的合格来源长什么样、正文如何句末引用与勾稽、以及交付前如何逐个核对数字准确性。**这套分级优先于排版、风格与篇幅**；与别处冲突时以本文件为准。

## 目录

- 一、三级数据分级（最高优先级）
- 二、两维约束：取数通道 × 来源级别
- 三、每个维度的合格来源（举例）
- 四、正文角色与允许措辞
- 五、引用机制：正文数字引用 + 文末参考文献（句末标注）
- 六、数据准确性核验（写作中 + 交付前）
- 七、关键事实硬门槛
- 八、缺证据的处理
- 九、竞品证据

## 一、三级数据分级（最高优先级）

进入正文或表格的每一条数据，先归入下面三个可信度等级之一，并据此决定它能不能作为展示数值、措辞要多克制。另有一条正交的取数通道（lane）红线：市场与财务数字的*展示值*必须经 `seed_finance_search`（金融数据库口径）取得，或来自用户文件、或显名"本研究测算"；`general_search`（新闻）里的同类数字只能作线索。

### 第一级 · 可直接作为事实陈述（最高可信）

两类来源同属一级：

1. **金融数据库（`seed_finance_search`）的市场 / 财务数据**——市场规模/增速/份额、价格、毛利率、产能利用率、资金/融资、估值（PE/PB/PS）、财务指标等行业与公司量化数字（Wind / Choice / iFinD / 同花顺 / Bloomberg 的行业库与财务库）。**这是一切市场/财务展示数字的取数通道。**
2. **一手官方数据与权威机构数据**（经 `general_search` 触达原文）——按数据维度举例：
   - **政策 / 法律**：中国政府网、全国人大网，及发改委/工信部/能源局等部委的政策与法律原文；
   - **监管 / 交易所动作**：证监会、沪深北交易所、央行的公告、规则、数据；
   - **公司经营 / 披露**：公司年度报告、季度报告、业绩快报、公司公告（巨潮资讯/交易所披露）、招股说明书、投资者关系活动记录表；
   - **宏观 / 行业官方统计**：国家统计局、央行、海关总署的官方数据发布，以及行业主管部委的统计（如国家能源局月度装机、工信部产销数据）；
   - **官方通讯社**：新华社等官方媒体的原发稿；
   - **机构研报**：券商研究所研报原文（署名、带日期）。

一级数据可直接写成事实（"披露 / 为 / 显示"），是正文与表格所有展示数值的**唯一合格来源**。

### 第二级 · 可引用，但须标注来源、措辞需限定

- 权威/主流财经媒体与行业垂直媒体的**原创报道**（财新、第一财经、彭博、路透等署名原创，采编可信，但非一手发布）；
- **行业协会与第三方研究机构数据**（中国光伏行业协会、中国汽车工业协会、乘联会、IDC、Gartner、IEA、BNEF、Canalys、群智咨询、SNE Research、克而瑞等的统计、榜单与市占率/出货估算——是协会与第三方机构的统计或测算，非政府官方统计、非一手披露）；
- 券商或机构**对未披露口径的一致预期/估算**（如对市占率、产能利用率、行业 CR_n 的测算，是机构测算而非披露事实）。

二级数据引用时必须**显名归因并限定措辞**——"据中国光伏行业协会数据""据券商研报估算""据 IDC 测算"，用"据/约/或/可能"，**不得写成一手披露的确定事实**。**市场/财务数字即便出现在二级来源里，也只能作为线索去 `seed_finance_search` 核验后再用数据库的数据，绝不直接展示二级来源给出的数字。**

### 第三级 · 仅作线索，原则上不入展示

- 聚合平台/数据汇编站、自媒体、UGC、论坛、股吧（前瞻/智研/中商/华经，今日头条、雪球、东方财富股吧等）；
- 市场传闻、社交讨论、无署名来源、来源不明的二手转载。

三级信息**不得作为任何展示数值的依据**。它最多是一条待核验线索：必须顺藤摸到一级（或至少二级）原文核验后，引用那个更高等级的来源；**核验不到就不展示**——宁缺毋滥，按数据缺口处理、正文标"公开信息有限"，绝不补造。

## 二、两维约束：取数通道（lane）× 来源级别（tier）

判断一个市场/财务数字能不能进正文，要分清两个维度——很多失真正来自把"通道"和"出处级别"混为一谈，两者必须**同时满足**：

| 维度 | 要求 |
|---|---|
| **取数通道（lane）** | 市场/财务数字的*展示值*必须经 `seed_finance_search` 取得（或用户文件、或显名"本研究测算"）。`general_search`（新闻）里的此类数字只能当线索核验，绝不能直接当展示值。 |
| **来源级别（tier）** | 该数字的*底层出处*必须是一/二级可靠信源。第三级（聚合站/自媒体/二手转载）不得作为某个市场数字的唯一来源——确无更高级别来源时须交叉互证另一独立来源，否则按缺口降级。 |

判定示例：

- ✅ `277GW`，通道 `seed_finance_search`，出处国家能源局（官方统计，一级）——通道对、出处一手官方。
- ✅ 券商研报原文里转述的公司披露硬数字：出处券商研报（一级），正文可写成事实。
- ✅ `CR5 62%`（券商对未披露口径的估算，二级）：正文写"据券商估算"；行业协会数据（如组件出货量，二级）：正文写"据中国光伏行业协会数据"。
- ❌ `277GW`，通道 `general_search`（新闻里的数字直接当展示值）——必须改由 `seed_finance_search` 取同口径值，新闻仅作线索。
- ❌ `CR5 62%`，唯一来源是雪球（三级）——须换一手/数据库/券商研报原文/官方来源，或交叉互证，或按缺口降级。

## 三、每个维度的合格来源（举例）

不同维度有各自的权威出处。标「一级」的可直接展示；标「二级」的须显名归因、措辞限定，且市场/财务数字仍要回 `seed_finance_search` 核验后用数据库口径。取不到就降级并标注。

| 维度 | 首选来源举例（级别） |
|---|---|
| 市场规模 | 国家统计局与部委月度数据（如能源局装机、工信部产销）、Wind/Choice 行业库（**一级**）；行业协会统计公报（光伏协会、汽车工业协会等）须显名"据 XX 协会"（**二级**） |
| 增速 / CAGR | 由一级规模数据自行测算（标"本研究测算"并写公式），或金融数据库直接口径（**一级**） |
| 竞争格局 / 集中度 | 券商研报原始测算（**一级研报原文**）；第三方调研机构（消费电子看 IDC/Canalys，面板看群智/Omdia）须显名（**二级估算**） |
| 价格 | 价格数据库（有色金属看 SMM、化工看百川盈孚、光伏看 PVInfoLink/InfoLink）、交易所或现货市场报价（**一级数据库口径**） |
| 盈利 / 利润率 | 公司年报/季报、Wind/Choice 财务库（**一级**） |
| 产能 / 利用率 | 公司公告与年报（**一级**）；行业协会、券商产能跟踪须显名（**二级**） |
| 估值 | 金融数据库（Wind/Choice），须标 TTM/静态/动态(LYR) 与时点（数据截至 YYYY-MM-DD）（**一级**） |
| 政策 | 部委原文、中国政府网、发改委/能源局/工信部正式文件（**一级**） |
| 需求 / 装机 / 出货 | 国家能源局月度装机、海关进出口数据（**一级**）；行业协会出货统计须显名（**二级**） |
| 出海 / 海外 | 海关总署进出口、目的国官方统计（**一级**）；IEA/BNEF/IDC 海外报告须显名（**二级**） |
| 成本 | 公司年报成本拆分、价格数据库（原材料价格）（**一级**）；券商成本曲线测算（**二级估算**） |
| 融资 / 并购 | 公司公告（定增/可转债/IPO）、交易所披露、Wind 融资库（**一级**） |
| 重点公司 | 仅用法定信披（年报/公告/交易所披露），样本取该行业代表性龙头与典型品种（**一级**） |

> 注意：行业协会与第三方研究机构（中汽协、光伏协会、IDC、IEA、BNEF）的数据是**二级**，须显名归因；只有券商研报原文在"机构"中属一级。券商*自己估算*未披露口径时仍属二级。

## 四、正文角色与允许措辞

每条进正文的数据，按它在正文里扮演的角色决定允许的措辞。**结论强度不得超出证据档位**：

| 正文角色 | 正文允许写法 | 条件 |
|---|---|---|
| 硬事实 | 显示 / 为 / 披露 | 须一级来源（或本研究测算） |
| 官方/公司表态 | 公司称 / 官方表示 | 官方或公司的定性表述 |
| 券商估算 | 据券商估算 / 研报判断 | 券商对未披露口径的测算 |
| 机构/协会测算 | 据 XX 机构/协会测算 | 研究机构/协会的统计或测算 |
| 媒体观点 | 据报道 / 市场讨论 | 媒体观点或市场说法 |
| 本研究测算 | 按公开数据测算 | 本研究派生值，须写清公式/口径与假设 |
| 作者推断 | 这可能意味着……，后续需验证 | 作者推断，不得写成定论 |

措辞要点：

- 产能、规划、目标、市占率、行业 CR_n、管理层展望等**容易被拔高**的数据，正文必须带相称的限定语与必须保留的限制条件（如"据券商研报估算……，公司年报未披露该口径"、"规划产能不等同于已完全释放"）。
- 券商研报原文里*转述公司披露的硬数字*可写成事实；券商*自己估算*未披露口径的数字，必须写"据券商估算"。
- 同一指标多源差异 >30% → 并列展示并解释口径，不取平均、不只取单源。

## 五、引用机制：正文数字引用 + 文末参考文献（句末标注）

正文用**紧凑的数字引用**承载可溯源，不在段落或表格里写长串来源名或完整 URL：

- **来源台账与阅读顺序编号**：检索阶段把每个来源连同取数通道/级别/基期/**源值（原文数字＋单位＋口径，照抄准）**登记成一份**来源台账**（写在对话上下文、不落本地盘）；台账先只记录临时来源标识和源值，不按检索先后分配最终编号。**最终引用编号按正文成稿后的首次出现顺序分配**：读者从上到下第一次遇到的新来源为 `[1]`，第二个新来源为 `[2]`，以此递增；同一来源多处引用复用同一编号。文末参考文献按这个最终编号顺序誊抄；若改稿导致某个来源首次出现位置变化，必须同步重排正文编号与文末参考文献。台账里的源值是第六节"数据准确性核验"逐个核对正文数字的基准。
- **正文出现的每个具体数据点（规模/增速/份额/价格/毛利率/估值/融资/产能/需求/政策标准/竞品指标等量化或具名硬事实）都标紧凑数字引用** `[1]`、`[2]`、`[1][3]`，**位置统一为该数据所在句子的句末、句号之前**：一句只含一个数据点或多点同源 → 合并放句末（"2025年中国光伏新增装机277GW、同比增长28%[1]。"）；一句含多点**异源** → 每点后紧跟其引用以免对错（"组件均价0.68元/W[2]，硅料毛利率约35%[3]。"）。"凡数据必有句末标注"是底线，不能只标关键句、其余裸奔。两个例外：① `本研究测算`值句末写"（本研究测算，公式见上）"而非 `[n]`，其输入数字仍各带 `[n]`；② 表格走表下"资料来源"行（单元格内需要时可加 `[n]`），不逐格标注。
- 表格（替代原配图的并列信息）在表格正下方加一行 `资料来源：<具体机构/文件>`，命名到具体机构或文件名（如"国家能源局，本研究测算""锐科激光2025年年报，华经产业研究院"），不写"行业调研""公开资料"这类空泛写法；表格单元格内需要时也可用 `[n]`。
- 报告末尾给一个**编号参考文献**列表，把每个引用编号映射到：来源名、发布或访问日期、相关字段/章节（如有）、**真实可点击的 URL**、以及一句话说明它支撑哪个判断。
- **引用勾稽（强制）**：每个正文引用编号都要有对应的文末来源条目；每个文末来源条目都要在正文至少被引用一次。

参考文献的合规要求：

- 只列有公开原文链接的来源（来自 `general_search` 或用户提供）；无公开原文的数据库数字照常支撑判断，但若平台无法给出可点击链接，则作为"未公开链接的尽调输入"在来源说明里披露，不占编号引用。
- 参考文献里**不得**出现空条目、仅域名首页链接、失效链接、纯文字数据库引用（如只写"Wind""企查查"而无具体记录页），或描述与所引判断不符的来源。
- ≥15 条，**按正文首次引用顺序**使用 `[1]`、`[2]` 格式誊抄，文末编号必须与正文引用编号一致；不要使用 `1.` / `2.` 作为参考文献编号；同一机构不同报告分列。

## 六、数据准确性核验（写作中 + 交付前）

引用解决"数字出自哪"，本节解决"数字抄得对不对"。基准是台账里照原文抄准的源值：正文每个 `[n]` 数字都应等于台账第 n 条源值。**写作中**边写边核，**交付前**对 `output/report.md` 逐项复核六维；上传后只抽查飞书渲染，不把飞书端作为事实校验主场：

1. **值＝源**：等于源值（仅允许声明过的四舍五入）；防抄串位（277↔227）、错量级（亿↔万、GW↔MW）、用旧值。
2. **口径＝源**：单位/口径一致（出厂/零售、含税、销量/销售额、收入/出货/装机、全球/中国）；口径不同须按写明公式换算成 `本研究测算`，否则不写。
3. **基期＝源**：与源一致并在正文标注。
4. **通道×档位**：重核第二节两维红线。
5. **前后一致**：同一指标在摘要/正文/表格/`回扣主线` 取值一致；多源差异 >30% 按第四节并列、不取平均。
6. **测算可复算**：`本研究测算` 值能由所写公式与带 `[n]` 的输入复算得出。

不过关就改成源值、换算或降级/标缺口，**绝不靠改小数点、凑平均、模糊措辞糊过去**。缺口本身就是结论（见第八节）。

## 七、关键事实硬门槛

决策关键的事实需要具体、可核查的支撑。除非来源列表里有直接可用的文章/文件链接、官方记录 URL、数据库记录 URL 或用户提供的文件链接支撑该确切主张，否则**不要把它写成已确认的事实**。

该硬门槛适用于：

- 市场规模、增速、渗透率、政策标准、补贴、基准、排名，以及"最大/最高/第一"类主张。
- 融资金额、轮次、投资方、估值、并购、IPO 申报。
- 价格、毛利率、产能、利用率、良率、出货量、库存、交付时间表。
- 具名客户关系、产能预定、合作、诉讼、专利、安全事件。

若来源只是一个泛泛的首页、域名级 URL、无署名的媒体摘要、社交帖或未经核验的管理层/PR 表述，则在报告中**降级**该主张，用"公开信息有限""据报道""公司口径称""需以工商/融资数据库/官方核验"等措辞，不要把它放进执行摘要、核心判断或估值结论里当已确认事实。若该主张对理解行业仍重要，作为带标签的待核验项或数据缺口保留，而非删除。

## 八、缺证据的处理

若证据缺失，不要用泛泛之词填补。写清：

- 缺的是什么；
- 为什么重要；
- 现有最佳替代/代理指标是什么；
- 下一步该如何核验（哪个数据库、哪份文件、哪类访谈）；
- 什么答案会改变本报告的判断。

**缺数据本身是一个分析结论**，作为数据缺口如实暴露，绝不通过切换隐藏来源或补造来填补。

## 九、竞品证据

对具名竞品，尽量核验：公司状态（上市/未上市/开源项目/国资关联）、融资（最近轮次、金额、日期、领投、累计融资、可信估值）、经营成熟度（披露的收入/ARR、客户类型、地域、部署模式）、战略禀赋（渠道、产能/成本曲线、技术基准、合规本地化、客户关系、资本）。若公开来源无融资数据，标"公开来源未披露"而非猜测。
