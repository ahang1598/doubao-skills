# 报告定稿规则与写作大纲（Report Finalization）

本文件是写作阶段的唯一蓝本：固定章节、给出每个板块的方法论与展开重点、规定表格纪律与写作风格。**篇幅以论证完整、信息充分、与主线一致为准**，靠把每个小节写透、多做量化测算与对比来撑起专业深度，而不是堆空话注水。

最高优先级仍是 SKILL.md 的质量标准：数据真实可溯源、判断贯穿一条主线、信息详实、克制专业。任何本地格式或章节规则若与该标准冲突，保留质量标准、把格式简化掉。

> **⚠ 写法（配合 SKILL.md 第 4 步"Markdown 分片"）**：不要一次写完整篇——正文按 6 批分别写入 `parts/` 下的固定 Markdown 分片，批6 连同风险、结语、参考文献收尾。每批写完先做单批校验，批3 后做阶段校验，六批完成后用脚本拼接为 `output/report.md` 并做整篇校验。**不要手工复制粘贴合并稿，也不要改变分片顺序。**
>
> **⚠ 写作前必读判断主线底稿（第 3 步产出）**：本大纲解决"写什么板块、每板块该展开到什么程度"，而判断主线解决"这篇报告到底要证明什么判断"。两者配合——写每个板块时，**开头用一句"亮明判断"的主题句**，**结尾用 `回扣主线` 小节把结论收回到全文主线**，正文数字与主线核心数据保持一致。**不要在正文里出现"本板块要证明的子命题："这类标签式说明**——子命题是内部抓手，读者应通过论证"看出"你在证明它（正反例见 `insight-spine.md`）。

## 总则（先读这 8 条）

1. **结论先行**：每个板块开头先给判断，再用证据展开；禁止"本报告将探讨…"式绕弯开场。
2. **有数有源、数字落句末**：正文每个具体数据点都要有出处，用 `[1][2]` 标在**句末**；表格在表下加"资料来源"行；文末给可点击参考文献并完成引用勾稽。写入前对照台账源值核对准确性（句末规则见 `data-grading-and-citation.md` 第五节、核对六维见第六节）。
3. **不编造**：查不到的数字宁可写"公开信息有限，据……推算"，或给出测算假设；不硬凑。
4. **成段论述为主**：用有逻辑递进的段落写分析，列表只用于真正并列的要点（风险条目、参考文献、对比清单）。不要把连贯论证拆成一堆短 bullet。每个小节在表格或列表前都要有一句小结或判断句引出，不允许某小节只有列表/表格、没有论述。
5. **每个板块至少 1 张表**：本版不出任何配图，原本用图承载的并列信息（规模趋势、产业链全景、各环节毛利率、市场份额、三情景对比等）一律用**结构化表格**呈现。
6. **时效**：市场规模/政策/技术以联网检索的最新结果为准；正文标注数据基期与年份，年份用解算出的真实数字（如 2026），不留占位符。
7. **专业深度（写出扎实论证的关键）**：篇幅靠"论点—数据—含义"的密度自然展开，不靠注水。**每个分析观点都要用检索到的具体数据（带数字与来源）支撑，并做一步量化测算或拆解（量×价、CAGR、CR_n、ROE 杜邦、敏感性区间等）或同业/情景对比，再点"这对决策意味着什么"；论点与数据一一对应，无据的论断不写——内容显得单薄时通常是论证不够，应补数据与测算、不补形容词**（把事实转成判断的镜头见 `analysis-framework.md`）。多用对比表承载并列信息，避免泛泛而谈与重复前文。
8. **主线绑定——用论证体现，不要标签式表述**：
   - 每个板块**开头第一段**用一句**"亮明判断"的主题句**承接该板块的子命题——写成一个有态度、可被证明的论断，正文随后用数据与逻辑去证明它。**严禁出现"本板块要证明的子命题：……"这类标签**。
   - 每个板块**结尾必须放一个 `回扣主线` 小节**（标题就叫"回扣主线"），用 2–3 句：先一句话**总结本章**，再**自然说清本章结论"印证 / 强化 / 限定（部分修正）"了核心判断、与哪个 T 信号联动**。这个小节兼具本章小结与回到主线的双重作用。用行文表达，不要写成"本板块支持主线"的生硬套话。**发现反证要如实写、不藏。**
   - 正文出现的核心数字与主线底稿"核心数据"对得上、不自相矛盾。

## 各板块方法论

### 核心观点（执行摘要）

- 开篇 2–3 句给核心判断 + 关键数字（规模、增速、所处周期位置）——这几个数字应与判断主线的"核心数据"一致。
- **必写"市场共识 vs 非共识判断"**：用一两句说清"市场已经知道、已被定价的是什么（共识）"，再点出"本报告比共识多看到了什么、预期差在哪（非共识判断）"。这是让报告有 insight、而非资料汇编的关键。
- 用一段写出"主线论证链"：列 3–4 条核心论据，每条注明在哪个板块展开（如"装机量价齐升，见板块一"）。
- 给出面向决策意图的一句话结论（投资 / 入局 / 战略落地）。
- Markdown 写作时，核心判断用引用块突出，含核心判断 / 关键风险 / 验证信号三个要点。

### 板块一：行业定义、规模测算与历史坐标

- **1.1 行业定义与边界**：定义本行业、产业链范围、与相邻行业的边界。
- **1.2 市场规模（多源交叉 + 自建测算）**：
  - 至少 2 个来源对照当年/上年规模，差异 >30% 时并列展示并解释口径差异（出厂/零售、含税、销量/销售额）。
  - 给一条**自建测算**：写清量×价（或渗透率×空间）的拆解公式与假设。配 1 张**市场规模与增速表**（替代原规模柱状/折线图）。
- **1.3 增速、宏观敏感性与生命周期**：CAGR 预测（窗口到 当前年+5）；说明本行业最相关的 1–2 个宏观变量（如地产、出口、补贴、利率）及历史弹性；判断处于导入/成长/成熟/衰退哪个阶段。
- **1.4 历史复盘与周期律（如适用）**：用因果链行文复盘 2–3 个关键转折，每个挂 ≥1 个可溯源锚点。

### 板块二：行业结构、竞争格局与格局演变（核心板块）

- 开篇一句话点明行业属哪类结构（产业链型/网络型/资产负债型/单店模型型，见 `industry-structure-playbooks.md`），仅作行文取向。
- **2.1 产业链 / 结构全景**：**必配 1 张产业链全景表**（替代原结构图）——见下文"表格纪律"的产业链全景表规格。
- **2.2 价值分布与利润池**：价值集中在哪个环节、被谁拿走。配**各环节利润池表**（各环节毛利率/价值占比）。
- **2.3 头部玩家**：CR3/CR5 集中度、第一/二阵营划分与定位；配**份额与阵营表**。
- **2.4 格局演变与终局推演**：关键成功要素（KSF）排序 + 3–5 年终局形态判断 + 1–2 条"反转信号"（什么发生会推翻本判断），反转信号汇总到板块四 4.4。
- **2.5 估值与融资并购**：A 股可比公司 PE/PB 区间与当前分位（高/中/低位）；近年代表性融资/并购倒序列举。配**可比公司估值表**。
- **2.6 供需、产能与价格周期（如适用）**：产能利用率、价格走势、库存位置。
- **2.7 重点公司简析（1–3 家，仅用上市公司公开披露）**：主业、份额、最近财务概况。

### 板块三：核心驱动力与制约因素

- **3.1 五维驱动—制约矩阵**：从需求、技术、政策、成本、竞争五个维度，各写驱动力与制约因素。配**五维矩阵表**。
- **3.2 技术路线对比（如适用）**：主流技术路线优劣、卡脖子环节、国产化进度。配**技术路线对比表**。
- **3.3 政策梳理（中国行研标配）**：
  - **必做：政策梳理表 ≥5 行**，列：发布时间 / 发文机关 / 文件名称(文号) / 政策层级 / 核心内容 / 影响方向。
  - 一段写政策周期定位（扶持初期/加码/退坡/规范）与边际变化；前瞻信号汇总到 4.4。

### 板块四：趋势研判与结构性机会｜窗口 当前年–次年（必要时延 1 年）

板块四不应该是泛泛的行业趋势描述，而应该试图完成行业调研报告中的结构性机会研判。为了完成这种结构性机会研判，你应该试图围绕当年或1-2年内行业内部结构性变化，理解当前和未来的产业阶段及变化，试图识别外部关键变量，以及产业中最具确定性与最具弹性的细分方向，思考产业链受益环节、兑现时间表、关键验证信号和潜在风险。

- **4.1–4.3 三大趋势**：每个独立的趋势小节应由一个清晰的中心判断统领，后续讲清做出这样趋势判断的原因，完成一条连贯的分析。具体而言，需要说清楚 该趋势判断与当前产业阶段之间的关系，为什么这个趋势可能发生或正在发生，是否有证据可以支持这样的趋势判断，它会如何影响产业链的价值分配，哪些环节因此受益或承压，以及后续应通过什么信号验证判断是否成立。**上述每小节的主题内容应该以自然段承载，每个自然段之间要有良好的逻辑衔接，在引用证据、数据的时候需要有来源**。若多方受益、阶段节奏或验证指标较多，优先使用表格结构化呈现，表格前后仍需用自然段说明其背后的判断逻辑和研究含义，
- **4.4 信号验证清单（汇总表）**：把 2.4 反转信号、1.4 复盘启示、3.3 政策前瞻信号集中成一张表：列"观察指标 / 阈值或方向 / 看多还是看空 / 数据来源"。这是全报告"该盯什么指标"的唯一总表。
- **4.5 一致预期与预期差**：市场主流预期是什么、本报告的预期差在哪、≥2 条反方观点及回应。（投资意图额外写一句"当前估值大致定价了哪一档情景"。）
- **4.6 出海/全球化（如适用）**：海外空间、竞争对手、本土玩家出海进展。
- **4.7 三情景推演（必做，板块四收尾）**：乐观/中性/悲观三档。可直接搬用判断主线的"三情景速览"。每档必须给**结果**，不能只列假设：①假设（前两行直接用 1.2 的量价参数）；②对应**规模/增速结果**；③**利润/毛利率方向**；④**对结论的含义**（估值/投资/入局判断如何变）；⑤触发条件（挂某个 T 信号）。配**三情景对比表**。**"只有定性描述、没有结果"是常见硬伤，务必避免。**

### 板块五：商业化路径、盈利质量与落地建议

- **5.1 盈利模型（二选一）**：A=单位经济（单位售价−单位成本=单位利润）；B=成本结构（拆成本大项+盈亏平衡）。资产负债型用 B 的变体（息差/承保利润率、成本收入比、信用成本）；单店模型型补单店模型拆解。
- **5.2 回报率与现金流画像（必做）**：ROE/ROIC 的水平与趋势 + 杜邦拆解（净利率×周转×杠杆）+ 现金流三指标（经营现金流、资本开支、自由现金流）。可比样本 ≥3 家时给中枢与区间，配**盈利质量对比表**。
- **5.3 落地路径**：只写已锁定的决策意图（投资就写投资视角；入局就写进入路径），不要四类角色都铺一遍。
- **5.4 进入壁垒**：技术/资金/牌照/渠道/规模等壁垒，与 2.4 的 KSF 排序呼应。
- **5.5 失败案例（可选）**：有 ≥2 条可溯源案例才单列，否则并入 2.3 或风险提示。
- **5.6 90 天行动建议（仅入局/战略落地意图写）**：3–5 条可执行动作。

### 风险提示｜5–7 条

- 每条写清"风险事件 + 触发逻辑 + 影响"，并与 4.4 的某个观察指标呼应（哪个指标超阈值即兑现）。
- 合规标配，不可省。

### 结语

- 收束主线，重申核心判断与最关键的 1–2 个观察指标；不引入新论点。

### 参考文献｜≥15 条（`[n]` 编号）

- **按正文首次引用顺序誊抄**：来源台账只负责保存标题、URL、机构、日期、口径、源值与支撑事实；最终编号以整篇正文的阅读顺序为准，第一次出现的新来源编号为 `[1]`，第二个新来源为 `[2]`，后续递增；同一来源多处引用复用同一编号。每条参考文献独立成行，使用 `[1]`、`[2]` 格式，不使用 `1.` / `2.` 有序列表；同一机构不同报告分列；与正文数字引用一一勾稽。
- 每条给真实可点击链接（来自 `general_search` 或用户提供）；无公开链接的数据库数字作为"未公开链接的尽调输入"披露，不占编号。

**通用表格纪律（稳定 Markdown 表格）**：

- 每个板块至少 1 张表；表格用于真正并列、需要对比或溯源的信息，不要把连贯论证塞进表格。
- 表头简洁，对比性强；实质性列用完整句子单元格。
- 不要使用合并单元格、HTML 表格或嵌套表格；宽表优先拆成多个小表，复杂对比优先拆成“总表 + 分表”。
- 不要连续堆叠多张大表——两张大表之间要有论述衔接。
- 每张表正下方加一行"资料来源：<具体机构/文件>"，命名到具体机构或文件名（如"国家能源局，本研究测算"），不写"行业调研""公开资料"这类空泛写法；含本研究测算的表必须出现"测算"二字。
- 表里的规模/价格/份额/估值/财务数字必须来自 `seed_finance_search` 口径（一/二级来源）。

### ★ 产业链全景表规格（招牌表，板块二必出）

产业链全景表是行业研报的"招牌表"，**必须信息丰富、有层次，不能是三行空白**。建议结构（行=环节层，列=维度）：

| 产业链环节 | 主要构成（4–6 项） | 价值/盈利特征 | 关键节点与注释（份额/毛利率/国产化率） |
|---|---|---|---|
| 上游 · 核心元器件/材料 | … | 毛利 50–70% | 光纤激光器 51%；泵浦源芯片 国产<30% |
| 中游 · 整机/组装 | … | 毛利 12–20%、价格战 | … |
| 下游 · 应用/运营 | … | 需求驱动 | … |

要点：

1. **3 行结构**（上游/中游/下游），环节名点出该层功能定位（如"上游·核心元器件"）。
2. **每行写"价值/盈利特征"**：用一句极简的价值/盈利特征——**这是体现"利润池"的关键**，三行连起来就讲清了利润分布与迁移方向。
3. **关键节点加注释**：给每层 2–3 个最重要的节点加份额/毛利率/国产化率注释，让读者一眼看到价值与卡脖子环节。
4. 节点与注释**必须真实可溯源**（份额/毛利率走 `seed_finance_search` 口径），查不到的注释宁可不写，不要编。
5. 这张表与 2.2 的利润池分析互相印证：表里的盈利特征/注释应与正文"利润从哪个环节流向哪个环节"一致。
