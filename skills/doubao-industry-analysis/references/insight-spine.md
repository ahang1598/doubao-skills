# 判断主线底稿（Insight Spine）

在检索取证结束、动笔写正文之前，先用一张短表把"这篇报告要证明什么判断、凭哪几个核心数据、哪些信号能验证或推翻"想清楚。之后写每个板块都回头对照它，保证全文一条主线，不写成"什么都讲一点的资料汇编"。

结构齐、能渲染成飞书文档的报告，也可能答不上"核心判断是什么、和市场共识差在哪、什么能证伪"。判断主线底稿正是把这件事先定下来：它本身只是一张短表，却决定了后面每一批正文的聚焦点。

## 一、什么时候做、产出什么

- **时机**：工作流程第 2 步（检索取证）做完、第 4 步（分批写正文）开始**之前**。
- **产出**：判断主线底稿**写在对话上下文里**，尽量在 **5 分钟内写完**（工作中间件，不进入正文、不落本地盘——本地文件跨回合会丢；写每批正文时对照它当尺子）。
- **原则**：先有判断、再写正文。若检索证据不支持你最初的判断，**先改这张表，再动正文**——不要把五大板块都写完了才发现主线立不住。

## 二、七格表（照这个模板想清楚）

逐格用真实行业内容替换 `…`：

```markdown
# {行业} 判断主线底稿

## 一句话主线
（一句话说清本报告的核心判断。要能被验证、能被推翻，不能是"行业空间大、长期看好"这种正确的废话。
 例：新增装机的确定性给估值托底，真正要验证的是利润修复；海风与出海决定上行弹性，电价市场化决定下行风险。）

## 七格表
| 格子 | 内容 |
|---|---|
| 决策问题 | 把用户那句话改写成一个**可判断**的问题（避免"这行业怎么看"这种泛问）。 |
| 市场共识 | 大家已经知道、已经被价格反映（定价）了的部分是什么。 |
| 非共识判断 | 本报告真正要证明、和共识不一样的判断是什么（不能只是"有机会"）。 |
| 核心数据 | 支撑主线的 **3–5 个关键数字**（规模/增速/份额/价格/毛利率等），每个带单位与年份。**这几个数字后面写正文时要反复用、不能自相矛盾。** |
| 因果链 | 用"驱动 → 传导 → 结果"写一条链，至少 4 个节点。 |
| 利润池/价值迁移 | 钱（利润）正从哪个环节流向哪个环节，谁受益、谁受损。链式行业按上游/中游/下游写。 |
| 验证与证伪信号 | 列 **5 个左右**带编号的信号 T1、T2…：哪些发生→印证主线（正向），哪些发生→推翻主线（反向/证伪），各注明大致阈值或方向。 |

## 三情景速览（先把结果想好，正文板块四 4.7 直接用）
| 情景 | 关键假设（用上面的"核心数据"取档） | 规模/装机结果 | 利润/毛利率方向 | 对结论的含义 |
|---|---|---|---|---|
| 乐观 | … | … | … | … |
| 中性（基准） | … | … | … | … |
| 悲观 | … | … | … | … |

## 各板块要证明的子命题（**内部抓手**：写正文时把它化成开头的"亮明判断"主题句，不要照抄成标签）
| 板块 | 这个板块要证明的子命题 | 对主线的作用（印证/限定） |
|---|---|---|
| 一、定义与规模 | … | 证明需求底线/历史坐标 |
| 二、格局与产业链 | … | 证明利润池迁移/格局终局 |
| 三、驱动与制约 | … | 证明政策与技术怎么传导 |
| 四、趋势与机会 | … | 证明预期差怎么验证（挂 T 信号） |
| 五、盈利与落地 | … | 证明盈利质量/回报能否支撑判断 |
```

## 三、填表的 6 条要点（决定这张表有没有用）

1. **主线要"可证伪"**：能写出"如果 X 发生，这个判断就错了"才算合格。写不出证伪条件，说明判断太空。
2. **核心数据要少而硬**：只挑 3–5 个真正支撑主线的数字，宁缺毋滥。这几个数字是后面全文的"锚"——正文里出现的规模、增速、价格、份额都要和它对得上，不能这段说 100GW、那段说 80GW。这些数字必须来自 `seed_finance_search` 的市场/财务硬数字，不得凭印象写。
3. **共识 vs 非共识要真的不同**：如果"市场共识"和"非共识判断"读起来差不多，等于没有预期差，重写。
4. **利润池要点名**：必须写清"利润从谁流向谁"。链式行业（制造/新能源/硬科技）务必写上中下游迁移方向。
5. **T 信号要可观测**：每个 T 要是能在公开数据里跟踪的（招标均价、并网量、毛利率、核准节奏等），不要写"行业景气度改善"这种没法量化的。至少 1 个正向、1 个反向。
6. **三情景先写结果再写假设**：很多报告三情景"只有定性描述"。这里强制先把**结果**（规模/利润方向/结论含义）想清楚，正文 4.7 直接搬过来，避免到时候只罗列假设、不给结果。

## 四、写正文时怎么用这张表（配合工作流程第 4 步分段写作）

- **每个板块开头**：照"子命题表"把该板块的子命题写成一句**"亮明判断"的主题句**（有态度、可被证明的论断），正文随后去证明它。**不要写成"本板块要证明的子命题：……"这类标签式说明**——子命题是你的内部抓手，读者应通过论证看出你在证明它。
  - ✗ 反面（标签式，禁用）："本板块要证明的子命题：利润池正从中游整机向上游元器件迁移。"
  - ✓ 正面（论断式主题句）："激光行业的利润正在沿产业链上移——中游整机在价格战中毛利已压至 12–20%，而上游泵浦源、特种光纤等核心元器件仍维持 50–70% 的高毛利，这一剪刀差正在重塑各环节的价值分配。"
- **每个板块结尾**：放一个 `回扣主线` 小节（标题就叫"回扣主线"），用 2–3 句先**总结本章**、再说清本章结论**印证 / 强化 / 限定（部分修正）**了核心判断、和哪个 T 信号联动。这是兼具本章小结与回到主线的"脊梁"小节。**发现反证要如实写**，不要藏着。用行文表达，不要写成"本板块支持主线"的生硬套话。
- **核心观点（执行摘要）**：开头就要让读者知道"大家已经知道什么（共识）"和"本报告新增的判断是什么（非共识）"，并点出那 3–5 个核心数据。
- **板块四 4.4**：把七格表里的 T 信号汇成一张"信号验证清单"表（观察指标/阈值或方向/看多看空/数据来源）。
- **板块四 4.7**：直接把"三情景速览"展开成正文，每档**必须给结果**。
- **风险提示**：每条尽量指回某个 T 信号（哪个信号超阈值→这条风险兑现）。

> 一句话：**这张表是写作前的"判断骨架"，不是附录。** 写正文的全过程都拿它当尺子，写完每一批回头对一眼"有没有跑偏、数字有没有打架、是不是在为主线添砖"。
