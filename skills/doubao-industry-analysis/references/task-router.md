# 任务路由（Task Router）

动笔检索或写作前先读本文件。它只决定三件事：本 Skill 该不该跑、这份报告在为哪个研究对象做承销、以及如何处理用户指定的焦点。它**不**定义正文排版、固定章节顺序、表格规格、引用规则或飞书渲染——那些在 `report-finalization.md`、`data-grading-and-citation.md` 与 `lark-doc-report-standard.md` 里。

## 一、确认这是一份完整行业报告任务

仅当用户想要一份**针对某一行业或赛道的中长期完整深度报告**时，才使用本 Skill：行业定义与规模、产业链与竞争格局、政策与驱动、景气周期、趋势研判与盈利质量。

用户可能强调一个侧重点，如规模测算、竞争格局演变、政策影响、估值、出海、某条技术路线、景气位置。该侧重点成为完整报告内部的一个**焦点**，不会把任务收窄成一个独立答案。

**不要用本 Skill** 处理下列任何一类（转对应场景）：

| 用户在问什么 | 转去哪里 |
|---|---|
| 该行业**今天**涨跌、热度、资金流向、龙头股表现 | 板块热度分析 |
| 某个**具体公司**怎么样、它有哪些竞对、估值贵不贵 | 公司分析 / 个股日报 |
| 某个**宏观事件**对市场的影响 | 市场信息 / 事件解读 |
| 行业内某公司的**财报 / 业绩** | 财报与业绩分析 |
| 一句话能答的窄问题、普通行业科普、翻译、生成性闲聊 | 不走本 Skill |

例如，"这个行业今天为什么大涨？"不应触发本 Skill；"某公司团队怎么样？"也不应触发，除非用户要的是把它放进一份完整行业深度报告里。

## 二、判定研究对象

套用报告结构前，先确定这份报告在为哪个**研究对象**做承销：

| 研究对象 | 何时选 | 报告含义 |
|---|---|---|
| 整体行业 / 赛道 | 用户给的是一个行业、赛道、细分领域 | 用标准行业研究逻辑：规模—格局—驱动—趋势—盈利全链条。 |
| 行业某一环节 / 子赛道 | 用户聚焦产业链某一环节（如上游材料、中游整机、下游运营）或某个子赛道 | 重点分析该环节的价值占比、利润池位置、议价能力、被替代风险与扩张路径。 |
| 某条技术路线 / 工艺路径 | 用户聚焦行业内多条技术路线的竞争（如电池技术路线、半导体制程） | 分析各路线的性能/成本/良率优劣、卡脖子环节、国产化进度、渗透率与终局。 |
| 景气周期位置 | 用户聚焦本行业当前处于周期哪个位置、何时反转 | 分析供需、产能利用率、价格、库存与历史周期律，给出周期定位与反转信号。 |
| 政策驱动型主题 | 政策正在创造或可能摧毁该市场（如补贴、出口管制、准入） | 重点分析政策是创造真实预算、仅放行活动、还是增加成本，及边际变化。 |

若多项适用，选择**决定价值分配**的那个作为主对象。无论选哪个，报告都要把它放进完整的行业研究框架里讲透，而非只答一个窄问题。

## 三、判定行业结构类型（决定分析侧重）

不同结构类型的行业，钱赚在哪、格局怎么演化、该盯什么指标都不同。先判定本行业属于哪一类，再到 `industry-structure-playbooks.md` 取对应的分析侧重与必查项：

- **产业链型**（制造 / 硬科技 / 新能源 / 医药 / 化工等）：价值沿上中下游分布，利润池在某个环节集中。**这是行业研究最常见的类型**，板块二必做产业链全景表与利润池分析。
- **网络型**（平台 / 互联网 / SaaS 生态）：价值来自流动性、网络效应、多边激励与用户黏性。
- **资产负债型**（银行 / 保险 / 券商等金融行业）：价值来自息差/承保利润、资产质量、杠杆与风险定价。
- **单店模型型**（连锁零售 / 餐饮 / 消费医疗等）：价值来自单店模型、密度、SOP、复购与坪效。

判定结果会影响：板块二的格局分析方式、板块五的盈利模型选择（产业链型用单位经济或成本结构、资产负债型用息差/成本收入比、单店模型型补单店模型拆解）。

## 四、记录用户焦点

若用户提到一个焦点，把它记为**报告完成后要回应的要求**，不要让它收窄、重排或替换完整报告。不要在完整报告草成之前，就为这个焦点单独跑补充检索。

工作顺序固定：

1. 先按完整行业报告的覆盖度取证、按五大板块写透。
2. 在报告最前面（执行摘要中或之前）用一句话回应用户焦点：`用户关注：{焦点}`。
3. 优先从已完成的报告里回应。若现有板块已支撑该回应，保持简洁并指向具体板块标题。
4. 若已完成的报告无法支撑该回应，再跑聚焦的补充检索，只补该回应缺的证据，并把它接回相关板块。

常见焦点：规模测算、竞争格局演变、政策影响、估值与分位、景气位置、某条技术路线、出海、预期差、某个细分赛道的机会。

## 五、交接

路由完成后：

- 用 `data-collection-protocol.md` 与 `data-grading-and-citation.md` 取证、分级、登记。
- 用 `industry-structure-playbooks.md` 选定行业结构分支与分析侧重。
- 用 `insight-spine.md` 想清楚判断主线（写正文前必做）。
- 用 `analysis-framework.md` 把事实转成判断。
- 用 `report-finalization.md` 作为五大板块的写作大纲与定稿规则。
- 用 `lark-doc-report-standard.md`（仅在内容完成后）作为飞书渲染与交付层。
