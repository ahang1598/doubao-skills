# 飞书文档报告标准（Markdown Upload）

本文件控制 Markdown 分片写作、本地拼接与整篇校验，以及最终导入飞书/Lark 文档时的交付门禁；它不决定分析判断或报告结构。正文在第 4 步按批写入 Markdown 分片，在第 6 步拼接为 `output/report.md`，再导入为飞书在线文档。

渲染必须服务于 SKILL.md 的质量标准：数据真实可溯源、判断贯穿一条主线、信息详实、克制专业。如果某个 Markdown 写法、表格样式或飞书展示效果让报告显得浅、拥挤、不可溯源或太像模板，简化渲染、保留内容质量。

## 目录

- 文档输出
- 分片与命名
- Markdown 写法
- 拼接规则
- 校验规则
- 导入与抽查
- 交付门禁

## 文档输出

- 访问可用且场景合适时，**创建一篇飞书/Lark 文档作为最终交付物**。
- 用描述性标题：`{行业}行业深度研究报告` 或 `{行业}行业深度研究——{一句话核心判断}`。
- 本地工作产物分为两层：`parts/*.md` 为批次分片，`output/report.md` 为最终上传源文件。
- 把文档链接返回给用户；当预期且可用飞书/Lark 交付时，不要只给本地路径。
- 文档里不要提及本 Skill 名称、内部检索流程、拼接脚本实现细节或本地载荷路径。

## 分片与命名

- 默认把 6 批正文写入 `parts/` 目录下的独立 Markdown 文件。
- 固定文件名：
  - `parts/01-executive-summary.md`
  - `parts/02-industry-definition-scale.md`
  - `parts/03-structure-competition.md`
  - `parts/04-drivers-constraints.md`
  - `parts/05-trends-opportunities.md`
  - `parts/06-commercialization-risk-refs.md`
- 主标题只允许出现在 `01-executive-summary.md`。
- `参考文献` 与免责声明只允许出现在 `06-commercialization-risk-refs.md`。
- 板块一至五对应的分片末尾必须包含 `### 回扣主线` 小节。
- 禁止手工新建临时合并稿、禁止把多个批次混写在同一个分片里、禁止改变分片顺序。

## Markdown 写法

- 用标准 Markdown 作为唯一写作源，不再使用旧正文格式。
- 标题层级固定：
  - `#` 主标题
  - `##` 核心观点、板块一至五、风险提示、结语、参考文献
  - `###` 各板块小节与 `回扣主线`
- 执行摘要的核心判断与每章 `回扣主线` 用引用块表达，例如：

```markdown
> **核心判断：** ...
> **关键风险：** ...
> **验证信号：** ...
```

- 不强依赖飞书专属高亮块；导入后只要求这些块在视觉上与普通正文可区分。
- 正文用紧凑数字标记作来源引用（`[1][2]`），**标在数据所在句子的句末**；表格在表下加 `资料来源：<具体机构/文件>`。
- 只使用 Markdown 原生可稳定导入的结构：标题、段落、列表、表格、引用块、粗体、斜体、代码、链接。
- 不使用 html 标签、内嵌 xml、自定义颜色、合并单元格或依赖飞书专属块级语法的写法。

## 拼接规则

- 最终整篇必须通过脚本拼接，不使用手动复制粘贴作为正式流程。
- 使用 `scripts/assemble_report.py`，按固定顺序读取 6 个分片并输出 `output/report.md`。
- 推荐命令：

```bash
python3 scripts/assemble_report.py
```

- 拼接脚本至少完成以下动作：
  - 检查 6 个分片文件都存在且非空。
  - 按固定顺序拼接，不扫描目录自动排序。
  - 片段之间统一补足空行，避免标题、表格、引用块粘连。
  - 检查主标题只出现一次。
  - 检查 `参考文献` 与免责声明只出现一次且位于最后一片。
  - 检查板块一至五各有一个 `回扣主线` 小节。
  - 兜底校正引用编号：调用 `scripts/renumber_references.py` 按正文首次出现顺序重排引用编号并同步重排文末参考文献。写作时"按首现顺序编号"的要求不变，此步仅保证最终交付一定满足 `[1]`、`[2]`、`[3]` 递增；编号已正确时为无操作。

## 校验规则

保留原有内容校验，但把在线同步校验改为本地结构校验与上传后抽查。

### 单批校验

- 每批写完立即检查：
  - 本批内容论证完整、信息充分、与主线一致。
  - 标题层级正确，没有跳级标题。
  - 必需表格与 `回扣主线` 小节齐备。
  - 本批新增数字已按台账完成源值核对，并在句末加引用。
  - 本批没有引入配图、html/xml 标签或旧正文格式残留。
- 推荐命令：

```bash
python3 scripts/validate_report.py --stage part --file parts/03-structure-competition.md
```

### 阶段校验

- 批3 写完后，对 `01` 到 `03` 片做一次阶段校验：
  - 前三片结构完整、顺序正确、文件非空。
  - 引用编号延续正确：正文首次出现的新来源必须按 `[1]`、`[2]`、`[3]` 递增；允许后文复用旧编号，但不允许新来源编号回跳成 `[1][4][2][3]`。
  - 核心观点、板块一、板块二已形成完整论证链。
  - 不上传飞书。
- 推荐命令：

```bash
python3 scripts/validate_report.py --stage stage
```

### 整篇校验

- 六批完成后，拼接前后都要做整篇校验；建议使用 `scripts/validate_report.py`。
- 整篇至少检查：
  - 9 个板块齐备，板块一至五都有 `回扣主线`。
  - 主标题、`参考文献`、免责声明只出现一次。
  - 正文与文末参考文献引用勾稽完整，参考文献使用 `[1]`、`[2]` 格式且 `>=15` 条。
  - 重要硬数据以句末引用呈现，并已对照台账源值完成准确性核验。
  - 本版无任何配图；原本用图承载的并列信息已全部用表格呈现。
  - 4.4 信号清单表、4.7 三情景表、产业链全景表齐备。
  - 每个小节在表格或列表前有一句小结或判断句引出，没有“只有表格/列表、没有论述”的小节。
- 推荐命令：

```bash
python3 scripts/validate_report.py --stage final --file output/report.md
```

## 导入与抽查

- 本地整篇校验通过后，再导入 `output/report.md` 生成飞书/Lark 文档。
- 使用 Drive 导入能力把 Markdown 转为在线文档，不走逐块更新。
- 推荐命令：

```bash
lark-cli drive +import --as user --file output/report.md --type docx --name "{行业}行业深度研究报告"
```

- 如果导入返回异步 ticket，继续查询：

```bash
lark-cli drive +task_result --scenario import --ticket <TICKET>
```

- 可使用 `scripts/upload_report.py` 封装导入与 ticket 查询。上传前可先 dry-run：

```bash
python3 scripts/upload_report.py --file output/report.md --title "测试行业深度研究报告" --dry-run
```

- 上传后做一次交付抽查，重点看：
  - 标题层级与目录导航是否正常。
  - 引用块、表格、列表是否被正确渲染。
  - 正文数字引用是否仍位于句末。
  - 文末参考文献链接是否可点击。
  - 分片拼接处是否出现空行丢失、表格断裂、标题粘连。
- 若上传后渲染异常，优先修正 Markdown 源文件并重新拼接上传，不在飞书端手工补丁式修文。

## 交付门禁

交付前确认：

- `parts/` 下 6 个 Markdown 分片齐备、命名正确、内容非空。
- `output/report.md` 由脚本按固定顺序生成，而不是手工拼接。
- 报告保留固定 9 板块结构，五大板块每章末尾有 `回扣主线` 小节。
- 执行摘要的核心判断与各章 `回扣主线` 已用引用块等 Markdown 原生方式突出呈现。
- **本版无任何配图**；原本用图承载的并列信息已全部用表格呈现；4.4 信号清单表、4.7 三情景表、产业链全景表齐备；每张表有具体 `资料来源` 行、含测算的表出现“测算”二字。
- 文档不无理由地表格堆叠；每个板块 `>=1` 张表。
- 每个小节在表格或列表前有一句小结或判断句引出，没有“只有表格/列表、没有论述”的小节。
- 重要硬数据（规模、增速、份额、价格、毛利率、估值、政策、产能、出货等）都在相关板块以**句末**数字引用呈现、可溯源，且已对照台账源值完成准确性核验（见 `data-grading-and-citation.md` 第六节）。
- 文档不出现“本板块要证明的子命题”这类标签；主线绑定通过亮明判断的主题句与 `回扣主线` 小节体现。
- 正文与文末参考文献引用勾稽；参考文献使用 `[1]`、`[2]` 格式且 `>=15` 条、链接可点击。
- 免责声明附在末尾、未删改。
- 飞书文档链接已可用，上传后抽查通过。

> 若上传失败或渲染异常：在对话中给出完整 Markdown 报告内容并说明限制，不要只给一个失败的链接。
