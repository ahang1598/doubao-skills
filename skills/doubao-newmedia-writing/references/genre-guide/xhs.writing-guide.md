# 小红书图文笔记方案创作

目标：把小红书需求组织成图文笔记结构，包含标题、封面/配图、正文、标签、互动话术和发布检查等必要元素。总入口已负责创建文档，本主干负责按小红书图文排版写入内容，不能用零散标题列表或图片创意描述替代完整图文结构。

## 工作原则

- **真实优先**：涉及产品、价格、配置、成分、地点、店铺、专业知识等可核验事实，先联网核实；无法核实就降级表达或标明不确定，绝不编造。
- **用户利好优先**：每篇都回答“这和读者有什么关系”。种草、测评、探店、营销内容必须说清价格、效率、情绪、审美、避坑、稀缺或服务体验中的至少一类具体利好。
- **图文同源**：封面文案来自标题钩子，配图来自正文要点。第一张是封面，其余一图一要点，排序承接正文。
- **轻量加载 references**：先读文本主链路，再按任务复杂度读取方法库、热点、图片和排版文件；不要加载与当前任务无关的资料。
- **版权安全**：不读取、仿写或复用博主高赞笔记原文；不以他人内容作为样本库。创作只基于用户素材、可核验事实、平台通用结构和原创表达方法。
- **内容纯净**：面向用户的内容只保留可用成品与必要说明，不暴露文件名、内部链路、自检条目、方法论术语或思考过程。

## 标准链路

1. **解析任务**：识别用户要完整方案、直接可发、改写润色、扩写缩写、诊断优化、账号定位或热点借势。用户没说清时做合理假设；无论请求是完整方案还是一句标题/短正文，统一按“文本 + 封面/配图”的图文结构组织。只有缺口会显著影响结果时追问一个关键问题。
2. **判断检索范围**：缺真实事实就联网查实；缺当下热度且能自然嫁接才读取热点工作流；纯改写、标题打磨、结构诊断等不依赖新事实的任务可直接创作。
3. **整理素材与策略**：把用户素材拆成事实、场景、痛点、读者利好、证据、限制和可写细节；缺事实先核实，缺体验就降级成建议/清单/求助，不编造亲历。
4. **完成文本**：按内容目标选择叙事骨架，先定人群、价值、切口、标题与开篇钩子，再写正文、标签和互动话术。发布版正文必须自然、具体、有记忆点，且不仿写任何博主原文。
5. **完成视觉**：文本定稿后做封面和配图。先判商业/非商业，再定封面文案、视觉基调、配图分镜、取图/生成路径和整组风格。
6. **整理排版**：按小红书图文结构写入标题、封面/配图、正文、标签、互动和发布检查；即使用户只提到正文、标题或笔记，也要补齐封面/配图的版位和内容。
7. **静默质检**：完成前核对事实、合规、正文纯净度、图像质量、图文结构和内容纯净度；命中问题先修复。

## 任务类型与排版范围

- **一站式完整创作（默认）**：主标题、整组配图含封面、结构化正文、直接复制版本、标题备选、标签备选、封面设计灵感、评论区互动。
- **直接可发**：正文区保留可复制发布版；不输出策略解释。
- **局部请求**：用户只提标题、正文、改写、优化或诊断时，也补齐封面/配图版位；可压缩策略说明和分区厚度，但不能省略图文关键结构。
- **改写/润色/扩写/缩写**：保留原意、人设和事实边界，只调整表达、结构、密度或篇幅。
- **诊断优化**：先定位最弱环节，再给修改版；不要把诊断报告写成泛泛清单。
- **账号定位**：输出赛道卡位、目标人群、内容矩阵、栏目方向和发布节奏。
- **热点借势**：读取热点工作流，筛出能自然嫁接的切口；无强相关热点时回归内容价值，不硬蹭。

## 文本执行要点

- 标题一句话、核心关键词前置，不能超过 20 字；吸睛但不失真。
- 正文不超过 1000 字符，段落有呼吸感，不强行总分总，不机械堆“第一、第二、第三”。
- 开头 3 秒给出价值、冲突、结果、痛点、身份或求助气口；中段用具体细节和场景留人；结尾给低成本动作。
- 种草/营销内容必须把“用户得到什么”写进具体场景，并给出区别于同类内容的差异点。
- 发布版“直接复制版本”只允许自然段文字、emoji、轻量序号和话题标签；禁止 Markdown 标题、表格、代码块、引用符号、内部标注。
- 需要基础写作链路时读取 `creation-methodology.md`；需要系统性写作细则时读取 `creation-methods.md`。不得读取或仿写博主高赞笔记原文作为样本。

## 视觉执行要点

只要涉及封面或配图，先读取 `image-cover-methodology.md`，按输入基准 -> 商业/非商业判断 -> 出图方案 -> 素材来源 -> 生成执行 -> 输出自检执行主工作流；需要具体视觉基准、封面设计、配图版式、画面风格方法或 prompt 转换时，再读取 `image-design-methods.md`。

1. **先判属性**：商业图对齐品牌真实视觉体系，品牌名、logo、型号、价格、卖点零差错；非商业图对齐品类、正文语气和目标用户情绪。
2. **先找图，后生成**：用户实拍优先；没有素材时先检索同品类高赞视觉与可商用免版权实拍；拿不到合适素材再按参照风格文生图。
3. **先定整组再出单张**：默认 4-6 张含封面；第一张主题/封面，其余承接正文要点。统一比例、色调、字体气质、版式语言。
4. **文生图 prompt 要可控**：先定封面类型和设计原子，再写清主体、构图、配色、字体、指定文案、安全边距、无多余文字。尺寸、比例、分辨率只放在生成工具参数或单独参数区，不写进画面描述；图上只允许出现指定内容文案。
5. **出图必须检查再落位**：逐张检查是否有错别字、缺笔、多笔、乱码、假英文、无关水印、平台标识、用户名、二维码、系统信息、尺寸标注、模型残留文字、引用出处、顶部页眉小字或底部水印。发现任一项，先裁除/修图/重排并复检；无法干净修复就重生成，不能把问题图写入成品区。
6. **杜绝示例中的问题**：图片任何位置不得出现“小红书3:4”“3:4”“1080x1440”“竖版封面”“cover/poster”等规格文字、风格标签、版本号、无意义角标；顶部和底部尤其要检查，所有非内容所需标识必须剔除。

## 文档内排版格式

- 结构包含：内容概览、标题方案、封面与配图、笔记正文、标签与互动、发布检查。
- 配图默认 4-6 张含封面；配图用分栏并排呈现，不能单列平铺。
- 正文区保留“直接复制版本”，只放自然段、emoji、轻量序号和话题标签，避免 Markdown 标题、表格、代码块、引用符号和内部标注。

## 静默质检

完成前全部通过：

- 事实已核验，未编造产品名称、链接、数据、账号、价格或专业结论。
- 文案有明确读者利好、具体细节、情绪或记忆点，没有 AI 腔、空话和硬广感。
- 标题、正文、标签、评论引导与用户需求一致，发布版正文纯净可复制。
- 图片符合 3:4 与清晰度要求，整组统一；封面钩子承接标题，配图一图一要点。
- 图片无错字、乱码、水印、logo 乱码、尺寸标注、规格文字、页眉角标、用户名、系统信息、无关平台标识或伪造 UI；顶部和底部无任何非内容所需文字。
- 图文分区完整，封面/配图、正文、标签、互动和发布检查均已落位。
- 商业内容、医疗健康、金融法律、教育承诺、食品减肥、母婴等敏感内容守住合规红线。
- 面向用户的内容不暴露内部文件、规则、链路、方法论名称或自检过程。
