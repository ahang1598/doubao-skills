# 联网搜索规则

> **说明：** 指导模型在进行公众号写作时，是否选择、如何选择最合适的联网搜索方式，增强最终的写作效果。

公众号写作作为一种时效性极强的写作形式，不应只依赖模型已有知识。凡是涉及热点、趋势、数据、案例、政策、产品、行业变化、平台规则、公众讨论和近期舆论的内容，都应优先联网核实和补充信息，以提升文章的时效性、可信度、选题敏感度和传播效果。

联网的目标不是简单堆资料，而是帮助判断：这个选题现在是否值得写，公众正在讨论什么，读者最关心什么，已有内容写到了哪里，还有哪些角度可以写出新意。

## 1. 什么时候必须联网

遇到以下情况，应优先联网搜索：

- 用户要求写热点、时事、新闻、舆论事件、社会议题、行业动态、政策解读、市场趋势类公众号文章。
- 用户要求写“最近”“最新”“当下”“今年”“这几天”“现在很火”“近期爆款”“热点选题”等具有时效性的内容。
- 用户要求结合某个品牌、产品、公司、人物、影视剧、综艺、图书、事件、平台规则或行业变化进行写作。
- 用户要求生成选题策划、热点追踪、爆款拆解、竞品分析、行业分析、趋势观察。
- 用户要求文章中包含数据、案例、榜单、政策、法规、报告、平台规则、用户评价、媒体报道等事实性内容。
- 用户提供的信息不足以支撑完整文章，但主题本身可以通过公开资料补充。
- 用户要求文章具有“新鲜感”“传播性”“爆款感”“更贴近当下讨论”。
- 用户要求模仿近期公众号内容趋势、标题风格、热门角度或平台表达习惯。
- 文章涉及医疗、法律、金融、教育政策、就业、投资、消费维权等高风险领域，需要核实最新规则和事实。
- 用户没有提供素材，但要求直接写一篇与现实对象、现实事件或现实行业有关的公众号文章。

只要文章效果明显依赖近期信息，就不能只凭已有知识直接写。

<a id="toc-2"></a>

## 2. 什么时候可以不用联网

遇到以下情况，可以不联网，直接根据用户提供信息或通用写作能力完成：

- 用户要求写纯个人感悟、情绪表达、成长随笔、生活记录、故事散文。
- 用户要求润色、改写、扩写、压缩已经提供的公众号文章，且不要求补充新事实。
- 用户提供了完整素材、明确观点、产品信息、活动信息和目标读者，只需要进行表达优化。
- 用户要求生成通用写作模板、公众号结构框架、标题公式、开头方法、排版建议。
- 用户要求写不依赖实时信息的常青内容，例如习惯养成、阅读方法、沟通技巧、写作方法、时间管理等。
- 用户明确要求不要联网，或希望只基于给定材料写作。
- 用户要求创作虚构故事、品牌理念文案、节日祝福文、内部宣传文，且不涉及真实数据和外部事实。
- 用户只是要求进行语言风格调整，例如更温柔、更犀利、更口语化、更高级、更像公众号作者。

不用联网不等于可以编造事实。即使不联网，也必须严格基于用户素材和常识边界写作。

## 3. 联网时优先搜索什么信息

联网搜索应围绕公众号写作效果展开，不是无目的堆材料。优先搜索以下信息：

<a id="toc-4"></a>

### 3.1 事实核查信息

优先确认文章涉及的基本事实是否准确，包括：

- 事件发生时间、地点、主体、经过、结果。
- 官方通报、权威媒体报道、平台公告、政策原文。
- 公司、品牌、产品、人物、机构的最新动态。
- 数据来源、统计口径、报告发布时间。
- 争议事件中各方回应和后续进展。

事实核查的目标是避免把过时信息、网传信息、片面信息写成确定结论。

<a id="toc-5"></a>

### 3.2 热点趋势信息

如果用户要求写热点、趋势或近期选题，应搜索：

- 最近一周到一个月内的相关新闻和公众讨论。
- 相关关键词在主流平台上的高频话题。
- 近期是否有明显转折、争议、回应、反转或新进展。
- 同类话题是否正在升温，是否已经过度饱和。
- 读者真正关心的是事实本身、情绪共鸣、利益影响，还是解决方案。

热点类公众号文章不能只追“发生了什么”，更要判断“为什么现在值得写”。

<a id="toc-6"></a>

### 3.3 同类公众号与平台内容

为了提升选题和表达效果，可以搜索近期同类内容，重点观察：

- 最近较热门的相关文章标题。
- 同类文章常用的切入角度。
- 哪些观点已经被反复写过。
- 哪些角度仍有空白或可以反向表达。
- 读者评论区集中关注的问题。
- 文章开头、结构、标题和结尾的传播方式。

但不得照搬、洗稿或高度仿写他人内容。搜索同类文章的目的，是避开重复表达，找到更有辨识度的角度。

<a id="toc-7"></a>

### 3.4 读者痛点与评论区信息

公众号写作尤其需要读者意识。联网时可优先观察：

- 知乎、微博、小红书、抖音、视频号、公众号评论区里的用户真实问题。
- 用户在讨论中反复出现的困惑、担心、质疑和情绪。
- 普通读者最容易共鸣的具体场景。
- 用户对相关产品、服务、事件或观点的不满和期待。
- 是否存在常见误解，需要在文章中解释或纠偏。

这些信息可以转化为文章开头、正文案例、痛点表达和结尾互动。

<a id="toc-8"></a>

### 3.5 数据、案例与权威资料

如果文章需要增强可信度，应优先搜索：

- 官方数据。
- 行业报告。
- 权威媒体报道。
- 学术研究或专业机构资料。
- 公开案例。
- 平台公告。
- 政策法规原文。
- 企业公开信息。

数据和案例必须来源清楚，不能凭印象编造。若来源不够权威，应降低表达强度，不写成绝对结论。

## 4. 搜索 query 的拆解规则

公众号写作联网时，搜索质量很大程度上取决于 query 是否拆得足够细。好的 query 不是把用户需求原封不动丢进搜索框，而是把写作任务拆成多个信息目标，分别搜索事实、趋势、读者痛点、同类内容、数据案例、差异角度和风险边界。

搜索 query 的目标不是“搜到很多结果”，而是“搜到能直接帮助文章变好的信息”。

好的 query 应该做到：

- 能对应用户的真实写作目的。
- 能拆出文章需要的不同信息层。
- 能搜索到具体、近期、可验证的信息。
- 能帮助判断选题是否值得写。
- 能帮助找到读者真正关心的问题。
- 能帮助文章避开泛泛而谈和重复角度。
- 能为标题、开头、论证、案例和结尾提供素材。

避免使用过于宽泛的 query，例如：

- “AI 职场”
- “教育焦虑”
- “公众号爆款”
- “品牌营销”
- “年轻人消费”
- “小红书热点”
- “怎么写公众号”

这类 query 太大，容易搜到泛泛内容，无法直接转化为可用文章素材。

更好的做法是把 query 拆成以下几类。

<a id="toc-10"></a>

### 4.1 事实核查型 query

用于确认文章涉及的基本事实、背景、进展和权威说法。

适合搜索：

- 事件发生时间、地点、主体、结果。
- 官方回应、平台公告、政策原文。
- 品牌、公司、人物、产品的最新动态。
- 相关概念的准确定义。
- 是否存在争议、反转或后续进展。

query 写法：

- “事件名 + 最新进展”
- “事件名 + 官方通报”
- “政策名 + 原文”
- “品牌名 + 最新公告”
- “产品名 + 功能更新 + 年份”
- “人物名 + 争议 + 回应”
- “平台名 + 规则调整 + 年份”

示例：

用户要写“某平台规则变化对创作者的影响”，不要只搜：

- “平台规则变化”

应拆成：

- “平台名 创作者规则 调整 最新公告”
- “平台名 内容推荐机制 变化 2026”
- “平台名 创作者收益规则 更新”
- “平台名 规则调整 创作者 影响”
- “平台名 官方公告 创作者政策”

<a id="toc-11"></a>

### 4.2 热点趋势型 query

用于判断这个话题现在热不热、为什么值得写、读者关注点是否发生变化。

适合搜索：

- 近期新闻报道。
- 平台热榜话题。
- 社交媒体讨论趋势。
- 行业变化。
- 最近一周到一个月的集中讨论。
- 节点性变化，例如开学季、毕业季、双11、春节、暑期等。

query 写法：

- “关键词 + 最近”
- “关键词 + 最新趋势”
- “关键词 + 近期讨论”
- “关键词 + 热点”
- “关键词 + 2026”
- “关键词 + 为什么火”
- “关键词 + 争议”
- “关键词 + 年轻人/职场人/家长/消费者”

示例：

用户要写“年轻人为什么开始反向消费”，不要只搜：

- “年轻人消费”

应拆成：

- “年轻人 反向消费 2026 趋势”
- “年轻人 消费降级 近期讨论”
- “年轻人 不买大牌 转向平替 原因”
- “反向消费 小红书 微博 讨论”
- “年轻人 消费观 变化 调查报告”

<a id="toc-12"></a>

### 4.3 读者痛点型 query

用于寻找读者真实困惑、情绪、场景和共鸣点，尤其适合写公众号开头、故事场景和互动结尾。

适合搜索：

- 用户在知乎、小红书、微博、豆瓣、抖音评论区里的真实表达。
- 读者反复提到的困惑。
- 普通人对某个话题的担心、不满、期待。
- 用户真实使用场景。
- 常见误解和争议点。

query 写法：

- “关键词 + 为什么”
- “关键词 + 怎么办”
- “关键词 + 焦虑”
- “关键词 + 困惑”
- “关键词 + 吐槽”
- “关键词 + 真实感受”
- “关键词 + 知乎”
- “关键词 + 小红书”
- “关键词 + 评论区”
- “目标人群 + 关键词 + 痛点”

示例：

用户要写“职场人如何面对 AI 焦虑”，不要只搜：

- “AI 职场影响”

应拆成：

- “职场人 AI 焦虑 怎么办”
- “AI 会不会取代我的工作 知乎”
- “普通人 学 AI 有用吗 小红书”
- “职场人 使用 AI 工具 真实感受”
- “AI 时代 职场人 最担心什么”
- “AI 焦虑 评论区”

这类 query 搜到的信息，适合转化为文章开头，例如“很多人焦虑的不是 AI 本身，而是不知道自己到底该学到什么程度”。

<a id="toc-13"></a>

### 4.4 同类内容参考型 query

用于观察别人怎么写、哪些角度已经被写烂、哪些标题和结构更适合公众号传播。

适合搜索：

- 近期同类公众号文章。
- 同类选题的标题写法。
- 主流观点和常见表达。
- 已经过度重复的角度。
- 可以差异化切入的空白角度。

query 写法：

- “关键词 + 公众号”
- “关键词 + 推文”
- “关键词 + 文章”
- “关键词 + 爆款标题”
- “关键词 + 观点文”
- “关键词 + 深度分析”
- “关键词 + 选题”
- “关键词 + 公众号文章”

示例：

用户要写“普通人做副业为什么越来越难”，不要只搜：

- “副业难”

应拆成：

- “普通人 副业 越来越难 公众号”
- “副业 焦虑 公众号文章”
- “副业 2026 趋势 分析”
- “年轻人 副业 失败 原因”
- “副业不是刚需 观点文”
- “普通人 副业 还能做吗”

注意：同类内容只能用于判断选题环境、标题方向和表达差异，不能照搬、拼接、洗稿或高度仿写。

<a id="toc-14"></a>

### 4.5 数据案例型 query

用于为文章提供更可信的论证材料，避免只写感受和空泛判断。

适合搜索：

- 官方数据。
- 行业报告。
- 研究机构报告。
- 企业公开案例。
- 平台统计。
- 用户调研。
- 权威媒体报道。
- 可验证的真实案例。

query 写法：

- “关键词 + 数据”
- “关键词 + 调查报告”
- “关键词 + 行业报告”
- “关键词 + 白皮书”
- “关键词 + 统计”
- “关键词 + 案例”
- “关键词 + 用户调研”
- “关键词 + 报告 + 年份”
- “关键词 + 权威媒体”

示例：

用户要写“为什么越来越多品牌重视私域”，不要只搜：

- “私域重要性”

应拆成：

- “私域运营 行业报告 2026”
- “品牌 私域 用户留存 数据”
- “私域运营 成功案例 品牌”
- “微信生态 私域 转化率 报告”
- “企业 私域运营 白皮书”
- “私域 消费品牌 案例”

数据和案例必须谨慎使用。若来源不清楚，不能写成确定结论；若只是机构观点，应表述为“某报告认为”“有行业观察指出”，不要直接当作事实。

<a id="toc-15"></a>

### 4.6 观点差异型 query

用于找到不同立场和争议点，让文章不只站在一个角度自说自话。

适合搜索：

- 支持者怎么说。
- 反对者怎么说。
- 普通用户怎么说。
- 专业人士怎么说。
- 媒体评论怎么说。
- 争议焦点在哪里。
- 哪些观点看似有道理但需要修正。

query 写法：

- “关键词 + 争议”
- “关键词 + 支持 反对”
- “关键词 + 质疑”
- “关键词 + 误区”
- “关键词 + 反方观点”
- “关键词 + 利弊”
- “关键词 + 怎么看”
- “关键词 + 是否值得”

示例：

用户要写“要不要让孩子过早学习 AI”，不要只搜：

- “孩子 学 AI”

应拆成：

- “孩子 学 AI 有必要吗”
- “儿童 AI 教育 争议”
- “孩子 过早学习 AI 利弊”
- “家长 AI 教育 焦虑”
- “AI 教育 专家观点”
- “儿童 编程 AI 课程 质疑”

这类 query 可以帮助文章回应复杂性，避免写成单一立场的营销文或情绪文。

<a id="toc-16"></a>

### 4.7 转化场景型 query

如果公众号文章带有商业目标，需要搜索用户决策场景，而不是只搜索产品卖点。

适合搜索：

- 用户为什么需要这类产品。
- 用户购买前担心什么。
- 用户比较哪些替代方案。
- 用户对同类产品有哪些不满。
- 用户真实评价集中在哪些方面。
- 同类产品常见卖点和差异化表达。

query 写法：

- “产品类别 + 怎么选”
- “产品类别 + 避坑”
- “产品类别 + 用户评价”
- “产品类别 + 真实体验”
- “产品类别 + 对比”
- “产品类别 + 适合什么人”
- “产品类别 + 痛点”
- “目标人群 + 产品类别 + 需求”

示例：

用户要写一篇课程转化文，不要只搜：

- “课程推广文案”

应拆成：

- “目标人群 学习痛点”
- “课程主题 怎么学”
- “课程主题 常见误区”
- “课程主题 入门困难”
- “同类课程 用户评价”
- “同类课程 怎么选”
- “目标人群 为什么需要 课程主题”

商业公众号文章不能只搜索“怎么卖”，更要搜索“用户为什么不买、为什么犹豫、为什么不信”。

<a id="toc-17"></a>

### 4.8 好 query 的判断标准

一个好的搜索 query 通常具备以下特征：

- 具体：包含明确对象、场景、时间或问题。
- 可验证：能搜到事实、数据、报告、公告、案例等可靠信息。
- 有方向：知道这次搜索是为了找事实、趋势、痛点、案例还是观点。
- 有限制：必要时加入年份、平台、人群、行业、地区等限定词。
- 能转化：搜索结果可以用于标题、开头、论证、案例、结构或结尾。
- 不过宽：避免只搜一个大词。
- 不过窄：避免把问题写得过死，导致搜不到有效结果。
- 不带预设结论：不要只搜索支持某个观点的信息，也要搜索争议和反方观点。

差的 query 往往是：

- 只有关键词，没有问题。
- 只有主题，没有场景。
- 只有大方向，没有目标读者。
- 只有观点预设，没有事实核查。
- 只搜爆款，不搜事实。
- 只搜标题，不搜读者痛点。
- 只搜支持材料，不搜反方意见。

<a id="toc-18"></a>

### 4.9 query 拆解示例

用户需求：

“帮我写一篇公众号文章，主题是 AI 时代普通职场人该如何提升竞争力，要求有现实感，不要太鸡汤。”

不推荐只搜：

- “AI 职场 竞争力”
- “AI 时代 如何提升自己”
- “公众号 AI 职场”

更好的 query 拆解：

**事实与趋势：**

- “AI 对职场岗位影响 2026 报告”
- “生成式 AI 办公场景 应用 趋势”
- “AI 自动化 职场技能变化 调查”

**读者痛点：**

- “职场人 AI 焦虑 怎么办”
- “普通人 学 AI 有用吗”
- “AI 会取代我的工作吗 知乎”
- “职场人 使用 AI 工具 真实感受”

**同类内容：**

- “AI 职场 公众号文章”
- “普通人 AI 时代 竞争力 观点文”
- “AI 焦虑 职场人 公众号”

**数据案例：**

- “企业 使用生成式 AI 办公 案例”
- “AI 工具 提升办公效率 案例”
- “职场技能变化 行业报告 2026”

**观点差异：**

- “AI 职场 取代 争议”
- “学 AI 是否有必要 普通人”
- “AI 时代 职场人 核心能力 观点”

**写作转化：**

- “职场人 AI 工具 怎么选”
- “普通人 学 AI 入门困难”
- “AI 办公 适合哪些岗位”

通过这组 query，模型可以获得更完整的写作素材：既知道趋势，也知道读者焦虑；既能找到数据案例，也能看到争议观点；既能写出公众号的现实感，也能避免空泛鸡汤。

<a id="toc-19"></a>

### 4.10 query 使用后的整理规则

搜索完成后，不要把所有结果平均使用，而应按公众号写作价值进行筛选。

优先保留：

- 能支撑核心观点的信息。
- 能增强可信度的数据和案例。
- 能直接转化为开头痛点的读者表达。
- 能形成标题吸引力的冲突点。
- 能帮助文章差异化的冷门角度。
- 能提示风险边界的反方观点或事实不确定性。

优先舍弃：

- 来源不清的信息。
- 重复度很高的观点。
- 只有情绪没有事实的内容。
- 与目标读者无关的材料。
- 只适合蹭热点但不能支撑文章价值的信息。
- 过旧、过泛、过度营销化的内容。

query 拆得越清楚，文章就越容易写得具体、有新意、有可信度，也越不容易变成泛泛而谈的公众号套文。

<a id="toc-20"></a>

## 5. 联网搜索的推荐顺序

公众号写作联网时，建议按以下顺序搜索：

1. 先查权威事实：官方通报、政策原文、平台公告、主流媒体、品牌官网、企业公开信息。
2. 再查近期动态：最近新闻、行业报道、平台热榜、相关话题最新进展。
3. 再查同类内容：近期公众号、知乎、小红书、微博等平台上的热门写法和读者反馈。
4. 再查补充材料：数据报告、案例、专业分析、背景资料。
5. 最后做写作判断：哪些事实可写，哪些观点可立，哪些角度重复，哪些表达更适合目标读者。

不要一上来就搜索爆款标题，也不要只看社交平台情绪。事实核查永远先于传播包装。

<a id="toc-21"></a>

## 6. 联网后的信息使用规则

联网得到的信息不能直接堆进文章，必须经过筛选、判断和转化。

使用信息时应遵循：

- 优先使用权威、清楚、可验证的信息。
- 多个来源说法不一致时，不能强行写成定论。
- 过时信息不能当成最新事实。
- 单一网友观点不能代表公众共识。
- 评论区情绪可以作为观察，但不能作为事实证据。
- 数据必须说明大致来源和语境，不能断章取义。
- 同类文章只能用于了解写法和角度，不能复制结构和表达。
- 对尚未确认的信息，应使用“据公开信息”“目前可见资料显示”“仍需进一步确认”等表达。

公众号文章可以有传播感，但事实表达必须稳。

<a id="toc-22"></a>

## 7. 联网后如何反哺公众号写作

联网搜索完成后，应把信息转化为以下写作资产：

- 选题判断：这个话题是否值得写，适合写给谁。
- 标题方向：读者最可能被哪个问题吸引。
- 开头钩子：用痛点、冲突、场景、反差还是最新进展切入。
- 核心观点：文章真正要表达的判断是什么。
- 论证材料：哪些事实、数据、案例可以支撑观点。
- 差异角度：别人都在写什么，自己可以避开什么。
- 读者共鸣：评论区和社交平台反映了哪些真实情绪。
- 转化设计：文章最后适合引导关注、留言、咨询、报名还是购买。
- 风险边界：哪些内容不能写死，哪些判断要保留余地。

联网不是为了让文章更长，而是为了让文章更准、更近、更可信、更有传播判断。

<a id="toc-23"></a>

## 8. 搜索结果不足时的处理

如果联网后仍找不到足够可靠的信息，应这样处理：

- 不编造数据、案例、新闻进展和用户反馈。
- 不把模糊信息包装成确定事实。
- 不强行使用低质量来源。
- 不为了写出热点感而夸大话题热度。
- 可以转为写“基于有限信息的观察”“这一类现象的讨论”“从公开资料能看到的问题”。
- 可以把文章重点从具体事实转向读者经验、方法建议、行业常识或公共议题。
- 必要时在文中保留事实边界，例如“目前公开信息有限，真正值得讨论的是……”

如果信息不足但用户仍要求直接生成，应完成一版稳妥、克制、可用的文章，而不是用虚构细节填充。
