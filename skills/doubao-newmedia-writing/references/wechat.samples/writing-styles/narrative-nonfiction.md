## 1. 类型定位：用一个具体的人，照见一种更大的处境

故事纪实类文章的核心不是“讲一个经历”，而是通过人物、场景、选择和细节，让读者进入一个真实处境，并从个体命运里看见行业、时代、关系、制度或生活困境。
它适合人物故事、访谈整理、非虚构特写、职业故事、社会切片、真实事件、历史叙事、地方记忆、口述文章等。
这类文章的质量取决于四件事：

- **人物是否具体**：读者能不能看见一个人，而不是一个标签。
- **处境是否有张力**：人物是否面对冲突、选择、代价或变化。
- **细节是否可信**：场景、动作、对话、物件是否来自真实材料。
- **主题是否能上升**：个体故事是否能抵达更大的问题，但不生硬拔高。
  一句话判断：**不是写“这个人很惨/很励志”，而是写“这个人为什么走到这里，又怎样被更大的环境塑造”。**

## 2. 选题方法：先找“人”，再找“处境张力”

故事纪实选题不要从抽象主题开始，而要从一个具体人物、一个关键选择、一个正在发生的变化开始。

<a id="toc-3"></a>

### 2.1 选题四步

1. **确定人物**：他/她是谁？职业、年龄、处境、身份有什么具体性？
2. **找到困境**：他/她正在面对什么问题？失去、转行、失败、等待、误解、老去、被替代、重新开始？
3. **提炼张力**：这个问题为什么不是简单的个人选择，而牵涉更大的环境？
4. **压成一句话**：用一句话说清“人物 + 处境 + 变化/代价”。
   差选题：一个小店老板的坚持。
   好选题：他撑过了最难的三年，最后发现小店最难守住的不是生意，而是人的信心。

<a id="toc-4"></a>

### 2.2 选题筛选问题

写之前至少回答：

- 人物是否足够具体？有没有姓名、职业、身份、经历或可替代化处理后的明确形象？
- 人物身上是否有一个正在变化的处境，而不是平铺直叙的经历？
- 文章是否有真实素材支撑：访谈、时间线、照片、记录、公开资料、现场观察？
- 个体经历能否连接到行业、时代、家庭、关系或社会结构？
- 是否有隐私、授权、事实边界问题？
  如果只有“某人很努力”“某人很不容易”，但没有场景、冲突和材料，不适合写成纪实长文。

<a id="toc-5"></a>

### 2.3 选题升级公式

- `人物身份 + 关键转折`：一个县城老师退休后，第一次承认自己跟不上学生了。
- `具体场景 + 命运变化`：关店那天，他把菜单擦了三遍。
- `个人选择 + 时代变化`：她离开大厂后，才发现“稳定”已经换了一种含义。
- `职业故事 + 行业处境`：一个外卖骑手的十年：从跑得快，到被算法追着跑。
- `人物命运 + 结构问题`：他不是不想留下，而是这座城市没有给普通人留下的空间。
  选题要避免：猎奇苦难、励志爽文、抽象群像、缺少素材硬写、把人物当观点工具。

## 3. 素材整理：先搭“事实骨架”，再找“可写细节”

纪实文章最怕一边写一边想象。动笔前要先把材料分层，明确哪些能写、哪些只能推测、哪些不能写。

<a id="toc-7"></a>

### 3.1 素材分四层

- **事实层**：时间、地点、人物关系、事件经过、公开记录、采访中明确说过的话。
- **场景层**：人物在哪里、做了什么、看见什么、听见什么、当时有什么物件或环境。
- **解释层**：这个经历背后的行业、家庭、制度、时代或关系背景。
- **主题层**：文章最终要让读者理解的处境、变化或问题。
  写作时要让事实层托住场景层，场景层支撑解释层，解释层再通向主题层。

<a id="toc-8"></a>

### 3.2 访谈/资料整理表

整理人物素材时，可按以下格式提炼：

| 维度    | 要找什么                | 写作用途    |
| ----- | ------------------- | ------- |
| 人物基本面 | 年龄、职业、地点、关系、经历节点    | 建立人物可信度 |
| 关键时间线 | 重要事件按时间排序           | 防止流水账混乱 |
| 冲突节点  | 最难、最纠结、最失控、最沉默的时刻   | 形成故事张力  |
| 选择与代价 | 人物做了什么，失去了什么，换来了什么  | 推动叙事    |
| 细节材料  | 动作、语言、环境、物件、习惯      | 形成真实感   |
| 背景资料  | 行业变化、政策环境、家庭结构、地方语境 | 增加深度    |
| 隐私边界  | 哪些不能写、需匿名或模糊处理      | 降低风险    |

<a id="toc-9"></a>

### 3.3 素材不足时怎么处理

- 没有现场，就不要写成“现场感很强”的特写。
- 没有原话，就不要加引号伪造对话。
- 没有心理依据，就写可观察动作，不写“他崩溃了”。
- 只有单方说法，要标明来源边界，避免下确定判断。
- 细节不足时，可以写成“人物侧写/口述整理”，不要伪装成深度纪实。

## 4. 标题写法：人物、场景、命运感三者至少占两个

纪实标题不要只写主题，要让读者一眼看见“谁发生了什么”。

<a id="toc-11"></a>

### 4.1 标题的核心组成

好标题通常包含：

- **人物**：一个小店老板、一个县城医生、一个离开大厂的人。
- **场景**：关店那天、凌晨三点、最后一班车、旧工厂门口。
- **变化**：离开、留下、失去、重新开始、被替代、终于承认。
- **悬念**：他为什么这样做？她后来怎样了？
- **主题暗线**：行业转身、城市代价、家庭困境、时代变化。

<a id="toc-12"></a>

### 4.2 常用标题模型

- **场景悬念型**：关店那天，他在门口站了很久。
- **人物命运型**：一个县城医生的二十年：从被需要，到被质疑。
- **关键选择型**：她离开北京后，终于不用在凌晨三点回消息。
- **反差变化型**：他曾经相信努力会有回报，后来发现规则变了。
- **时代切面型**：一个普通人的失业，照见一个行业的转身。
- **物件承载型**：那本写满欠账的账本，记录了一家小店的最后一年。

<a id="toc-13"></a>

### 4.3 标题自检

标题要问三件事：

1. 是否能看出人物或处境，而不是只有抽象主题？
2. 是否有真实内容支撑，不靠夸张制造悬念？
3. 是否尊重人物，不消费苦难、不猎奇、不把人生写成爽文？

## 5. 开头写法：用一个“能承载主题的瞬间”把读者带进去

纪实开头最重要的是“进入现场”。不要先介绍人物有多厉害、多不容易，也不要先讲大背景。

<a id="toc-15"></a>

### 5.1 好开头的标准

一个好开头应同时做到：

- 有人：读者知道镜头对准谁。
- 有动作：人物正在做什么，而不是作者在评价。
- 有场景：时间、地点、环境能让读者进入。
- 有张力：这一刻和人物处境有关。
- 有余味：读者想知道为什么会这样。

<a id="toc-16"></a>

### 5.2 五种精彩开头形式

<a id="toc-17"></a>

#### 形式一：关键动作开头

用一个动作带出人物处境。
写法：`人物 + 动作 + 场景 + 动作背后的异常感`
示例句式：

- 他把卷帘门拉到一半，又停住了。
- 她删掉那封辞职邮件后，重新打开，改了三个字。
- 老周把最后一张菜单擦干净，才发现门口已经没有排队的人。
  适合写离开、失败、转折、告别、关店、转行。

<a id="toc-18"></a>

#### 形式二：物件开头

用一个物件承载人物命运。
写法：`物件 + 状态 + 与人物关系 + 暗示变化`
示例句式：

- 那本账本的最后一页，只剩下三个名字。
- 她的工牌还放在抽屉里，照片上的笑已经和现在不太一样。
- 店里最旧的那张桌子，见过所有热闹，也见过最后一次打烊。
  适合写小店、职业、家庭、城市记忆、历史叙事。

<a id="toc-19"></a>

#### 形式三：反常瞬间开头

先写一个“不合常理”的瞬间，让读者产生疑问。
写法：`表面反常 + 简短解释留白`
示例句式：

- 失业后的第一个月，他反而睡得比过去更早。
- 离开大城市那天，她没有哭，只是把手机静音了。
- 父亲住院后，他第一次发现，家里最沉默的人其实知道最多。
  适合写人物变化、关系转折、成长后知后觉。

<a id="toc-20"></a>

#### 形式四：一句原话开头

用人物说过的一句话进入，但必须是真实材料，不能虚构。
写法：`“原话。” + 说话场景 + 人物状态`
示例句式：

- “不是不想干，是干不动了。”他说这句话时，手还扶在货架上。
- “我那时候以为，只要忍一忍就过去了。”她说完笑了一下，没有继续解释。
  适合访谈稿、口述整理、人物特写。

<a id="toc-21"></a>

#### 形式五：结果倒放开头

先给出结果，再倒回人物如何走到这里。
写法：`结果 + 反差 + 追问`
示例句式：

- 三年前，他还计划开第二家店。现在，他在找人转让最后一台冰柜。
- 她曾经最怕离开，如今却把所有行李装进了两个纸箱。
  适合有清晰时间线和强转折的故事。

<a id="toc-22"></a>

### 5.3 开头避雷

不要这样开头：

- “在这个时代，每个人都不容易。”
- “他是一个坚强、努力、善良的人。”
- “随着行业的发展，很多人面临困境。”
- “今天我们要讲一个感人的故事。”
  这些开头都在解释，没有把读者带入故事。

<a id="toc-23"></a>

## 6. 正文结构：不要按经历流水账，要按“冲突递进”组织

纪实文章不是人物年表。时间线可以作为底层骨架，但正文要围绕冲突和变化展开。

<a id="toc-24"></a>

### 6.1 推荐结构

1. **场景开头**：用一个关键瞬间进入人物。
2. **人物当下**：交代他/她是谁，现在面对什么。
3. **问题出现**：第一次裂缝、压力、变化或误判从哪里开始。
4. **冲突加深**：问题如何一步步扩大，人物如何应对。
5. **背景穿插**：解释个人困境背后的行业、家庭、城市或制度原因。
6. **关键选择**：人物做了什么决定，付出了什么代价。
7. **结果与余波**：决定之后生活发生了什么改变。
8. **回到主题**：用细节或克制判断把个人故事推向更大处境。
9. **余味结尾**：回到场景、物件、动作或一句话。

<a id="toc-25"></a>

### 6.2 每一节要回答一个问题

- 第一节：读者为什么要关心这个人？
- 第二节：这个人最核心的困境是什么？
- 第三节：困境是如何形成的？
- 第四节：他/她做过哪些努力？为什么不够？
- 第五节：这背后更大的环境是什么？
- 第六节：人物最终如何选择？代价是什么？
- 第七节：这个故事留下了什么问题？
  小标题不要写“01 人生转折”“02 命运无常”，要写出本节信息：
- 更好：第一家店关掉前，他还相信旺季会回来。
- 更好：真正压垮他的，不是房租，而是人流变了。
- 更好：她离开大厂后，才开始重新学习“准时下班”。

<a id="toc-26"></a>

## 7. 细节写法：细节必须推动叙事、呈现人物或支撑主题

细节不是越多越好。好的细节像钉子，把人物钉在真实生活里。

<a id="toc-27"></a>

### 7.1 三类有效细节

- **动作细节**：人物做了什么。比如擦桌子、反复改邮件、把钥匙放回抽屉。
- **物件细节**：人物和什么东西发生关系。比如账本、工牌、旧手机、药盒、车票。
- **语言细节**：人物说了什么，或沉默了什么。原话优先，概述次之。

<a id="toc-28"></a>

### 7.2 细节使用公式

`细节 + 解释一层 + 不解释过满`
示例：

- 他把门口的招聘启事撕下来，折了两次才扔进垃圾桶。那张纸贴了三个月，没有人来问过。
- 她把工牌放进抽屉，没有扔。那是她在这座城市留下的少数几样东西之一。
  写细节时不要急着升华，让细节自己说话。

<a id="toc-29"></a>

### 7.3 细节自检

每个细节都问：

- 它是否来自材料或可确认观察？
- 它是否让人物更具体？
- 它是否推动故事往前？
- 它是否支撑主题，而不是只为了好看？
  没有功能的细节删掉；未经核实的细节不要写成真实发生。

***

<a id="toc-30"></a>

## 8. 背景穿插：把“解释”嵌进故事，不要写成报告

纪实文章需要背景，但背景不能压过人物。最好的背景是从人物遭遇里自然长出来。

<a id="toc-31"></a>

### 8.1 背景插入时机

适合在这几处插入背景：

- 人物第一次遇到问题时：解释问题不是偶然。
- 人物努力失败时：说明个人努力为何不足以解决结构变化。
- 人物做选择前：交代他/她为什么没有更多选择。
- 人物结果出现后：把个体命运推向行业、时代或社会语境。

<a id="toc-32"></a>

### 8.2 背景写法公式

`人物遭遇 → 背景解释 → 回到人物`
示例结构：

- 先写他店里的客人变少。
- 再写街区人流、消费习惯、平台抽成或租金变化。
- 最后回到他如何调整菜单、延长营业时间、仍然没能改变结果。

<a id="toc-33"></a>

### 8.3 背景不要这样写

- 不要突然插入大段行业报告。
- 不要把人物故事变成观点文章。
- 不要用背景替人物下结论。
- 不要用“时代”“命运”“结构”这些词掩盖具体分析。
  背景只负责让读者理解：这个人的困境为什么不是孤立事件。

***

<a id="toc-34"></a>

## 9. 人物写法：立体，而不是正确

人物不需要完美，也不应该只被写成受害者、英雄、失败者或某种观点的证据。

<a id="toc-35"></a>

### 9.1 写人物的三个层次

- **外在处境**：他/她正在面对什么现实问题。
- **行为选择**：他/她做了什么，为什么这样做。
- **复杂性**：他/她的选择有哪些合理之处，也有哪些局限。

<a id="toc-36"></a>

### 9.2 不要替人物过度心理描写

除非材料明确提供，不要写：

- 他内心彻底崩溃。
- 她一夜没睡，想了很多。
- 他终于看清了人生真相。
  可以改成可观察表达：
- 他停顿了很久，才继续说下去。
- 她把手机屏幕按灭，没有再回复。
- 他第二天照常开门，只是把营业时间改短了两个小时。

<a id="toc-37"></a>

### 9.3 多方视角让人物更可信

可加入：

- 人物本人说法。
- 家人、同事、顾客、朋友、旁观者声音。
- 公开资料或时间线。
- 环境变化和外部数据。
  多方视角不是为了“凑材料”，而是避免故事只剩单一情绪。

***

<a id="toc-38"></a>

## 10. 语言风格：克制、具体、有画面，不替读者哭

故事纪实的语言要像镜头，不要像旁白煽情。

<a id="toc-39"></a>

### 10.1 推荐表达

- 多用具体动词：拉下、折起、停住、翻开、擦掉、递过去。
- 少用抽象评价：坚强、伟大、不易、心酸、命运无常。
- 用短句处理关键动作，用长句承接背景解释。
- 情绪从场景和动作里出来，不从作者感叹里出来。

<a id="toc-40"></a>

### 10.2 句子改写方向

差：他非常难过，觉得人生没有希望。
好：他说完那句话后，把杯子转了半圈。水已经凉了，他没有喝。
差：这个行业正在经历巨大的时代变化。
好：过去靠熟客撑起来的小店，现在要同时面对更贵的房租、更少的路人和更低的外卖利润。

<a id="toc-41"></a>

### 10.3 金句使用原则

纪实文章可以有金句，但金句必须从人物处境里长出来。
好金句应做到：

- 贴着人物，不漂浮。
- 有判断，但不替人物喊口号。
- 能把个体经历压缩成更大的理解。
  示例：
- 很多人的离开，不是因为不热爱，而是终于算清了继续留下的代价。
- 一个行业的转身，最先落到普通人身上时，常常不是新闻，而是一张贴不出去的招聘启事。

***

<a id="toc-42"></a>

## 11. 结尾写法：回到人物，用余味收束

纪实结尾不需要喊口号，也不需要替读者总结“我们都应该怎样”。

<a id="toc-43"></a>

### 11.1 四种有效结尾

<a id="toc-44"></a>

#### 场景回扣

回到开头的场景，但让读者看到变化。

- 开头写关门，结尾写钥匙交出去后，门口的灯还亮了一会儿。

<a id="toc-45"></a>

#### 物件收束

用物件承载时间和命运。

- 那本账本被他放进纸箱最底下，没有扔。

<a id="toc-46"></a>

#### 动作留白

用人物最后一个动作结束。

- 她把工牌放回抽屉，关上灯，第二天没有再去那栋楼。

<a id="toc-47"></a>

#### 克制判断

用一句不夸张的判断点明主题。

- 这不是一个人不够努力的故事，而是一个普通人重新计算生活代价的过程。

<a id="toc-48"></a>

### 11.2 结尾避雷

不要写：

- “愿每一个努力的人都被世界温柔以待。”
- “这就是时代洪流下普通人的命运。”
- “我们都应该珍惜当下，勇敢前行。”
  这些结尾太满、太泛，会削弱纪实文章的真实感。

***

<a id="toc-49"></a>

## 12. 不同任务的处理方式

<a id="toc-50"></a>

### 12.1 直接创作

先明确：人物是谁、核心处境是什么、素材是否足够。输出时优先写完整成稿，必须包含标题、场景开头、人物冲突、背景穿插、转折、结尾余味。
如果素材不足，要降低确定性，不要虚构场景和对话。

<a id="toc-51"></a>

### 12.2 根据访谈整理

先做三件事：

1. 提取时间线：人物经历按顺序排清。
2. 提取冲突线：最重要的困境、选择、转折是什么。
3. 提取细节线：哪些原话、动作、物件、场景可以直接使用。
   成文时不要照访谈顺序平铺，要按故事张力重组。

<a id="toc-52"></a>

### 12.3 润色/改写

重点优化：开头是否有场景、人物是否具体、冲突是否清楚、背景是否自然、结尾是否有余味。
不得新增未经确认的细节、对话、心理活动和事实。

<a id="toc-53"></a>

### 12.4 扩写

扩写不是加抒情，而是补充：关键场景、人物选择、背景解释、多方视角、转折代价。
没有材料支撑时，只能提出“可补充方向”，不能直接编进正文。

<a id="toc-54"></a>

### 12.5 压缩

<a id="toc-55"></a>

## 保留人物、核心冲突、关键细节、转折和结尾余味。删除重复背景、空泛抒情、无功能细节和生硬升华。

<a id="toc-56"></a>

## 13. 输出前自检清单

<a id="toc-57"></a>

### 13.1 事实与边界

- 是否虚构了对话、动作、心理、现场或细节？
- 是否把单方说法写成确定事实？
- 是否处理了隐私、匿名、授权和敏感信息？
- 是否区分了材料事实、作者判断和背景解释？

<a id="toc-58"></a>

### 13.2 人物与叙事

- 人物是否具体，而不是标签？
- 是否有清楚的处境张力？
- 是否避免把人物写成英雄、受害者或观点工具？
- 是否有冲突、选择、代价和变化？

<a id="toc-59"></a>

### 13.3 结构与深度

- 开头是否进入场景，而不是概括介绍？
- 正文是否围绕冲突递进，而不是流水账？
- 背景是否嵌入故事，而不是突然变成报告？
- 个体故事是否抵达更大的处境，但没有生硬拔高？

<a id="toc-60"></a>

### 13.4 细节与语言

- 每个细节是否有功能？
- 情绪是否来自场景，而不是作者强行煽情？
- 语言是否具体、克制、有画面？
- 结尾是否有余味，而不是口号？

***

<a id="toc-61"></a>

## 14. 快速执行模板

<a id="toc-62"></a>

### 14.1 选题模板

- 人物：谁？
- 处境：正在面对什么？
- 张力：为什么难？
- 变化：发生了什么转折？
- 背景：背后连接什么更大问题？
- 标题方向：人物 + 场景/选择 + 命运感。

<a id="toc-63"></a>

### 14.2 开头模板

`一个具体瞬间 → 人物动作/原话 → 场景细节 → 暗示困境 → 简短转入正文`

<a id="toc-64"></a>

### 14.3 正文模板

`场景开头 → 人物当下 → 问题出现 → 冲突加深 → 背景解释 → 关键选择 → 结果余波 → 回到人物 → 余味结尾`

<a id="toc-65"></a>

### 14.4 细节模板

`可观察细节 → 轻微解释 → 留一点空白`

<a id="toc-66"></a>

### 14.5 结尾模板

`回到开头场景/物件/动作 → 点出变化 → 留下克制判断或余味`
