# 分镜图与封面图

默认采用“封面图优先 + 关键分镜图生成”的稳定方案。用户明确要求“不要图片”时不生成实际图片，但仍保留封面方案、视觉参考区说明、图片用途、可执行提示词、完整分镜表和文字执行信息。

优先级：
P0：封面方案必须提供 1-3 个；图片工具可用时，封面图必须优先生成。若封面图生成失败，封面区必须保留 `待生成 + 封面图提示词`。
P1：关键分镜图，默认目标 3 张，最多 5 张；图片工具失败时允许少于 3 张，但必须为缺失图片写 `待生成 + 图片生成提示词`。
P2：全量分镜图，仅当用户明确要求“每镜都生成图 / 全量分镜图”时执行。

图片状态至少记录：类型（封面/关键分镜）、对应镜号或方案名、状态（success/pending）、图片或 prompt。

## 视觉内容选项

| 场景 | 输出 |
| --- | --- |
| 默认丰富内容且图片工具可用 | 先生成 1-3 张封面图，再选取关键分镜，目标生成 3 张关键分镜图，最多 5 张 |
| 图片工具失败或部分失败 | 对失败对象原位写 `待生成 + 图片生成提示词` |
| 用户明确不要图片 | 不生成实际图片，但仍保留封面方案、视觉参考区说明、图片用途、可执行提示词、完整分镜表和文字执行信息 |


## 分镜图规则

视觉参考区（关键分镜图）只做视觉参考，不承载脚本文字。

硬规则：
- 全文只允许 1 个「视觉参考区」。
- 默认目标 3 张关键分镜图，最多 5 张；失败时可少于 3 张，但必须补 prompt 占位。
- 每张图对应一个关键镜头：开场钩子 / 核心动作或产品展示 / 情绪高潮或结果收束。
- 分镜图必须并排展示，默认 3 列网格，禁止一张图独占一整行。
- 每张图只允许 1 组说明，避免图片 caption 与下方文字重复。
- 图片失败时写 `待生成 + 图片生成提示词`。

单元格格式：
`镜XX · 景别 · 机位`
[图片]
`8-16 字动作 caption`

完整逐镜头信息以 7 列分镜表为准。

## 关键分镜默认选择：
1. 开场钩子镜头。
2. 核心动作 / 产品展示镜头。
3. 情绪高潮 / 结果收束镜头。

### 产品种草：
- 痛点镜头
- 产品使用/展示镜头
- 结果或氛围镜头

### 剧情类：
- 冲突镜头
- 转折镜头
- 情绪落点镜头

### 探店/美食/场景类：
- 环境建立镜头
- 主体特写镜头
- 人物体验/结果镜头

## 封面方案与封面图

- 默认提供 1-3 个封面方案。
- 图片工具可用时，优先生成封面图。
- 封面图生成失败时，必须在封面方案区写 `待生成 + 封面图提示词`。
- 封面图放在独立封面区。
- 封面图生成成功后必须插入封面方案区对应方案，不要追加到文档末尾。
- 除非用户明确要求，不要把标题文字画进封面图。
- 标题作为外部文案提供。
- 不编造事实、承诺、价格、排名或百分比。

比例参考：

- 抖音/快手/视频号：通常 9:16。
- 小红书：通常 3:4 或 4:5。
- B站：通常 16:9。

## 视觉质检

- 已生成图片已回填原位；文档末尾没有重复的图片预览区。
- 未成功生成的封面图或关键分镜图已在原位写 `待生成 + 图片生成提示词`。
- 没有把未生成的图片说成已生成。
- 图片提示词中没有要求画面文字，除非用户明确要求。
- 人物/产品一致性已处理。
- 视觉参考区（关键分镜图）和封面图区分开。
- caption 没有重复。
