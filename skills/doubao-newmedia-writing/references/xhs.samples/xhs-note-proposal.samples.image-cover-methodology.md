# 文生图创作工作流

> 定位：文本是笔记的根基与灵魂，图片从用户指令、标题与正文里长出来。本文件只保留「从文本到整组图片」的主工作流：识别输入、判断笔记的属性、制定出图方案、选择素材来源、生成与自检、整理输出。具体视觉基准、设计原子、封面设计、爆款封面方法论、整组内页设计、画面风格方法与 prompt 转换见 `xhs-note-proposal.samples.image-design-methods.md`。

## 目录

- 一、输入基准：先读用户指令、标题、正文与事实素材。
- 二、商业 / 非商业判断：决定视觉基调、素材约束与信息容错度。
- 三、出图方案：决定出几张、每张图承担什么任务、调用哪些设计方法。
- 四、素材来源：用户素材、对标视觉、免版权实拍、AI 生成的优先级。
- 五、生成执行：检索、prompt、并行分批、生成前后自检闭环。
- 六、输出与交付检查：排序、尺寸、清晰度、无水印乱码、合规。

---

## 一、输入基准

出图前先把四类输入对齐，避免图片脱离文本乱长：

1. **用户指令**：主题、品类、品牌/产品/地点、目标人群、指定风格、指定图片数量、是否真人出镜、是否只要封面或整组图。
2. **文本标题**：提炼封面主钩子；封面可以换成更短、更适合画面的表达，但承诺必须和标题一致。
3. **正文内容**：拆出配图分镜。每张图只承接一个正文要点，避免一张图塞太多信息。
4. **事实素材**：产品参数、价格、地点、店铺、品牌视觉、用户照片、官方素材、截图证据。可核验信息先核实，图片不得凭空造证据。

信息缺口处理：用户没指定图片数量、风格、是否真人出镜时，按同品类高赞视觉和正文需求合理默认；只有缺口会直接影响真实性或交付安全时，再追问一个关键问题。

---

## 二、商业 / 非商业判断

先判断图片属性，再决定视觉基调和素材边界。

### 2.1 商业图

有明确品牌、产品、门店或服务在卖时，按商业图处理。商业图服从品牌真实视觉体系，不让 AI 自由发挥。

- **先摸品牌调性**：检索品牌/产品官方视觉，包括主色与辅色、标准字体、logo 规范、包装、官方海报与详情页版式。
- **关键信息零容错**：logo、品牌名、产品名、型号、卖点、价格、门店名、地址等必须与事实一致；拿不准就核实或回退到用户素材/官方素材。
- **真实优先**：产品外观、包装、门店实景、菜单、价格牌、截图证据优先用真实素材，不用 AI 捏造“证据图”。
- **字体和版式随品牌走**：标题与文字气质贴合品牌，不和品牌人设打架。

### 2.2 非商业图

生活分享、美妆护肤、知识科普、情绪表达、攻略复盘、无品牌诉求的内容按非商业图处理。

- **从内容角色定调**：先判断这篇是在给清单、讲步骤、做对比、晒结果、讲故事、表达态度还是营造氛围；再从正文里提炼读者情绪、信息密度、可信度要求和点击钩子。品类只作为检索入口，不直接套固定滤镜或固定气质；最终视觉调性以同赛道近期高赞图的真实共性为准。
- **仍需对标**：没有品牌约束，更要优先检索同品类高赞视觉，提炼字体、配色、构图、版式共性，保证像这个赛道正在火的那批，而不是一眼 AI。

## 三、出图方案

### 3.1 判断出图范围

- **整组图**：本技能默认出整组图。完整方案、连图带文、封面/配图需求，都按正文要点规划整组配图。
- **只要封面**：用户明确只要封面时，图片部分只做第一张，把标题钩子视觉化；仍需按图文排版规范保留必要版位。
- **只要配图/素材图**：用户明确不要封面时，图片部分按正文要点输出内页图，但仍保持统一视觉系统；仍需按图文排版规范保留必要版位。

默认输出 **4-6 张**（含封面）。用户指定数量则从其要求。

### 3.2 判断每张图出什么

- **第 1 张封面**：承接标题钩子，只讲一个核心主题，让用户 1 秒判断“这篇和我有关”。
- **第 2-3 张关键内页**：承接正文最强价值点、核心证据或最容易收藏的步骤/清单。
- **后续内页**：按正文逻辑继续拆解，做到一图一要点，可用清单、流程、对比、场景、证据、总结等形式。

先写一版分镜清单：`序号 -> 图片任务 -> 主文案/主体 -> 素材来源 -> 设计方法`。分镜清楚后再找图或生成，避免出一张改一张。

### 3.3 调用设计方法

- 判断封面是否合格、拆解同赛道高赞图：读取 `xhs-note-proposal.samples.image-design-methods.md` 的「视觉基准」。
- 选择封面版式、写封面文案、安排内页配图：读取「封面设计」「爆款封面方法论」「整组内页设计」。
- 降低 AI 味、设定出图红线：读取「视觉质检底线」；选择字体/配色/真实摄影/插画/版式方法时读取「按画面风格选择专业方法」。
- 写文生图 prompt：读取「Prompt 转换」。

## 四、素材来源

整组图遵循：**用户真实素材 > 官方/授权素材 > 对标视觉 + 可商用免版权实拍 > AI 生成**。

### 4.1 用户真实素材

用户提供产品实拍、探店现场、效果前后对比、场景图时优先采用。真图不等于直接发，需先统一加工再编排版式：

- 统一裁成 3:4（1080x1440），主体居中，保留安全边距，不变形拉伸。
- 横图二次构图裁成竖图，或上下补色块成竖版，避免整组比例混乱。
- 统一色温、明暗、饱和、滤镜，让封面和内页像同一组作品。
- 适度提亮、增清晰、压噪点、拉对比，去掉杂物、路人、反光杂讯；保留真实质感，不过度磨皮。
- 涉及车牌、证件、真实姓名、手机号等隐私时打码或化名。

### 4.2 官方/授权素材

商业图涉及品牌、产品、门店、服务时，优先使用官方图、用户授权素材或可核验截图。不得伪造 logo、包装、价格牌、菜单、证书、检测报告、销量截图等证据。

### 4.3 对标视觉与免版权实拍

用户没有素材时，先做两类检索：

- **对标视觉**：搜同品类、同选题、同类型的高赞/高流量笔记，提炼字体气质、关键词高亮方式、配色、构图、版式骨架、是否用真人/实拍/截图。
- **可商用实拍**：在 Unsplash、Pexels、Pixabay 等可商用/免版权图库按品类关键词找高清实拍。只用明确可商用/免版权的图，不搬运他人笔记原图、网图、带水印或来源不明的图，并留存来源链接。

跨平台找优质设计参照时，可扩展到小红书、Pinterest、站酷、Behance、Dribbble 等，用“品类 + 封面/模板/排版/摄影/配色”组合检索。

### 4.4 AI 生成兜底

前几档都拿不到合适素材，再用 AI 文生图。AI 生成必须基于真实爆款素材对标视觉，不凭空发挥。

## 五、生成执行

### 5.1 检索与分镜并行

能并行就并行：品牌/事实核验、同品类视觉对标、免版权素材检索可同时进行。文本定稿后，先完成整组分镜，再按素材来源逐张执行。

### 5.2 Prompt 生成

写 prompt 前先确定：图片任务、商业/非商业属性、封面类型、设计原子、主体、构图、配色、字体、指定文案、素材来源、比例与安全边距。

- 基于对标，不自行发挥：把对标图的具体特征写进 prompt，如米白纸感背景、超粗黑体标题、亮黄马克笔高亮关键词。
- 每条 prompt 必带去 AI 味要求：调用 `xhs-note-proposal.samples.image-design-methods.md` 的「视觉质检底线」。
- 图上中文尽量短，只呈现指定文字，不额外添字。长文案、品牌名、价格、参数、型号等零容错信息优先后期版式排字或使用真实素材。
- 版式文字型封面可自行生成，但备忘录体/聊天体必须照真实 iOS 备忘录或真实微信界面复刻，不让 AI 即兴画假 UI。
- 同组图重复关键外观与风格描述，保证同一配色、同一版式语言、同一字体气质。

封面类型到 prompt 的转换顺序、九要素填槽框架与各类 prompt 模板见 `xhs-note-proposal.samples.image-design-methods.md`。

### 5.3 生成前后自检闭环

**生成前先把风险写进 prompt 或版式策略：**

- 明确画面只出现指定文案；禁止额外生成尺寸说明、平台字样、作者名、logo、水印、用户名、二维码、引用出处、模型签名、乱码英文、假 UI 文本。
- 不把“小红书 3:4 / 1080x1440 / cover / poster”等规格信息写成画面文案；这些只作为技术约束，不允许出现在图上。
- 需要精准中文时，优先短标题 + 后期版式排字；长文案、品牌名、价格、参数、型号等零容错信息不要交给模型自由生成。
- UI 类画面优先用真截图或可控模板；无法保证真实界面时，改成中性信息图，不生成假微信、假备忘录、假系统栏。

**生成后逐张检查，处理：**

- 配色是蓝紫塑料渐变、全图高饱和、廉价霓虹？改低饱和浅底 + 单点亮色后重出。
- 文字糊、乱码、假英文、错别字、缺笔、多笔、挤压变形？文案另排或重写 prompt；无法逐字确认就不交付。
- 画面上方出现“小红书3:4”“1080x1440”“竖版封面”等规格文字，或底部出现 logo 乱码、模型签名、作者水印、用户名、二维码、引用出处？先裁切/修图/重排；只要影响主体或无法干净去除，立即重出。
- 人手、人脸、产品、文字、logo 畸变、多指、扭曲？换图或重生成，畸变图绝不出。
- 画面过度完美、悬浮无影、塑料反光、磨皮过头？加“真实摄影、自然光、真实景深、轻微生活痕迹”后重出。
- UI（备忘录/聊天/软件界面）是 AI 画的假界面？换真截图或模板。
- 成图低清、噪点、压缩色块、放大糊？重出高清，绝不用小图硬撑。


### 5.4 并行与分批

一次最多并行生成 4 张，超过分批；同组图尽量同批生成以保持风格统一。整组图必须 100% 落地：找不到实拍、联网失败、某一档拿不到图，都不能跳过，向后兜底到“照参照风格文生图”补齐。

## 六、输出与交付检查

- 排序是否正确：第 1 张为封面/主题图，其余按正文逻辑排列，一图一要点。
- 是否符合 3:4（1080x1440），整组是否同字体气质、同配色、同版式语言。
- 是否先找了对标与免版权实拍，AI 生成是否照着参照风格，而非凭空发挥。
- 是否过了生成前后自检闭环：无塑料蓝紫渐变、无全图高饱和、无塑料反光、无悬浮无影、无过度磨皮、无畸变、无假 UI。
- 图上中文是否零差错：用词准确、字形正确、无错别字、缺笔多笔、乱码，且只呈现指定文案。
- 是否无无关杂讯：水印、平台 logo、用户名、二维码、引用出处、假 UI 文字、尺寸标注、平台规格字、模型签名等一律不得出现。
- 主体是否突出，安全边距是否足够，脸、关键文字、产品细节是否被裁切。
- 含真实截图的页是否清晰可读、不拉伸变形。
- 图片是否无侵权盗用、无虚假对比/伪造数据、无擦边敏感元素。
