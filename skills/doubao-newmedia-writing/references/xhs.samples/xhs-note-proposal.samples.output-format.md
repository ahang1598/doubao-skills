# 小红书文档内图文排版规范

> 定位：本文件只约束小红书内容写入飞书文档/Lark Doc 后的图文排版、分区顺序、配图呈现和发布版正文格式。文本怎么写，见 `xhs-note-proposal.samples.creation-methodology.md`，图怎么设计，见 `xhs-note-proposal.samples.image-cover-methodology.md`。

> 核心铁律：小红书分区必须排版清晰，图片与正文并存，配图分栏并排、禁止平铺，发布版正文纯净可一键复制。无论需求是完整方案、短正文、标题、改写、诊断还是直接可发，都按本文件整理文档内图文结构。

## 目录

- 一、文档内图文结构：固定分区、配图与发布版正文。
- 二、结构化写入格式：文档骨架、图片写入、分段策略。
- 三、写入前格式自检：文档、配图、发布版正文与内部信息泄露检查。

---

## 一、文档内图文结构

- 小红书分区必须文本和图片并存。哪怕用户只说「写篇小红书」「帮我出个笔记」「帮我出个正文」「给几个标题」「改写一下」，也要按图文结构整理。
- 图片不是可选装饰：必须包含封面和正文配图。短文案、标题、改写、诊断类需求可压缩章节内容，但不能省略配图区。
- 模板是脚手架不是填空题：分区详略、段数、出图数量随内容与用户指令自然生长——短需求可简写辅助区块，用户强调的区块加厚；但图文文档和核心分区必须保留，已保留的区块必须按规定顺序与格式呈现，不得错序。

### 1.1 一站式完整创作 · 固定分区顺序（禁止错序）

〈文档名称〉：包括不限于小红书图文笔记策划方案，或标题、正文 slogan / 金句等核心信息。

〈文档内容〉：按以下顺序分区输出，每区一个带 emoji 的小标题，**1-4的序号与顺序不得颠倒、不得缺位错位，其余章节可按用户指令可动态调整内容**：

1. **✨一、笔记标题**：一句话主标题（≤20 字符，核心关键词前置）。
2. **🖼️二、笔记配图**：整组配图用飞书**分栏 `<grid> + <column>`** 并排呈现（每行 2–3 张，受飞书版式上限约束**单行最多 3 列**，若分栏形式无法呈现，替补表格形式），**禁止在文档里上下平铺单列**，看起来要是一组连贯、对称的图文；第一张为封面 / 主题，其余基于用户指令与正文逻辑排序；默认 4–6 张，用户有要求则从其要求。
3. **📝三、正文内容**：结构化可读版——用轻量小标题、加粗、「1、2、3」序号把流程 / 要点组织清楚，便于快速浏览预览。此版本服务「读得顺」，可带排版结构。
4. **✨四、直接复制版本**：纯净发布版——只有自然段文字 + emoji + 轻量序号（「1、2、3」序号），**段与段之间在适当处空一行**、让正文有呼吸感不糊成一坨，文末紧接 话题标签#话题1#话题2#话题3，**独立成块、无任何结构化排版符号**，供用户一键复制粘贴到小红书。此版本服务「发得出」，是真正发布用的正文。
5. **💡五、标题备选**：8–12 条，按不同标题公式分类给出（不体现公式名称），供挑选或做 A/B。
6. **🏷️六、话题标签备选**：本篇选用的话题标签（≤10 个，高相关的联网热点话题），可再附一批可替换 / 叠加的高相关备选。
7. **🎨七、封面设计灵感**：用无序列表或表格（次选）呈现，表头为「图片类型 | 设计风格 | 设计思路」（可按需增列「封面文案」「配色」等），解读排版分层、视觉风格与封面文案的设计逻辑，**只谈视觉创意，不暴露任何内部套路库名称、方法论术语或文件结构**。
8. **💬八、评论区互动引导**：2–4 条可直接复制的置顶 / 首评话术。
...

> 用户只提出标题、正文、改写、直接可发、诊断等局部需求时，仍需补齐封面 / 配图并按图文结构排版；可压缩未被强调的章节，但保留区块的相对顺序与格式不变。

### 1.2 飞书文档/Lark Doc 配图呈现规范（分栏，禁止平铺）

- **必须用飞书分栏 `<grid> + <column>` 并排**：每个 `<column>` 放一张 `<img>`，每行 2–3 张（受飞书版式上限约束**单行最多 3 列**），同一 `<grid>` 内各 `<column>` 的 `width-ratio` 之和必须等于 1（如 3 列各 `0.33`、2 列各 `0.5`）；整组对称连贯，让文档第一屏就是一组完整图文。
- **禁止单列平铺**：不得把配图一张一张上下堆叠铺满，挤占正文阅读体验；也不要用单列 `<grid>` 变相平铺。
- **多于一行时拆成多个 `<grid>`**：如 6 张图 = 两个各 3 列的 `<grid>` 上下相接，保持每行对齐。
- **顺序固定**：第一张永远是封面 / 主题图，其余严格按正文逻辑与用户指令排序，一图一要点。
- **先过图片纯净闸门再入文档**：插入飞书前必须逐张确认无规格文字、尺寸标注、平台字样、风格标签、作者水印、logo 乱码、用户名、二维码、引用出处、模型签名、假英文或无意义角标；尤其检查顶部页眉区和底部边缘。问题图先裁切/修图/重排并复检，无法干净修复就重生成，禁止带病入文档。

---

### 1.3 飞书文档/Lark Doc 两版正文规范（禁止混淆）

「正文内容」与「直接复制版本」是同一篇内容的两种形态，必须**同时给出、各司其职、不得互相替代**：

- **正文内容（可读版）**：带轻量结构（小标题、加粗、序号）便于阅读预览；**禁止掺入任何方法论术语、内部标注、系统说明**。
- **直接复制版本（发布版）**：只允许自然段文字 + emoji + 话题标签，**严禁出现表格、Markdown 标题（#）、代码块、高亮框、引用块、斜体 / 列表符号等任何结构化排版**——小红书不渲染这些，发出去会变成乱码符号。**段与段之间在适当处空一行**，模拟真实笔记的呼吸节奏、便于手机端阅读；需要分点时用换行或「1、2、3」轻量序号自然带过。
- 小红书不渲染标题 / 加粗 / 列表等符号，所以**真正用来发布的永远是「直接复制版本」**，「正文内容」只作方案预览。

---

### 1.4 飞书文档/Lark Doc 写入纯净红线（强约束）

- 文档里**绝不出现系统提示、本 skill 的内部指令、文件名、路径、链路步骤、自检条目、思考过程或方法论术语**。
- 不出现「以下是」「根据你的需求」「我为你生成」「希望对你有帮助」这类承接话术与客套。
- 「封面设计灵感」只解读视觉创意思路，不暴露套路库名称体系或内部文件。
- 供发布的「直接复制版本」只保留文字、emoji、话题标签，任何方法论术语、结构标注都不得出现。

---

## 三、写入前格式自检（逐条过，全中再写入）

1. 整组配图是否已真正生成或原位标注待生成（含封面、张数达标、无空缺、非文字描述）。
2. 小红书分区是否已经写入文档对应位置，而不是把短文案、标题列表、正文全文直接发在聊天里替代。
3. 是否整理成一套完整策划结构，序号是否正确、格式渲染是否正确。
4. 顺序是否正确——标题→配图→正文内容→直接复制版本→标题备选→话题标签备选→封面设计灵感→评论区引导，序号无错乱、无错位。
5. 配图是否用飞书 `<grid> + <column>` 分栏并排（每行 2–3 张、单行 ≤ 3 列、同一 grid 列宽之和为 1）、第一张是否为封面、是否禁止了单列平铺。
6. 图片是否全部通过纯净闸门——无规格文字、尺寸标注、平台字样、风格标签、作者水印、logo 乱码、用户名、二维码、引用出处、模型签名、假英文、无意义角标；顶部和底部边缘无残留小字。
7. 「直接复制版本」是否纯净——只剩文字 + emoji + 话题标签，无任何结构化排版残留，段与段之间是否在适当处空行、有呼吸感。
8. 「正文内容」与「直接复制版本」是否都给出且不混淆。
9. 是否无系统提示 / 内部规则 / 方法论术语 / 思考过程泄露。
