# 文本笔记创作工作流

> 用途：完成小红书文本笔记创作。

## 读取策略

- 完整创作、深度改写、诊断优化、账号定位或合规敏感内容：读 `xhs-note-proposal.samples.creation-methods.md` 对应章节。
- 热点、节点、趋势、平台当下热度或实时事实：读 `xhs-note-proposal.samples.hotspot-guide.md`，并只使用可核验信息。
- 涉及封面或配图：读 `xhs-note-proposal.samples.image-cover-methodology.md`；需要具体视觉方法时再读 `xhs-note-proposal.samples.image-design-methods.md`。
- 不读取博主高赞笔记原文、样本库或同类笔记正文来仿写；文本质量依靠用户素材、事实核查、通用结构、品类经验和原创表达完成。

## 文本主链路

1. 定人群：明确这篇写给谁。
2. 定价值：至少命中帮选择、帮避坑、帮提升、帮共鸣、帮发现、帮表达中的一种。
3. 整素材：区分已知事实、用户体验、产品/地点信息、可核验证据、不可写或需降级的信息。
4. 定切口：只选一个主钩子，别同时堆满价格、颜值、情绪、参数和故事。
5. 定标题：标题 <=20 字，核心关键词前置，承诺必须能被正文兑现。
6. 定正文骨架：按内容类型选叙事线，再把素材放入“开头钩子 -> 中段证据/场景 -> 结尾动作”的链路。
7. 定互动与关键词：首段、标签、图片文字自然覆盖核心搜索词；结尾给低成本动作。

## 常用骨架

- 痛点解决型：痛点 -> 痛感加深 -> 解法。
- 种草转化型：我的苦恼 -> 我的问题 -> 我试的解法 -> 真实体验与变化 -> 适合谁。
- 干货教程型：为什么值得做 -> 是什么 -> 怎么做。
- 测评对比型：先立标准 -> 横向/纵向对比 -> 适合/不适合。
- 故事种草型：日常情境 -> 冲突 -> 转折 -> 变化。
- 合集清单型：收藏承诺 -> 分类逐条 -> 取舍建议。

## 快速规则

- 开头 3 行接住标题承诺，优先给价值、冲突、结果、痛点、身份或求助气口。
- 中段用真实细节、对比、步骤、场景、踩坑、价格、适合/不适合留人。
- 结尾给收藏、评论、搜索、选择建议或后续预告。
- 直接复制版本只保留自然段、emoji、轻量序号和话题标签。
- 不仿写、不改写、不复用博主高赞笔记原文；如果素材不足以支撑亲历口吻，就写成清单、建议、测评框架、求助帖或明确标注的信息整理。
