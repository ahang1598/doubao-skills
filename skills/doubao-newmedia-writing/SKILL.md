# 豆包新媒体写作

目标：根据用户指令或提供的素材，先确认创作类型，再交付为一份真实可打开、可继续编辑、可发布前检查的飞书文档/Lark Doc。明确命中创作类型后，飞书文档/Lark Doc 创建是第一执行动作和唯一交付通道；每个创作类型的文档须包含文本内容和图片内容。

## 路由边界

- 小红书图文笔记：用于小红书优质笔记、爆款笔记、图文笔记、种草文、探店攻略、好物测评、账号定位、热点借势、封面/配图文生图、可直接发布的中文文案与策划方案生成。
- 公众号文章：用于生成、润色、改写、扩写、压缩、策划和评估适合微信公众号发布的非商业类文章，覆盖新闻热点类、专业解读类、故事纪实类、认知观点类、情感治愈类五大写作需求。
- 短视频分镜脚本：仅当用户明确需要 3 分钟以内的短视频分镜、拍摄脚本、逐镜头脚本、storyboard、镜头表、视频脚本表格，或需要把选题、口播稿、文案、产品信息、营销卖点、活动信息、已有脚本改成可拍摄的短视频分镜时使用。
- 复合型创作方案：当用户同时要求上述 2 个或 3 个创作类型时使用，例如“小红书图文笔记 + 短视频分镜脚本”“小红书 + 公众号 + 短视频脚本”。

## DO NOT USE WHEN

出现以下技能不适用/需求不明时，不创建飞书文档/Lark Doc，用 1-2 句话承接并追问用户明确需求，并简要阐明本技能的核心功能（小红书图文笔记、微信公众号文章、3 分钟以内短视频分镜脚本），引导用户使用本技能进行创作。

- 技能不适用：用户只要标题、选题、开头、正文、文案、标题党、爆款标题、评论区话术、口播稿等内容，但未出现“命中判定”中的平台名或通俗类型名。
- 需求不明：用户只说“帮我写”“做个内容”“做个爆款”等模糊需求，无法判断平台或成品形态。

禁止强制编造平台或成品形态。

## 总工作流

1. 解析用户指令或提供的素材，按“命中判定”确认是否明确命中单类型或复合型创作；命中后提取主题素材、受众、发布目的、篇幅或时长、图片或分镜需求、事实核查点。
2. 读取 `references/genre-guide/lark-doc.writing-guide.md`，立即用 `lark-cli docs +create` 创建一份飞书文档/Lark Doc 最小占位，并确认拿到 `document.url` 或 `document_id`。
3. 按创作类型读取主干工作流，并直接进入对应生成答案：
 - 小红书图文笔记读取 `references/genre-guide/xhs.writing-guide.md`；
 - 公众号文章读取 `references/genre-guide/wechat.writing-guide.md`；
 - 短视频分镜脚本读取 `references/genre-guide/short-video.writing-guide.md`；
 - 复合型创作先读取 `references/genre-guide/composite.writing-guide.md`，再按命中的类型逐一读取对应主干。
4. 由命中的主干继续读取对应 samples，完成文本创作、格式排版、图片创作等，并持续写入第 2 步已创建的文档内。
5. 静默自检事实、文档结构、排版、图片状态、路由一致性和文档入口；只交付文档链接和必要说明。

## 自检规则

- 是否命中 `DO NOT USE WHEN`；命中时是否先追问/引导而不是直接生成随机答案或创建文档。
- 是否把“爆火标题/爆款标题/热门文案”等泛化表达误判为小红书、公众号或短视频分镜脚本。
- 是否在明确命中单类型或复合型创作后，第一执行动作就是创建飞书文档/Lark Doc，并拿到 `document.url` 或 `document_id`；若失败，是否停止下游创作并按 `lark-doc.writing-guide.md` 排障重试。
- 是否按用户选择的创作类型路由：单类型只读取命中主干；复合型先读取 `composite.writing-guide.md`，再逐一读取命中的类型主干。
- 是否由类型主干读取 samples，而不是在总入口直接展开具体方法。
- 是否按命中平台的文档内排版格式和视觉要求写回文档。
- 复合型创作是否保持各类型独立分区和差异化排版，避免把小红书正文、公众号文章和短视频分镜表混排成随机结构。
- 复合型创作是否先写 `方案总览` 外层骨架，再进入各类型分区；各类型格式不得覆盖复合型顶层结构。
- 是否把文本、图片、表格、图片占位和验真信息等写入文档对应位置；任一类型都不得只有文本、没有图片区。
- 是否核查或降级处理了不确定事实。
