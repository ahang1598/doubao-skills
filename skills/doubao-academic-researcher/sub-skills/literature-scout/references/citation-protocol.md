# 引用核验协议

## 核验流程

对每条文献按两步核验：

**第一步：存在性核验**
用 `scholar_search` 搜论文完整标题（或关键片段 + 第一作者姓氏）。
- 搜到、标题作者匹配 → 存在性通过
- 搜到但标题/作者有出入 → 标记，核实是否同一篇
- 搜不到 → 判为 UNVERIFIABLE，不进综述（见下方"灰区 = 不使用"）。不要标"[未经核验]"后继续使用

**第二步：引述准确性（陈述级）**
正文中引述的发现是否和论文标题/摘要一致。核验作用域是陈述级——每一条归因于文献 [N] 的实质陈述（不限于显式引述句）都必须能在 [N] 自身的标题/摘要中找到支撑；找不到支撑的降档措辞或删除，禁止把多篇文献合成的判断挂到单篇名下。只有元数据可用时，不编造方法细节或具体数字。

## 5 级判定

| 判定 | 含义 | 允许使用 |
|---|---|---|
| VERIFIED | 存在，引述一致 | ✓ |
| MINOR | 存在，措辞微偏差 | ✓ 修正后 |
| MAJOR | 存在，引述不符 | ✗ 修正或删 |
| UNVERIFIABLE | 无法确认存在 | ✗ |
| PAYWALL | 存在但元数据不足 | △ 限于元数据层 |

## 字段级核验（正交于条目级判定）

上面的 5 级判定作用于**条目**——这篇论文能不能用。会被打印的**题录字段**（发表年份、卷、期、页码、DOI、期刊等级、被引数）另有一层字段级核验，由 `sub-skills/citation-guard/SKILL.md` 执行，三态标注：

| 状态 | 含义 |
|---|---|
| 已核验 | 来自 Crossref API 或出版商/期刊官方页面，附来源 |
| 二手核验 | 来自非官方网页（聚合站、图书馆页、机构新闻），附 URL |
| 未核验 | 各通道均未命中；字段值不得打印，如实写"未核验" |

题录字段的值只允许从本次会话的工具返回中抄录，不允许从记忆生成、不允许按格式规律构造。**存在性核验通过不代表题录字段可信**——一篇 VERIFIED 的论文照样可能被配上编造的 DOI 和页码，两层核验相互独立。

## 灰区 = 不使用

无法确认的引用不进综述。"好像有一篇论文做过 X"但搜不到 → 不引用，用 `[待确认：该方向未检索到相关工作]` 占位。

**统一占位规范**：正文中所有"无法核实/待补"的内容一律用 `[待确认：...]` 前缀（如 `[待确认：该方向未检索到相关工作]`、`[待确认：背景数据需核实]`），不要用 `[未经核验]` 等其他变体。literature-scout 输出里单独列待核验文献的 `### 待确认` 小节是同一家族的分区呈现。

## 编造引用的零容忍

编造引用是红线。检测信号：
- 标题太完美匹配你想引用的观点
- 作者+标题+年份+期刊组合完全搜不到
- 同一作者在列表中出现过多次但你只搜到过一次

发现编造 → 立刻删除 + 检查依赖该引用的所有判断是否还站得住。
