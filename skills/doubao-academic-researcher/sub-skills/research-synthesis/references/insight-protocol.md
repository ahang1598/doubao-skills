# 洞见协议（insight-protocol）

**目的**：把综合从"覆盖式汇总"推到"有判断的分析"。本协议只作用于**综合与写作环节**——综述正文、检索报告的小结/共识/争议章节——不触碰检索与题录核验，两种任务模式通用，零核验成本。

**洞见的操作定义**：读者读完全部入选文献的摘要也得不到的、且有证据链支撑的结论。既不是复述（那是摘要的工作），也不是无根猜测（那是把幻觉从题录层搬到结论层）。

## 结论阶梯（L0–L4）

| 级 | 名称 | 形态 | 示例 |
|---|---|---|---|
| L0 | 复述 | 转述单篇发现 | "A(2020) 发现老龄化降低家庭碳排放" |
| L1 | 聚合 | 跨篇分布/共同模式 | "8 篇微观研究中 6 篇发现负效应" |
| L2 | 裁决 | 对分歧给出**带证据权重的有条件立场**，或明确判定证据缺口 | "微观层面负效应证据高度一致（5/6 篇）；宏观正效应集中出现在生产侧核算口径的研究中——分歧主要由口径驱动，而非真实矛盾" |
| L3 | 机制 | 解释模式/分歧**为什么**存在，且解释可被入选文献的属性表检验 | "口径假说可检验：入选文献中改用消费侧口径的 2 篇宏观研究，效应方向即与微观研究一致" |
| L4 | 生成 | 可检验假设 / 研究设计建议 / 决策含义 / 预测 | "若口径假说成立，用 MRIO 消费侧 + 省级面板重估应得到负效应——现有入选文献无人做过这一组合，这是最高杠杆的下一步" |

**楼层规定（硬性）**：
- **结论位的每一条 ≥ L2**。结论位按交付物类型映射——**综述**：摘要中的核心发现句、各证据章节（分类分支、综合讨论）的收尾句、结论章节对每个 RQ 的回答；**检索报告**：执行摘要与"关键发现 / 研究共识 / 争议"章节的每一条。引言、研究方法、分类框架等程序性章节不适用收尾句楼层——不要为凑楼层往程序性章节里塞裁决句。
- 全文 ≥ 3 条 L3、≥ 1 条 L4。**规格化的证据缺口（写清缺少什么设计/什么数据的研究、它能裁决哪个问题）计入 L3/L4 楼层**——当入选证据不足以支撑机制解释时，楼层的合法出口是把缺口写精确，不是把推断写过头（后者等于把幻觉从题录层搬到结论层，判本协议失败）。
- L0/L1 只能作为证据铺垫出现，**不得出现在任何结论位**。

## 深问题（写作前执行，组织轴）

基于经核验的文献清单生成 5–8 个深问题，三判据同时满足：① 任何单篇摘要都无法回答；② 该领域专家会真正关心；③ 答案会改变后续研究设计或实践决策。

生成法（视角摩擦）：让三个视角各自出题后取并集——**怀疑者**（这些结论哪里站不住？哪个共识可能是方法性假象？）、**方法论者**（结果差异有多少是方法/口径/数据造成的？）、**实践者/设计者**（这对我下一步的决策或研究设计有什么用？）。

"讨论/关键发现"部分以**回答深问题**为组织轴（taxonomy 继续负责覆盖检查，两者不冲突）；每个深问题的回答 ≥ L2。回答不了的深问题不许删除——转化为 L3/L4 素材（证据缺口本身就是发现）。

## 分歧裁决程序（矛盾是洞见的富矿，不是尴尬）

发现矛盾后必须走完程序，禁止以"结论不一，见仁见智"收尾：

1. **列候选调节变量**：数据层级（宏观/微观）、测算口径（生产侧/消费侧、直接/间接）、识别策略、时期、地区、样本、指标定义；
2. **建证据表**：每篇 ×（发现方向，各调节变量取值）——在交叉对比表上扩列即可；
3. **三种合法结局，三选一**：
   - **有条件立场**：分歧被某调节变量解释 → 条件化结论 + 表决计数（"微观 5/6 负、宏观正效应全部来自生产侧口径"）；
   - **证据缺口**：现有文献属性不足以区分假说 → 明确写出**缺少什么设计的研究才能裁决**（这是 L3/L4 素材，不是逃生门——不写清缺什么就不算走完程序）；
   - **真实分歧**：同条件下方向仍相反 → 这本身是重要发现，提出最可能的深层解释作为可检验假设。

矛盾越硬，裁决产出的洞见越值钱；跳过硬矛盾是综合环节最大的浪费。

## 数值层

交叉对比表在方向之外提取**幅度与条件**（摘要级精度即可）。模式挖掘三问：幅度是否随层级/口径/时期系统性变化？离群值是谁、其方法有何不同？效应量级在文献间相差几倍、为什么？**数值声明只能来自实际提取值——没有就写没有，不估计**（该纪律与 citation-guard 的只抄不编同源）。

## 洞见反思（成稿后执行；消融证据表明反思环节对新颖性的贡献最大）

草稿完成后逐条写下："**这份报告教给读者的东西里，哪些是读完全部入选摘要也得不到的？**"每条标注 L 级。L3 及以上不足 3 条 → 回到裁决程序与深问题，重写对应章节，**不允许原样交付**。这一清单可直接作为执行摘要（检索报告）或摘要核心发现句（综述）的骨架。

**agent 模式的落盘约定**：环境以文件形式工作时，本清单写入内部工作文件，小节标题**固定写作"【洞见反思清单】"**、逐条带 L 级标注——固定标题兼作回归测试识别锚点（与"【题录核验台账】"同一约定）。该清单是内部工件，**不进交付物**。

## 有根推断纪律（防"推断性链接"幻觉——顶级系统错误分析中的头号残余错误类型）

- 每条 L2+ 结论附推理链：前提文献[编号] → 推断步骤 → 结论，置信语言按 `references/hedge-calibration.md` 的阶梯选取；
- L3/L4 必须显式标注推断性质（"一个可检验的解释是""据此推断"），不得表述为已被文献证实的事实；
- 推断不得与任何入选文献的实际发现冲突；
- 洞见是"证据支持的推断"，不是"新的事实声明"。**把幻觉从题录层赶到结论层，即判本协议失败。**

## 反平庸黑名单（出现即改写，每处替换为具体版本）

- "未来需要更多研究" → 缺少**什么设计/什么数据**的研究、它能裁决**哪个**问题；
- "各有优劣 / 见仁见智" → **什么条件下**谁更优、依据是哪几篇；
- 以"结论不一致"作段落结尾 → 走裁决程序；
- "随着……的发展 / 取得了长足进展"类填充句；
- "具有重要意义"而不说**对谁**、**改变什么**。
