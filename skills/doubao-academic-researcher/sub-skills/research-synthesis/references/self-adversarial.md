# 自对抗审查

综合完成后、呈现之前，做一轮自我质疑。目的不是追求完美，而是抓住"自己写的时候不觉得有问题、别人一看就会问"的盲区。

## 三个回溯问题

### 问题 1：如果重新开始，研究简报会变吗？

回到 Step 0 的研究简报，看现在了解的东西是否会让你换角度。
- 不会变 → CLEAR
- 会变一点但核心不变 → 在局限性提一句
- 会大幅改变 → frame-lock，认真考虑回 Step 0

### 问题 2：有没有未审视的前提假设？

你的综述是否建立在没明说的假设上？例如：
- 默认"更高准确率 = 更好"？
- 默认"英文数据集的表现能代表其他语言"？
- 默认"当前趋势会延续"？

发现 → 在局限性或相关主题章节加限定。

### 问题 3：结论是被证据支撑的，还是被选择性引用构造的？

对每条核心结论：
- 有没有**反对这条 claim 的证据**？有的话呈现了吗？
- 只引了支持的论文，是因为确实没反对的，还是没搜/没引？
- 把 claim 反过来说，反面命题站不站得住？

发现选择性引用 → 补搜反面证据，在争议部分呈现。

## 视角遗漏检查

- **学科**：跨学科话题是否只从一个学科写了？
- **地域**：文献是否过度集中在某些地区的研究组？
- **方法**：是否只关注了某一类方法（如深度学习），忽略了传统方法？
- **批判**：综述是否过度正面？有没有呈现失败、局限和批评？

发现遗漏 → 在争议与开放问题或局限性中补充。

## Concession 规则

自对抗中发现某条结论有问题时：
- 反面证据直接推翻 → 修改 claim，降级为条件判断
- 部分削弱 → 保留 claim，加限定
- 角度不同但不矛盾 → 在争议部分呈现两种视角
- 只是"感觉可能有问题"但找不到反面证据 → 不修改

不要因模糊的不安软化一个有证据支撑的判断。

## 产出

不产出独立报告。产出是对综合结果的修正：修改的 claim、新增限定、补充到争议与局限的内容。三个问题都没发现问题 → 直接进呈现阶段，不硬造自我批评。
