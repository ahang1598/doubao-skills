# 证据综合与审查

你拿到了一份经核验的文献清单，每篇附关键发现摘要。你的任务是把这些碎片综合成一份有判断力的分析：不是逐篇罗列，而是按主题组织、交叉对比、呈现矛盾、给出结论。

这一步是整个调研中最决定产出质量的环节。检索找到了论文，核验确认了真实性，但"把 30 篇论文变成一份专业综述"——这个转化才是真正的工作。

**执行本阶段前，先读取 `sub-skills/research-synthesis/references/insight-protocol.md`**——结论阶梯（L0–L4）、深问题、分歧裁决程序、洞见反思清单与反平庸黑名单都在其中；这一读取是流程步骤，不是可选参考。

## 反朴素管线：先分配证据，再写

朴素做法是"读完所有论文，然后自由写一段综述，最后在合适的地方插入引用"。**不要这样做。** 这种做法的引用往往是事后贴上去的装饰，极易张冠李戴或编造。

正确做法：

1. **先确定主题分组**
2. **把每篇论文的关键发现分配到对应主题**
3. **每个主题拿到一组固定文献**，综合时只从这组里写，不越界引用；引用一律写**引用键** `[@key]`（键即台账里的名字，见主 SKILL.md"编号冻结环节"），全程不写数字编号——数字由交付编译铸造；需要台账外的论文 → 回调 literature-scout 追加入台账（新键），不在综合层直接添加
4. **写完后检查**：正文每条 claim 是否都能追溯到分配给该主题的文献

## 三步法

### 第一步：引文抽取

对每篇论文抽取直接回答研究简报的关键信息：

- **核心发现**：一两句话概括
- **方法特征**：用了什么方法？和其他论文的区别？
- **适用条件**：发现在什么条件下成立？已知局限？
- **与其他论文的关系**：支持、挑战、还是扩展了谁？

不相关的论文在这一步整篇丢弃。

### 第二步：主题聚类 + MECE taxonomy

按**研究问题或方法流派**聚类，不按论文。大纲每个节点标记：

- **呈现方式**：paragraph（多篇论文的细致综合）还是 table（并排对比）
- **分配的文献**：这个节点引用哪几篇

**MECE 原则**：分类轴互斥且完备。空格 = gap = finding。一个好的 taxonomy 不只是分了类，还能通过空格揭示这个领域的盲区。

跨分类的论文（属于多个类别的）要识别出来——它们往往是最有洞见的，正是因为它们挑战了分类轴本身。

### 第二步半：深问题生成（组织轴）

taxonomy 定好后、动笔前，按 insight-protocol 的三视角法（怀疑者 / 方法论者 / 实践者）生成 5–8 个深问题——单篇摘要答不了、专家真正关心、答案会改变设计或决策的问题。“讨论/关键发现”部分以回答深问题为组织轴；答不了的转化为证据缺口素材，不许删。

### 第三步：逐节串行合成

按大纲顺序一节一节写。每节输入 = 研究简报 + 分配的文献 + 已写好的前面几节。

**串行而非并行**。并行写各节会导致内容重复、风格不一、节间无过渡。

**每节开写前，先重打本节台账切片**（内部脚手架，不进交付物）：把分配给本节的文献列一遍——`[@引用键] 第一作者, 年份, 短标题`——写作中引用直接落键，写完本节对照切片确认没引到本节之外。键是名字不是数字，模型写作时本来就在想名字，几乎没有记忆负担；数字编号在整个写作过程中**不存在**，由交付编译铸造（见主 SKILL.md"编号冻结环节"）。

## 每个主题章节的思维骨架

写每个主题章节前，先在脑子里过这条论证链（不打印出来）：

1. **这个主题的 claim 是什么？** 不是"覆盖了哪些工作"，而是"这些工作放在一起告诉我们什么"。 claim 的楼层至少 L2（裁决级，分级见 insight-protocol）——L0 复述与 L1 聚合不配作为一节的 claim。
2. **最强的正面证据？** 哪几篇最直接支持？独立来源的收敛比单一来源更有力。
3. **反面证据或限定条件？** 哪些工作与 claim 矛盾或限制其适用范围？
4. **方法/条件差异怎么解释分歧？** 数据集？样本量？指标？时间窗口？
5. **和其他主题什么关系？** 支持、挑战、还是补充？

**自检**：写完后删掉所有引用，剩下的还是一篇有判断的分析（不是事实罗列），说明论证链是通的。

## 交叉对比写法

**这是综述和网页摘要的核心区别。** 人写的综述 citation 互联密度 0.14，GPT 直出只有 0.02。差距在于：人会在同一句子里比较两三篇论文，AI 倾向于逐篇孤立总结。

**操作规则**：每个主题段落中，至少一个句子同时引用并对比两篇以上论文。一个段落只引用一篇论文，它更像论文摘要而非综述。

详细的正反面写法示例见 `sub-skills/research-synthesis/references/synthesis-framework.md`。

## 矛盾呈现 → 分歧裁决

矛盾是洞见的富矿，不是需要遮掩的尴尬。发现矛盾后走完裁决程序（完整程序见 insight-protocol）：列候选调节变量（层级/口径/识别策略/时期/地区）→ 在交叉对比表上扩列建证据表 → 落入三种合法结局之一：**有条件立场**（分歧被调节变量解释，给条件化结论+表决计数）、**证据缺口**（明确写出缺什么设计的研究才能裁决——不写清缺什么不算走完）、**真实分歧**（同条件仍相反，本身即发现，提出可检验的深层解释）。**禁止以“结论不一，见仁见智”收尾；跳过硬矛盾是综合环节最大的浪费。**

## 自对抗审查

综合完成后，做一轮自我质疑（详见 `sub-skills/research-synthesis/references/self-adversarial.md`）：

1. **如果重新开始，研究简报会变吗？**（检查 frame-lock）
2. **有没有未审视的前提假设？**（如默认"准确率=更好"）
3. **结论是被证据支撑的，还是被选择性引用构造的？**（检查 confirmation bias）
4. **哪个视角被系统性遗漏了？**（学科、地域、方法、批判视角）

发现问题 → 修正 claim 或补充限定，不要为走流程硬造自我批评。

## 7 维内部门禁

综合过程中用 7 个维度做检查（详见 `sub-skills/research-synthesis/references/quality-gates.md`）。这些不对外呈现，是内部质量保证：

- **Angle**：有判断角度还是只在罗列？（CRITICAL）
- **Coverage**：关键工作都覆盖了吗？（MAJOR → 回调 literature-scout 补搜）
- **Citation**：引用真实、引述准确？（CRITICAL → 回调 literature-scout 核验）
- **Taxonomy**：按主题不按论文？MECE？（MAJOR）
- **Calibration**：判断强度 ≤ 证据强度？（MAJOR）
- **Weaving**：句内交叉对比？（MAJOR）
- **Insight**：结论达到裁决级（≥L2）了吗？矛盾裁决了吗？有读遍摘要也得不到的东西吗？（MAJOR → 按 `sub-skills/research-synthesis/references/insight-protocol.md` 的裁决程序与深问题重写对应章节）

## 产出

综合完成后，产出**结构化的综合分析**，按 survey paper 的标准组织：

- **摘要**（一段话概述）
- **引言**（为什么需要这篇综述 + 研究问题 RQ1/RQ2/RQ3 + 本文组织方式）
- **研究方法**（检索视角、纳入排除、证据类型说明）
- **分类框架**（MECE taxonomy 展示）
- **按分类分支组织的分析**（每个分支有 claim + 正反证据 + 对比表）
- **综合讨论**（跨分支的交叉分析、趋势、矛盾）
- **开放问题与未来方向**
- **结论**（逐条回答引言里的 RQ——结论是被证据推导出来的，不是预告的）

详细骨架和格式纪律见 `references/output-structure.md`。

## 模型自供知识的边界

- 基础背景（公认事实）→ 直接使用
- 具体方法细节、数据、发现 → 必须来自文献清单，不从记忆补充
- 引用 → 只写台账里的引用键 `[@key]`，不写数字；参考文献节由编译打印，不手写（单写架构：同一信息不允许有两个独立书写来源，数字这个信息在写作阶段干脆不存在）
- 不确定的背景 → 用 `scholar_search` 确认后再用，或标 `[待确认]`

## 洞见反思（成稿后，交付前）

逐条写下：“这份报告教给读者的东西里，哪些是读完全部入选摘要也得不到的？”每条标注 L 级。L3 及以上不足 3 条 → 回到裁决程序与深问题重写对应章节，不允许原样交付。该清单可直接作为执行摘要（检索报告）或摘要核心发现句（综述）的骨架。agent 模式下写入内部工作文件，固定标题“【洞见反思清单】”、逐条带 L 级，不进交付物（固定标题用于回归测试识别）。
