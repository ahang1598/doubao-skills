# 核验台账与分级声明

## 台账格式

台账小节的标题**固定写作"【题录核验台账】"**（固定标题同时用于回归测试识别，不要改写成其他措辞）。每篇经过核验的文献一行或一小块：

| 文献 | 字段 | 值 | 溯源 | 来源 URL | 核验时间 |
|---|---|---|---|---|---|
| Zhu et al. 2012 | DOI | 10.1016/j.enpol.2012.05.068 | 已核验（crossref） | api.crossref.org/works/... | 2026-07-09 |
| Zhu et al. 2012 | 期 | —（该卷无期号） | 已核验（crossref） | 同上 | 2026-07-09 |
| 某中文文献 | 页码 | 45-58 | 二手核验 | （聚合页 URL） | 2026-07-09 |
| 某文献 | 期刊等级 | — | 未核验 | — | — |

规则：
- 溯源取值只有四种：**已核验（crossref）/ 已核验（官方页面）/ 二手核验 / 未核验**。三档统计不得合并——把"二手核验"计入"已核验"、或声明里只报两档，均违规。
- 值列只能填本次会话工具返回中实际出现过的字符串；未核验行的值列写"—"。
- 来源 URL 列只能填本次会话工具返回中出现过、且已核对内容支持该值的链接；找不到真实来源时该行溯源写"未核验"、URL 写"—"，禁止放一个未访问过的链接占位。
- 期刊等级行的来源 URL 不得为指标聚合站/内容农场（LetPub、各类 journal-metrics 站点等）；只有此类来源时该行按“未核验”处理。
- 台账在核验发生时即时追加，不允许写完报告后凭产出反写。
- 出稿前做一致性反查：台账里的每个值与正文、参考文献中所有出现处一致；不一致必须消解后才能交付。
- 篇级完备性：最终清单中每篇文献在台账中至少有一组记录；没有台账记录的文献不得计入“已核验”统计，其题录字段一律“未核验”。
- 篇幅考虑可按文献聚合成"一篇一行、字段并列"的紧凑格式，但溯源与 URL 不可省略。

## 分级声明模板

诚信声明 / 核验声明 / 真实性声明由台账汇总**生成**，结构固定为五段：

1. **核验方法**：一句话说明实际使用的通道与工具（如"经 Crossref API 与公开网页核验，scholar_search 用于存在性确认"）。
2. **分档报数**：已核验 n 项；二手核验 n 项（逐条列出 + URL）；未核验 n 项（逐条列出）。三个 n 必须等于台账中对应溯源标注的行数——从台账数出来，不凭印象写；逐项列举与台账行一一对应，不重复不遗漏。
3. **未能访问的数据库/渠道**：逐一列名（如 Web of Science、Scopus、EconLit 订阅库），并说明对应需求以何种方式替代或未满足。
4. **日期**：检索范围截止日期 + 核验执行日期。
5. **复核建议**：提示用户对二手核验与未核验条目自行通过官方渠道复核。

## 反例（禁止出现的写法）

- ✗ "所有文献的题录信息均经过核实，确保准确无误"——存在任何二手核验或未核验字段时使用全称量词。
- ✗ 声明里写"全部核实"，局限性里又写"部分信息基于摘要和题录进行推断"——同一报告内自相矛盾，红线错误。
- ✗ 给字段值配一个格式合法但本次会话从未访问过的链接（如凭记忆写出的 LetPub / 期刊指标聚合站 URL）——**伪造溯源比缺失溯源更严重**，它让编造的值看起来经过了核验。
- ✗ 正文写"第 3 期"、台账或来源编号却对应第 5 期——同一字段多处出现值不一致，出稿前必须消解。
- ✗ 报告开头写"本次检索基于 Web of Science 开展"、末尾声明才承认未访问 WoS——披露必须出现在声称发生的位置。
- ✗ 为满足用户"须确认全部真实"的措辞要求而在未完成核验时照写模板——用户要求的是核验行为，不是核验措辞。
- ✓ "10 篇核心文献中：题录字段经 Crossref 核验 8 篇；经网页二手来源核验 1 篇（见台账第 9 行，建议复核）；1 篇的期号未能核验，已标注'未核验'。本环境无法直接访问 Web of Science 与 Scopus，相关核验以 Crossref 与出版商公开页面替代完成。"
