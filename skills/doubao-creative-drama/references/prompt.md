# 视频提示词 Ref

当关键帧已经获得用户确认/由用户提供，需要执行视频生成时读取本文件。

## 目标

运用已确认的分镜片段、资产和关键帧图，生成对应的视频，一次最多生成 2 个视频。

## 生成门禁

生成视频前必须让用户确认：

- 对应的 `storyboard.md` 分镜片段：必须明确到集数、章节、片段编号、绝对时间范围和镜头列表原文。
- **每个视频时长**：当前全局锁定 15 秒(下游生视频模型目前仅支持 15s,无需向用户澄清);30s 与 15s + 30s 双版本分支作为项目侧预留开关保留在下文"30 秒版本适配"章节,未来生视频模型支持 30s 后由项目侧一次性切换,或用户主动指定时启用。
- 画幅比例：`16:9`、`4:3`、`1:1`、`3:4`、`9:16` 或 `21:9`。
- 生成模式：图生视频、首帧驱动视频、参考视频生成。
- **本片段的关键帧列表**：必须包含 `frame.md` 阶段为该片段动态锁定的 1 到多张关键帧图（例如 `EPxx_CHxx_SEGxx_KF01`、`_KF02` …），用作该片段视频生成的图片锚点。
- 当前批次片段数量，一次最多生成 2 个视频。
- 台词、音效、音乐和字幕要求；默认不生成字幕，只有用户明确要求字幕时才允许生成字幕。
- 负向约束。

没有确认时，不得进行视频生成。

**分镜绑定硬约束：**

- 每个视频 Prompt 必须绑定一个已确认的 `storyboard.md` 连续分镜片段（15s 或 30s），不建议脱离分镜表自由编写新剧情、新镜头或新动作。
- 生成每个视频 Prompt 前，必须先列出"对应分镜映射表"，包含：视频编号、集数、章节、片段编号、片段时间范围、使用的镜头编号、对应分镜原文摘要、转换后的 Prompt 时间段。
- 如果找不到对应的分镜原文、片段时间戳不连续、镜头内容缺失，必须停止视频生成，返回 `storyboard.md` 阶段补齐或要求用户提供正确分镜。
- 如果用户只提供关键帧但没有提供对应分镜，不能直接根据关键帧脑补视频动作；必须先补齐或确认分镜。
- 一次最多生成 2 个视频，必须分别列出每个视频的分镜映射表，不能用一个笼统 Prompt 覆盖多个片段。

## 输入引用规则

**参考素材动态编号**
与生图阶段类似，在生成视频 Prompt 时，必须严格根据本次传入的素材进行动态重新编号，并说明每个引用的用途。**不建议传入无关资产**。

统一引用格式：

- `@图片1`、`@图片2` 等。
- `@视频1`、`@视频2` 等。
- `@音频1`、`@音频2` 等。

**关键帧引用规则（本 skill 采用的核心方式）：**

`frame.md` 阶段为每个片段动态锁定了 1 到多张关键帧。视频生成阶段的引用规则：

- **画面元素让关键帧图承担，Prompt 文字不复述**：关键帧图本身已承载构图、材质、服装、光源方向、色温、道具位置等具体画面元素；Prompt 文字**不建议再逐一复述这些元素**（否则会与关键帧图信息冲突，或触发下游对同一细节的双重解释）。Prompt 文字部分只负责描述**运镜、动作、剧情推进、声音（台词/音效/音乐/旁白）**，画面细节交给关键帧图。
- **关键帧只声明"时间锚点位置 + 命中类型"**：Prompt 中必须明确该关键帧在片段视频中的时间锚点身份——**首帧（0 秒开场）/ 中段锚点（对应镜头绝对时间点，视频画面必须经过此帧状态）/ 尾帧（片段末尾收束画面）**——并附上命中的判定标准（决定性瞬间 / 情节转折 / 身份亮相 / 状态突变 / 空间锚点 / 视角切换 / 群像定格 / 叙事开合 / 光影转折）。示例："@图片1 为片段一首帧""@图片2 为片段一 9-12 秒中段锚点（决定性瞬间）""@图片3 为片段一尾帧（悬念揭示）"。
- **单张关键帧片段**：单张关键帧默认作为该片段的**首帧**（若关键帧对应镜头不在 0 秒起点，则声明为对应时间点的中段锚点）。
- **多张关键帧片段**：按时序声明"首帧 / 中段锚点（对应绝对时间点）/ 尾帧"三种角色，`@图片1`、`@图片2`……与关键帧序号 `_KF01`、`_KF02` 保持时序对应。微电影向"光影转折"命中点若涉及色温冷暖翻转/光源熄灭/剪影等特殊状态，可在 Prompt 时间段的运镜描述里补一句状态变化提示（如"色温从冷转暖""灯泡熄灭"），但不建议展开描述光源方向与色相细节——那些由关键帧图承担。
- **上下片段承接**：生成非首个片段时，可以把上一片段的尾帧作为当前片段的开场连续性参考传入（此时该图在当前片段中标注为"承接参考帧"，非当前片段的首帧）。

用途标注示例（关键帧只声明时间锚点身份 + 命中类型，画面元素让图承担）：

- `@图片1：作为片段一首帧（EP01_CH01_SEG01_KF01，命中：空间锚点建立｜镜头1｜00:00）。`
- `@图片2：作为片段一 9-12 秒中段锚点（EP01_CH01_SEG01_KF02，命中：决定性瞬间/道具首次亮相｜镜头4｜00:09-00:12），POV 俯看铁盒的画面必须经过此帧状态。`
- `@图片3：作为片段一尾帧（EP01_CH01_SEG01_KF03，命中：情节转折/悬念揭示｜镜头5｜00:12-00:15），12-15 秒收束画面必须落到此帧状态。`
- `@图片4为林远角色资产参考（面部、发型、服装、身高比例以此为准，全片段稳定）；@图片5为铁盒道具资产参考（尺寸、锁扣位置、划痕分布以此为准）。`

**场景参考使用边界：**
场景资产图默认只在 `frame.md` 关键帧生成阶段使用。进入视频生成阶段后，关键帧本身已经承载了场景空间、构图、光影、道具位置等信息，因此视频 Prompt 默认不建议再额外传入场景资产图，避免关键帧与场景资产图之间产生参考冲突。只有当用户明确要求补充场景参考，或本片段锁定的关键帧数量为 1 且无法完整表达空间信息时，才允许额外传入场景资产图，并必须说明原因。

## 全局风格锚点与热门美学风格

为了保证与资产图的一致性，在视频 Prompt 的最开头，必须强制加入与项目设定完全一致的**全局风格限定词**，且**必须与 `scriptwriter.md` 剧本头 `风格：` 字段、`assets.md` 全资产统一锚点一字不差**。风格路径由上游 `scriptwriter.md` 决定：题材命中非真人强信号（用户直接指定画面制作形式，或内容本身无法用真人拍摄）时走"3D 玄幻/2D 水墨/2D 日漫赛璐璐"等非写实类型词；无非真人强信号时走真人电影路径，风格字段直接写"某某风格的电影"（如"王家卫风格的电影""是枝裕和风格的电影"）。视频阶段不得擅自切换风格路径。

**六种美学词与真人电影大师锚点的映射（选择时按剧本头风格字段自动对齐）：**

| 美学词（下方组合使用） | 常用大师锚点（写作"某某风格的电影"） | 适配题材 |
|---|---|---|
| **治愈清新**：自然光、柔和色调、干净治愈、日系空气感、温暖舒服 | 是枝裕和 / 李安 / 岩井俊二 | 都市治愈、亲情和解、青春成长 |
| **复古胶片**：轻微颗粒感、复古色调、胶片质感、漏光效果、怀旧氛围 | 王家卫 / 杜可风 / 贾樟柯 | 怀旧、年代、人物记忆 |
| **赛博朋克**：霓虹灯光、蓝紫对比色、科技未来感、潮湿的街道、暗调高级 | 雷德利·斯科特（《银翼杀手》）；若走非真人路径则写"赛博朋克风格"或"3D 科幻风格" | 科幻、未来都市 |
| **国风意境**：水墨晕染、留白构图、古典雅致、汉服飘逸、烟雨朦胧 | 若走真人路径可挑侯孝贤/王家卫；若走非真人路径写"2D 水墨"或"3D 国风" | 古风、仙侠、水墨短片 |
| **高级暗调**：低亮度、高对比度、极简构图、光影层次丰富、情绪内敛 | 大卫·芬奇 / 库布里克 / 罗杰·迪金斯 / 诺兰 | 独立文艺、悬疑、高级暗调微电影 |
| **纪实感**：原生自然光、手持镜头感、生活化场景、记录感、不完美的真实 | 达内兄弟 / 肯·洛奇 / 贾樟柯（纪实向） | 社会议题型微电影、独立成长 |

除上表外，`韦斯·安德森/PTA/昆汀 风格的电影`（对应广告剧情/短剧亮相戏/品牌微电影）与 `马利克/阿彼察邦/河濑直美 风格的电影`（对应影展参赛短片/意境类微电影）也是常用锚点，直接沿用剧本头 `风格` 字段作为全局风格限定词。

**风格限定词写法硬约束**：
- 走真人电影路径时，剧本头 `风格` 字段应已由 `scriptwriter.md` 主动匹配为"某某风格的电影"（如"王家卫风格的电影""是枝裕和风格的电影"或带代表作限定的"王家卫《花样年华》风格的电影"），视频 Prompt 全局风格限定词**必须一字不差沿用**，禁止简化为"电影风格""港风""复古胶片"等单独美学词，也不得擅自替换为其它大师。
- 允许在大师锚点后追加上表美学词组作为 Prompt 内部更细粒度的画面描述（如"王家卫《花样年华》风格的电影；结合复古胶片美学：轻微颗粒感、暗调霓虹、漏光效果"），但主锚点表述不得替换。
- 若视频阶段读到剧本头 `风格` 字段仍为"电影风格""影视级布光"这类留白式写法，说明上游未完成主动匹配，应停止视频生成并回到 `scriptwriter.md` 阶段补齐大师锚点后再继续。
- 走非真人路径时（题材命中 `scriptwriter.md` 两条非真人强信号），风格字段直接沿用"3D 玄幻""2D 水墨"等类型词，不得擅自套用真人电影写法。

## 多主体场景定义规则

当场景中存在多个主体时，必须在 Prompt 开头分别定义，并使用稳定标签区分，以防止 AI 混淆角色特征：

**示例**：
- @图片1 中的高个子男人是警察。
- @图片2 中的矮个子男人是小偷。

**执行建议与注意**：
- **标签唯一性**：简称应具备唯一性与稳定性。
- **强制指代**：后续描述中需持续使用对应主体名称（如 红衣女子），避免指代混乱。
- **不建议省略**：每次涉及该主体动作时，明确指代，不要省略主语。

## 特殊符号规则

| 内容 | 符号 | 例子 |
|---|---|---|
| 音乐 | `()` | `(背景中播放着快节奏的摇滚乐)` |
| 音效 | `<>` | `<远处传来狗叫声>` |
| 台词 | `{}` | `{你好，世界}`。如果台词为小语种（非中英文），需标注语种，如：用日语说道`{こんにちは}`。 |
| 旁白 | `『』` | `『旁白：三年前的那个雨夜，一切都改变了。』` |
| 字幕 | `【】` | `【第一章：启程】` |

**注意**：Prompt 中**不建议使用 -- 符号**。该符号之后的所有内容都可能被模型截断，不会被解析或响应。

## Prompt 结构与示例

结合以上规则，完整的 Prompt 结构应按以下顺序编写：

1. **参考素材列表和用途**（含全局风格限定词、逐张关键帧的时间点与叙事功能标注）。
2. **对应分镜映射表**（必须列出视频编号、集数、章节、片段编号、片段时间范围、镜头编号、分镜原文摘要、Prompt 时间段）。
3. **时长（15s 或 30s）、比例**。
4. **时间戳动作规划**（融合机位、光影、微表情，覆盖分镜的每个镜头）。
5. **声音设计**（台词、音效、音乐）。
6. **全局质感与连续性约束**（含关键帧对齐要求）。
7. **负向约束**：无论什么情况，Prompt 末尾都必须加入"禁止生成字幕、文字、LOGO、水印"等约束；除非用户明确要求生成字幕，才可以移除"禁止生成字幕"，但仍需禁止无关文字、LOGO、水印。

## 从分镜到视频 Prompt 的转换方法

`storyboard.md` 的分镜输出是逐镜头规划，格式通常为：

```markdown
- **镜头1 (mm:ss-mm:ss)**：[景别｜视角类型｜机位状态｜角度与朝向] 画面内容... {台词} 『旁白』 <音效> (音乐) 【字幕】 [转场]
```

转换为本文件的视频 Prompt 时，不要原样复制分镜表，也不要保留"镜头1、镜头2"等列表格式，而是按以下方式合并为可执行的视频生成指令：

1. **片段选择**：先确定本次生成的视频对应哪一个 15s 或 30s 片段。必须保留该片段的集数、章节、片段编号、绝对时间范围和镜头原文，不建议跳过这一步直接写 Prompt。
2. **素材绑定**：必须将该片段在 `frame.md` 阶段动态锁定的**所有关键帧图**作为视频参考素材传入，再按需传入角色资产、道具资产、参考视频或音频。场景资产默认不在视频生成阶段传入。所有素材动态编号为 `@图片1`、`@图片2`、`@视频1`、`@音频1`，并在 Prompt 开头逐张声明**时间锚点身份（首帧/中段锚点/尾帧）+ 命中的判定标准 + 对应绝对时间点**，画面元素让关键帧图承担，Prompt 文字**不重复描述**图中已有的构图、材质、服装、光源、道具位置等元素。
   - 关键帧数量与顺序按 `frame.md` 生成的 `_KF01`、`_KF02` … 时序传入。
   - 生成非首个片段时，可选择性把上一片段的尾帧作为连续性参考传入（在当前片段中标注为"承接参考帧"，非当前片段的首帧）。
   - 如果缺少对应片段的关键帧，必须返回 `frame.md` 补齐，不能只根据分镜表直接生成视频。
3. **分镜映射表**：在完整 Prompt 前列出本视频的映射关系，逐条说明 `storyboard.md` 的每个镜头被转换到 Prompt 的哪一个时间段。
4. **时间戳合并**：把分镜里的绝对时间戳保留下来，转换为 `0-3 秒`、`3-6 秒`、`6-8 秒` 这样的连续动作规划；相邻镜头如果动作连续，可以合并成一个较长时间段，但必须保留动作因果。不得新增分镜表里没有的剧情、角色、道具、场景或动作结果。30s 片段直接扩展到 `0-30 秒` 全区间，规划时把两组 15s 镜头视为连续序列。
5. **镜头语言转写**：把 `[景别｜视角类型｜机位状态｜角度与朝向]` 转成自然语言，例如"中景，第三人称客观视角，镜头水平右移，正面拍摄林远"。
6. **动作与剧情扩写**：把分镜中的动作、表情、手部、道具交互、空间关系和转场扩写成连续视频动作；扩写内容聚焦在**运镜速度、动作幅度与时长、剧情推进、声音**上，**不重复描述关键帧图中已有的画面元素**（构图/材质/服装/光源/道具位置由图承担）。扩写只能补充物理动作细节，不能改变剧情结果。
7. **关键帧对齐**：在 Prompt 时间段中简短声明关键帧作为该段的**首帧 / 中段锚点（对应绝对时间点）/ 尾帧**身份，例如"@图片1 为本段首帧"、"9-12 秒画面必须经过@图片2（_KF02 中段锚点）状态"、"以@图片3 为尾帧收束"。**Prompt 文字不再罗列关键帧图中已有的画面元素**，画面细节完全由关键帧图承担；仅在需要显式约束的连续性维度（角色面部一致、道具尺寸不变、锁扣位置不移等）上补一句负向约束。
8. **声音整合**：把 `{台词}`、`<音效>`、`(音乐)`、`『旁白』` 合并到对应时间段里；如不需要字幕，必须在约束中写"禁止字幕文字"。
9. **连续性约束**：在 Prompt 末尾补充角色面部、服装、道具、场景、光影、关键帧对齐、动作自然度、无穿模、无漂移等约束，并默认加入"禁止生成字幕、文字、LOGO、水印"；只有用户明确要求字幕时才允许生成字幕。

**对应分镜映射表格式：**


## 30 秒版本适配（预留分支）

**启用条件**：本章节为项目侧预留全局开关分支,当前默认 **不启用**（下游生视频模型仅支持 15s）。仅在以下两种情况之一才按本节适配：①未来生视频模型支持 30s,项目侧一次性把全局开关切换到 30s;②用户主动明确要求 30s 或 15s + 30s 双版本。启用后本片段以 30 秒生成的规则如下：

- 关键帧列表通常来自 `frame.md` 为 30s 片段锁定的 2-5 张关键帧，Prompt 中每张关键帧仍需标注命中的判定标准与对应绝对时间点（例如 0-3 秒、9-12 秒、17-20 秒、26-30 秒）。
- 时间戳规划从 0-30 秒连续覆盖分镜镜头，允许多个镜头合并为较长时间段以维持画面稳定。
- 关键帧对齐要求同 15s 版本，各时间段必须落到对应关键帧命中的判定标准状态；时序最靠后的关键帧对应片段末段状态，不再单独称为"结束帧"。
- 双版本并行时，两个视频共享同一批关键帧（15s 版通常是 30s 版的前一半或后一半），必须在映射表中说明是同片段的哪一个 15s 段，避免混淆。

## 质量检查

最终生成前需确认：

- Prompt 已列出对应分镜映射表，且每个 Prompt 时间段都能追溯到 `storyboard.md` 的集数、章节、片段、镜头编号和分镜原文。
- Prompt 严格遵循对应片段分镜，没有新增分镜表中不存在的剧情、角色、道具、场景或动作结果。
- 视频参考素材已包含 `frame.md` 为该片段动态锁定的所有关键帧，每张关键帧均在 Prompt 中声明**时间锚点身份（首帧/中段锚点/尾帧）+ 命中的判定标准 + 对应绝对时间点**。
- Prompt 文字部分未复述关键帧图中已有的画面元素（构图/材质/服装/光源/道具位置），文字聚焦在运镜、动作、剧情、声音；连续性维度（面部一致、服装剪裁、道具尺寸/锁扣位置）以负向约束方式落在末尾约束段。
- 角色、服装、道具、场景和光影保持一致。
- 时长（15s / 30s）、比例、模式、批次数量明确。
- Prompt 末尾已加入"禁止生成字幕、文字、LOGO、水印"等负向约束；除非用户明确要求字幕，否则不得生成字幕。
- 单批生成不超过 2 个视频。
