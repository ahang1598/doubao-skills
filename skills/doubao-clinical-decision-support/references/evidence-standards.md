# 证据检索与评价标准

本文件只定义证据来源、检索顺序、预算、CSV 台账、全文阅读和质量评价。执行顺序见 `core-workflow.md`，图表见 `clinical-diagrams.md`，报告结构和交付 QA 见 `output-and-qa.md`。

## 1. 单一证据源

literature-log CSV 是完整报告唯一的内部结构化证据源，同时承担：

- 检索候选记录。
- 初筛和纳排记录。
- 全文/摘要阅读状态。
- 正文引用和图表节点映射。
- 核心证据的关键片段、病例适配性与局限性记录。

不要再创建独立 JSON 证据表、citation map、第二张筛选表或第二套引用映射。正文中的指南/关键研究卡片、段落或简表只是从 CSV 核心记录生成的展示视图，不是新的数据文件，不要求回写或单独校验。

## 2. 检索顺序和来源优先级

完整报告先创建 CSV，再按以下顺序检索：

1. `general_search`：先搜中国临床指南、本土专家共识、国家/学会规范，再补充国际权威指南。优先接受国家卫生健康部门、国家监管机构、中华医学会及其专科分会、国际权威学会、指南发布机构或正式期刊页面；核对题名、发布机构、年份/版本、适用人群和可访问原文链接。
2. `scholar_search`：完整报告在指南/共识检索后的必经步骤，用于主动检索与问题直接相关的关键研究论文，丰富证据类型并核对推荐依据；优先系统综述/Meta、RCT、大型队列、诊断研究和高质量期刊论文。按问题选择研究类型，不为凑数量扩展。
3. `general_search` 说明书补充：药品问题优先按“`小荷医典 + 药品通用名 + 药品说明书`”检索，并核对药品名称、剂型、来源和链接。
4. `medical_search`：只用于药品说明书的最后补充；仅当上一步找不到、打不开或无法核验所需说明书时调用。
5. WebFetch/`web.fetch`、PDF 或 OCR：仅当现有信息不足以核对关键推荐、入排标准、效应量、亚组、禁忌或安全边界时，定向阅读相关章节或表格，不默认遍读全文。

搜索优先级为 `general_search 指南/共识 → scholar_search 关键研究 → general_search 说明书 → medical_search 说明书补充`。前两步共同构成完整报告的基础证据检索；不为凑数量运行额外搜索。说明书无论来自哪一步，都必须记录标题、来源和可访问链接。

当已获得可追溯的权威指南/共识，并已通过 `scholar_search` 覆盖相关系统综述或关键原始研究，两类证据对目标人群、当前问题、关键风险和适用边界已作出清楚回答，且没有会改变结论的证据冲突时，即视为信息足够：可停止扩展检索，也无需详细阅读整篇指南或论文；只核验能支撑结论的官方推荐页或必要关键章节。只有摘要的研究仍归为辅助证据。

来源优先级：

| 等级 | 来源 | 使用边界 |
|---|---|---|
| A | 最新中国或国际权威指南/共识、Cochrane 或高质量系统综述、高质量 RCT/大型队列、药品说明书、FDA/EMA/NMPA 等监管来源、顶级或核心期刊关键研究 | 可支撑核心结论 |
| B | 质量较好的观察性、真实世界或诊断研究，权威教材/专著、卫生政策、技术报告和专科规范 | 补充证据与实践背景 |
| C | 小样本单中心研究、病例系列/报告、会议摘要、预印本、质量一般的叙述性综述、机制/动物/体外研究 | 前沿线索或低确定性补充 |
| D | 自媒体、营销软文、问答社区、搜索结果页、来源不明转载、疑似垃圾期刊或无法核验页面 | 不作为依据 |

中国临床实践问题优先查最新版中国指南、本土共识和国内规范，再用国际指南和高质量研究互证。经典里程碑研究不受年份限制；“最新进展”默认优先近 1-5 年。

关键词按问题需要覆盖疾病中英文名/缩写/分型、干预或药物通用名/商品名、对照、关键结局、特殊人群和研究类型，不用大量同义词反复搜索同一问题。

## 3. 检索预算和覆盖

预算用于控制上下文，不用于降低医学质量：

| 场景 | search_budget | fetch_budget | 停止条件 |
|---|---:|---:|---|
| 简单、直接、风险低的快速回答 | 1-2 个高价值批次 | 0-1 份定向核验 | 权威指南/共识与相关研究已直接回答并覆盖必要风险 |
| 聚焦单一问题，如单药安全或单检查选择 | 1-3 个批次 | 0-2 份定向核验 | 已覆盖最新指南、必要说明书或关键研究，且无明显冲突 |
| 常见复杂病例、多合并症或多方案比较 | 2-4 个批次 | 1-4 份定向核验 | 指南/共识、必要关键研究和安全来源已覆盖，新证据不再改变结论 |
| 罕见病、证据冲突、高风险安全问题或指南缺失 | 可按需超过 | 可按需超过 | 只扩展可能改变结论或安全边界的来源 |

完整报告按问题需要覆盖，不为达到数量继续搜索：

- 指南/共识优先中国指南和本土共识，通常保留 1-3 条最直接、最权威的来源。
- 关键研究必须进行专门检索；按问题保留最相关的系统综述、RCT、队列或诊断研究，丰富证据类型，但不强制凑足固定数量。
- 涉及药品、禁忌或特殊人群时，安全/说明书/监管来源 1-3 条。
- 本土实践、真实世界或机制证据按需补充。

证据不足时直接披露，不用低质量来源补数量。

## 4. CSV 生命周期和字段

完整报告开始时，从 `assets/literature-log-template.csv` 创建 UTF-8 with BOM 的：

`literature-logs/{{YYYYMMDD}}-{{case_slug}}-literature-log.csv`

文件名和内容不得包含患者姓名、身份证、手机号、住院号等敏感信息。CSV 的本地绝对路径只用于内部附件交付，不写入报告或最终回复；默认不生成 Excel 副本。

CSV 只保留 13 个字段，按工作阶段分组：

| 阶段 | 字段 | 填写时机 |
|---|---|---|
| 候选入库 | `evidence_id`、`search_context`、`title`、`source_and_year`、`publication_type`、`abstract_or_summary`、`url` | 每轮高相关候选返回后立即填写 |
| 初筛与阅读 | `evidence_level`、`status` | 初筛及定向打开原文后更新 |
| 核心证据 | `anchor_text`、`key_excerpt_or_location`、`appraisal_and_applicability`、`use_in_report` | 纳入核心证据后补齐 |

填写规则：

1. `evidence_id` 是每条记录的唯一编号，候选入库时即分配并保持稳定，如 `E001`、`E002`。
2. `search_context` 合并检索批次与问题，简写为 `S1-guideline | CKD 用药指南`；不保存冗长的完整检索式。
3. `source_and_year` 合并期刊/机构与年份，如 `KDIGO，2024` 或 `N Engl J Med，2020`。
4. `evidence_level` 合并初筛来源等级和原文分级，如 `A 级来源；原文 1A`；原文未分级时写 `A 级来源；原文未报告`。
5. `status` 合并筛选结果和阅读深度，使用如 `待筛选/未打开`、`纳入核心/全文`、`纳入核心/指南正文`、`纳入辅助/仅摘要`、`排除/未读` 或 `待核验/无法访问`。
6. `appraisal_and_applicability` 使用 `纳入理由：...；病例适配：...；局限：...` 合并记录；`use_in_report` 使用 `结论 1；正文；图：支持` 简写引用和图表用途。
7. 每轮检索先追加候选，再筛选、精读、绘图或写报告；排除和待核验记录不删除。
8. `url` 必须是可点击 HTTP(S) 原文、摘要、PubMed、期刊页、DOI 解析页或官方入口；DOI 不再单独设列，需要时写成 `https://doi.org/...`。
9. 核心记录必须补齐 `evidence_level`、`anchor_text`、`key_excerpt_or_location`、`appraisal_and_applicability` 和 `use_in_report`。
10. 含逗号、换行或双引号的字段按标准 CSV 规则转义；交付前运行 `scripts/validate-literature-log.js <csv>`。

`quick_answer` 不创建 CSV，只保留 1-3 个可点击核心来源。

## 5. 证据准入和原文阅读

核心证据必须同时满足：

1. 有可点击原文、摘要、PubMed、期刊页、DOI 解析页或官方入口。
2. 已打开正文、指南原文、说明书或可核验页面，并如实记录阅读状态。
3. 能定位到支持当前结论的推荐、表格、段落、摘要结果或论文正文位置。
4. 与病例人群、疾病阶段、干预/对照和关键结局直接相关。

只有题名、DOI/PMID 字符串或搜索片段而没有链接时，不能作为核心证据。只能读到摘要的资料可标记为 `纳入辅助/仅摘要`，用于背景、研究线索或一致性补充，不能单独支撑关键疗效或安全结论。

定向阅读重点：

- 指南/共识：推荐条文、推荐强度/证据等级、适用与排除人群、禁忌、监测、安全性、推荐理由和更新日期。
- 系统综述/Meta：PICO、检索范围、纳入研究、偏倚、异质性、主要结果、敏感性分析和结论边界。
- RCT/队列/真实世界研究：样本量、入排标准、干预/暴露、对照、主要终点、效应量、置信区间、安全性、亚组和局限。
- 诊断研究：目标人群、指标检查、参考标准、敏感度、特异度、LR、AUC、PPV/NPV 和误判后果。
- 说明书/监管来源：适应证、禁忌、剂量、特殊人群、相互作用、警示和更新时间。

长网页或 PDF 先定位章节、页码、表格或关键词；每份核心来源只保留 1-3 条可定位关键片段或摘要性提炼，不把整份文档塞入上下文。全文不可访问时不反复 Fetch，同步更新 `status`。

## 6. 快速质量评价

`evidence_level` 中的“来源等级”是初筛评价，不等于原文 GRADE 或正式证据等级；若原文报告分级，在同一字段中原样记录，没有则写 `原文未报告`。

| 证据类型 | 重点评价 |
|---|---|
| 指南/共识 | 机构权威性、系统检索、证据等级、推荐强度、适用范围、利益冲突和更新时间 |
| 系统综述/Meta | PICO、检索充分性、纳入研究质量、异质性、发表偏倚、敏感性分析和稳健性 |
| RCT | 随机、分配隐藏、盲法、样本量、主要终点、ITT、安全性和失访 |
| 观察性/真实世界研究 | 数据源、混杂控制、暴露/结局定义、选择偏倚、缺失数据、随访和外推性 |
| 诊断研究 | 参考标准、代表性、盲法、诊断性能和误判后果 |
| 机制/基础研究 | 模型与临床人群差异，不直接外推临床疗效 |

不要声称完成正式 GRADE 评级，除非原文采用 GRADE 或等价体系；无原文分级时只能写“按研究设计和偏倚风险初步判断”。

## 7. 引用、图表和文档展示

CSV 直接驱动三个出口：

1. 正文引用：使用 `anchor_text + url` 生成可点击来源名，并标注期刊/机构、年份和来源级别。
2. 图表节点：仅使用 `纳入核心` 或明确标注的 `纳入辅助/仅摘要` 记录，节点能反查 `evidence_id` 和 URL。
3. 完整参考文献：从 CSV 按核心证据、辅助参考/仅摘要、排除/待核验分组，展示 `标题｜来源｜链接`。

指南与关键研究章节可以自由使用段落、卡片或简表呈现核心发现、病例适配性和局限性，但内容必须来自 CSV 核心记录。不要把 CSV 全量复制进正文，也不要再创建独立证据表文件或要求第二次字段校验。

正文首次提到具体指南、共识、研究、说明书或监管来源时，名称本身必须是超链接；普通 `[G1]`、`[R1]` 只作为辅助索引，不能替代链接。citation 样式组件可以使用，但必须承载同一个可点击来源。

结论采用条件性表达：`相对确定`、`倾向支持`、`可作为备选`、`需谨慎讨论`、`存在争议`、`证据不足` 或 `当前不支持`。避免写成直接医嘱或使用“强推荐”“强烈支持”“强烈建议”等高程度措辞。
