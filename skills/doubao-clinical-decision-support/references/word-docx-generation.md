# Word/docx 生成规范

本文件用于用户明确要求 Word、docx、正式报告、可打印归档材料时。生成方式必须引导模型使用已有 `lark-doc` Skill；`office-word` 是 `lark-doc` 内部处理本地 Word/PDF 与强格式交付的模块，不是独立入口。本规范参考 `lark-doc` 的 `office-word/office-word-skill.md` 中的通用 Word 交付、docx 创建和排版规则，并按循证医学报告场景裁剪。

## 路由原则

- 先调用已有 `lark-doc` Skill；按其顶层路由，用户明确要求交付 Word/docx 时读取并执行 `office-word/office-word-skill.md`。
- 若 `lark-doc` 的 `office-word` 模块不可用，但存在同环境认可的 Word/docx 生成能力，可按本文件规范生成 `.docx`，并在最终回复中说明未能走 `lark-doc` 路由的原因。
- `assets/word-report-template.md` 只作为内容骨架，不是最终 Word 产物。用户要求 Word/docx 时，默认交付可打开的 `.docx` 文件；Markdown 只用于工具不可用时的可重试草稿。
- 若用户提供了 Word 模板或既有 `.docx`，优先保留原模板样式，只替换内容；不要从零生成导致格式退化。

## 默认医学报告版式

| 项目 | 默认规范 |
|---|---|
| 纸张 | A4 |
| 页边距 | 上、下、左、右各 2.5 cm |
| 正文中文 | 宋体，小四，1.5 倍行距，两端对齐，首行缩进 2 字符 |
| 正文英文/数字 | Times New Roman，与中文同字号 |
| 一级标题 | 黑体，三号，居中，段前 12 磅、段后 6 磅 |
| 二级标题 | 黑体，小三或四号，左对齐，段前 10 磅、段后 6 磅 |
| 三级标题 | 黑体，四号，左对齐 |
| 表格 | 表题置于表上；表头浅蓝或浅灰底；边框浅灰；单元格上下留白；表格宽度不超页面正文宽度 |
| 图 | 图题置于图下；图中证据节点必须在图例或正文中有链接和来源级别 |
| 页眉 | 报告短标题或“循证医学决策支持报告” |
| 页脚 | 页码居中；正式归档版可加日期或版本号 |

## 文档结构

Word 报告比飞书文档更适合打印和归档，应保持清晰、克制、专业：

1. 封面或标题区：病例主题、报告类型、生成日期、适用对象。
2. 总结论：用短段落和小表格先给结论、确定性、风险和适用边界。
3. 病例与临床问题：病例结构表 + PICOTS/PECO。
4. 临床问题分析和解答：置于指南/共识和关键研究之前，先给问题总结，再按实际问题自由组织病例线索、证据回答、风险边界和需要补充的信息；可引用后文 `[G1]`、`[R1]` 或研究简称。
5. 证据关系与决策逻辑：可用渲染后的流程图、表格图或 Mermaid 附代码；按病例现场生成，不照抄固定示例，必须有图例表。
6. 指南/共识证据：推荐内容、等级、病例适配性和原文定位。
7. 关键研究证据：期刊/来源、研究类型、级别、关键片段、局限性。
8. 风险、安全性与下一步：按风险等级和行动建议整理。
9. 检索策略与证据覆盖。
10. 完整参考文献（含链接）：核心证据、辅助/仅摘要、排除/待核验分组表格；CSV 不作为本地链接写入正文或最终回复。
11. 免责声明。

不要把飞书 callout、互动块或过多颜色硬搬到 Word。Word 版用标题层级、表格、页眉页脚、编号和适度底色体现专业感。

## docx 生成硬规则

- 不要把 Markdown 文件改名为 `.docx`。
- 不要用纯聊天正文替代 Word 交付。
- 不要在正文段落里用换行符堆版式；每个自然段应是独立段落。
- 图表必须编号，并在正文中引用，如“见图 1”“见表 3”。
- 表格必须设置稳定列宽；不要让超宽参考文献表横向溢出页面。
- 长 URL 不直接铺满表格；正文用锚文本链接，附录可另列 URL 或 DOI/PMID。
- 核心证据解读内容较多时，按指南、关键研究和安全来源拆分段落或简表；不要复制 CSV 全量字段，也不要压成不可读的超宽表格。
- 参考文献优先使用表格；核心证据、辅助参考/仅摘要、排除/待核验分组展示，每条展示记录必须有可点击原文/摘要链接。

## 表格排版建议

医学循证报告常见表格较宽，默认使用以下策略：

- 病例结构表：3 列，`维度 / 当前信息 / 对决策的影响`。
- 结论表：4 列，`结论 / 确定性 / 关键依据 / 风险边界`。
- 指南表：必要时拆为 6 列以内；原文定位和链接放在表格单元格、正文括号或附录表中，不使用脚注或上标。
- 关键研究表：正文只放 6-7 列；分区/影响因子、关键片段、局限性可放附录扩展表。
- 完整参考文献（含链接）：不要原样粘贴全 CSV；按核心、辅助、排除/待核验分组做表格。CSV 只作为附件交付，不在正文或最终回复中列本地链接。

## 验收检查

交付前至少检查：

- 产物是真正 `.docx`，能被 Word 或兼容工具打开。
- 标题层级不跳级，目录/导航可识别。
- 页边距、字体、行距、页眉页脚稳定。
- 表格没有明显溢出、重叠、被截断或不可读。
- 图表有编号、图题和图例，证据节点可追溯。
- 正文首次提到关键文献时必须有超链接；引用编号只能辅助定位，不能替代链接；不得把指南/研究名写成无链接小标题。
- 文末“完整参考文献（含链接）”完整，分为核心证据、辅助参考/仅摘要和排除/待核验；标题包含“链接”，默认使用 `标题｜来源｜链接` 三列表格，表格中的展示记录都有可点击来源。
- 若未能生成 `.docx`，最终必须明确写出原因，并提供可直接重试的完整 Markdown/内容草稿。
