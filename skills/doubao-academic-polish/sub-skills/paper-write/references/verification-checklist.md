# Verification Checklist

S4 交付前审查。整篇论文必须逐项过；单章节/段落在心里过一遍。

## Step 1: 无来源声明扫描

逐句检查，确保正文里没有追溯不到来源的事实性声明（不涉及任何内联标签——正文交付时零方括号标签）：

- [ ] 每个事实性声明都能追溯到用户输入、scholar_search 结果、或 L0 领域常识？
- [ ] 没有只凭模型记忆（L4）写出的事实句？
- [ ] 追溯不到来源的句子：用 scholar_search 验证；失败则改写为不依赖该事实的措辞，或删除？

## Step 2: 内联各项审查（对应 S4 第三步；引用存在性已上移至 S4 第二步独立核查）

### 2.1 文献审查

> **引用真实性（下面前 3 项）由 S4 第二步的 OrganizeAgent 独立核查执行**——整篇论文 / Final Mode / 累计引用 ≥ 3 条时，由全新上下文的 citation-verify agent 用 `scholar_search` 逐条核查，触发分档与协议见 `citation-verification.md`（触发条件以该文件的表为准）。这里把这 3 项列出，是给单段落 / 引用 ≤ 2 条时做内联兜底用；独立核查触发时，以独立核查的 VERIFIED / METADATA_MISMATCH / NOT_FOUND / OVERCLAIM / INCONCLUSIVE 结论为准。其余各项（事实、证据匹配、跨章节、AI 痕迹、§2.6 公式表格、§2.7 交付格式）始终在本步内联完成，独立核查不覆盖。

- [ ] 正文中每个 `[N]` 引用可追溯到用户输入或 scholar_search？
- [ ] 没有凭模型记忆编造的参考文献？
- [ ] 没有编造文献的方法论描述（如把 review 写成 "meta-analysis"）？
- [ ] 文内 `[N]` 标识齐全（不是只在文末列参考文献而正文没有引用）？
- [ ] References 节已逐条输出，编号与正文 `[N]` 一一对应？
- [ ] References 中每条包含作者、标题、会议/期刊、年份（无法确认的条目已删除，缺少的非必要字段已省略）？
- [ ] 引用格式一致（默认 numeric citation-sequence `[1]`, `[2]`, ...，或用户指定的 citation style）？

### 2.2 事实审查

- [ ] 制度细节（政策规则、政府计划机制）来自用户输入？没有自行补充？
- [ ] 案例引用号、法条编号来自用户输入？没有凭记忆写？
- [ ] 科学比较值有来源？
- [ ] 没有编造用户未提供的统计结果（covariate p-values、聚合统计）？
- [ ] **没有编造用户未提供的程序细节？** 包括：
  - 量表题目数（如 "10-item scale"）
  - 量表锚点（如 "0=never to 4=always"）
  - 聚合统计（如 rwg）
  - Consent 方式
  - 软件版本
  - 实施时间
  - 缺失数据率
  - 实现细节（beam size、bins 数量、采样策略）
- [ ] **没有编造用户未提到的应用场景或使用领域？**（如：光伏电站、自动驾驶、医疗诊断——用户没说就不写）
- [ ] **没有编造用户未描述的技术机制、物理原理或因果解释？**（如：频谱混叠、梯度耦合——用户没说就不写）
- [ ] **没有编造用户未提供的规模描述？**（如：hundreds of / fewer than ten / tenfold / order of magnitude——用户没给就不写）

### 2.3 证据匹配审查

- [ ] 每个强判断有证据或边界限定？
- [ ] L3（元数据）来源没有被写成 full-text-level 判断？
- [ ] 用户给的数据按其性质呈现（真实结果 vs 预期结果）？
- [ ] 贡献声明可追溯到实验/图表？
- [ ] Draft Mode 的规划数据都标注了？

### 2.4 跨章节审查

- [ ] Discussion 解释 why，没有重复 Results 的 what？
- [ ] Conclusion 回答 Introduction 的问题，没有逐节总结？
- [ ] Abstract 和 Conclusion 不是换词重复？
- [ ] 同一个发现没有在超过 3 个章节出现？

### 2.5 AI 痕迹审查

- [ ] 对照 `../../../references/ai-tone-guardrails.md` 扫过？
- [ ] 没有夸大词（"innovative" / "pioneering" / "groundbreaking"）？
- [ ] 动词强度匹配证据强度？
- [ ] 没有内部检查痕迹泄露到正文？
- [ ] 输出中没有任何方括号占位标签（`[citation needed]`、`[to be verified]`、`[to be specified by authors]`、`[to be confirmed]` 及其变体）？
- [ ] 没有过程前言（"Let me first analyze..."、"Based on the evidence..."、"Following the CER framework..."、"首先让我分析一下..."、"根据 S2 的蓝图..."）？

### 2.6 公式与表格格式审查

- [ ] 公式使用 `$$...$$`（行间）和 `$...$`（行内），没有裸写下标或 HTML 标签？
- [ ] 公式后符号说明与公式内符号完全一致？
- [ ] 结构化数据（实验结果、消融、统计）用表格呈现，没有压成连续正文？
- [ ] 表格有表题、列头、单位？表题表注没与正文粘连？
- [ ] Draft Mode 的占位表格有标准结构（非空泛"表 X 待补"）？
- [ ] 正文没有 `**...**`、`_..._`、emoji？

### 2.7 交付格式审查（仅整篇论文/多章节时适用）

- [ ] 产出需要文件交付吗？（单段落/单章节 → 对话输出即可，跳过本节）
- [ ] 如需文件：Markdown 源稿已生成？
- [ ] 如需文件：飞书文档转换已完成，或已说明"待转飞书文档"？
- [ ] 如需文件：公式在飞书中用公式块呈现（非纯文本下标）？

## 不过时怎么办

- 单项不过 → 修复再交付
- 无法修复（缺用户材料）→ 标出风险，不伪装成终稿
- Final Mode 不过 → 不说"可提交"
