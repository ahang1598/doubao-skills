# 深度结构梳理方法

主 SKILL.md 里的流程足够处理大多数场景。这份参考文件提供三套更深入的方法，适用于论文结构问题比较严重、或者作者希望做一轮彻底的结构重整时使用。需要时再翻，不必每次都读。

---

## 一、段落功能矩阵

当一篇论文的结构混乱到"哪段该留、哪段该挪、哪段该删"都说不清的时候，最有效的做法是先给每个段落标注它的"工作"，再看这些工作排在一起是否讲了一个完整的故事。

### 怎么做

逐段（或逐小节）过一遍，对每个写作单元回答四个问题：

| 问题 | 说明 |
|---|---|
| **这个段落在干什么？** | 它的论证功能是什么——铺背景、指出缺口、交代方法设计理由、呈现证据、解读意义、还是承认局限？ |
| **它在为红线服务吗？** | 这个功能和论文的核心主线是什么关系？直接推进、间接支撑、还是偏离了？ |
| **它靠什么站住？** | 支撑它的是实验数据、一手材料、文献引用、逻辑推理，还是什么都没有？ |
| **它和前后段是什么关系？** | 上一段到这一段的过渡自然吗？这一段结束后，下一段接得上吗？ |

### 常见的段落功能标签

不同章节有不同的典型功能。理结构时，先看每段实际在做什么，再看它是否在该出现的位置：

**引言**：建立领域重要性 → 概述已知 → 指出缺口 → 提出问题/目标 → 预告方法 → 列出贡献

**方法**：概述整体框架 → 交代设计理由 → 描述具体步骤 → 说明评估方式

**结果**：交代实验设置 → 呈现核心发现 → 对比基线 → 解读发现的含义 → 过渡到下一组结果

**讨论**：回答引言提出的问题 → 解释为什么出现这个结果 → 和前人工作比较 → 承认局限 → 指出意义

如果某个段落的功能和它所在章节的职责不匹配——比如引言里就开始解读实验结果，或者讨论里在重复呈现数字——那就是一个错位，需要把它挪到正确的位置。

### 什么情况下该用这个方法

- 作者说"我知道结构有问题但不知道哪儿有问题"
- 一篇论文读起来"哪儿都有点不对但说不上来"
- 引言超过三页还没说清楚要做什么
- 同一个论点在多个章节里反复出现

## 二、承诺-回应闭合表

一篇论文在引言里会做出若干"承诺"——我们要解决什么、要验证什么、要回答什么。这些承诺在结果和讨论里必须被逐一兑现。闭合表就是把这个对应关系显式地列出来，检查有没有"承诺了但没兑现"或"兑现了但没承诺"的断裂。

### 怎么做

从引言的最后几段提取所有承诺（通常是贡献列表、研究问题、或"本文将要…"的句子），然后对照结果和讨论：

| 引言承诺 | 结果里哪个小节回应了它 | 讨论里怎么收束的 | 状态 |
|---|---|---|---|
| 承诺 1：… | Results §X.X | Discussion 第 X 段 | ✓ 闭合 / ✗ 断裂 / △ 弱回应 |
| 承诺 2：… | — | — | ✗ 完全缺失 |
| （结果里的意外发现） | Results §Y.Y | Discussion 第 Y 段 | △ 有结果但引言没铺垫 |

### 三种断裂模式

1. **承诺悬空**：引言说"我们将验证 X"，但结果里找不到对 X 的验证。最常见的原因是实验没做完就开始写引言，承诺留在那里忘了删或忘了补。
2. **结果孤立**：结果里有一个重要发现，但引言里完全没有铺垫。读者会觉得这个发现是"碰巧做出来的"，而不是"专门去验证的"。
3. **弱回应**：引言承诺了 X，结果里有沾边的内容但没有正面回答。比如引言说"我们将分析 A 和 B 的因果关系"，结果里只报了相关系数，没有做因果推断。

### 什么情况下该用这个方法

- 引言贡献列表超过三条
- 结果部分有五个以上小节
- 讨论读起来和引言"两张皮"
- 审稿人反馈"贡献 X 在文中没有得到充分支撑"

## 三、闭卷重写法

当一段文字的问题不是措辞而是逻辑——改句子已经救不了，需要重新组织——但又不想丢掉其中的事实和数据时，可以用闭卷重写。

### 怎么做

1. **提取事实**：从原文中把所有硬事实抽出来——数据、数字、引用、图表引用、实验条件。只记事实，不记原文的句子结构和论证顺序。
2. **合上原文**：不再看原文的段落安排。
3. **从骨架出发重写**：按照你已经理好的红线和段落功能，用那些事实重新组织一段新的文字。每个段落先想清楚它的功能是什么，再把对应的事实填进去。
4. **打开原文核对**：写完之后，和原文对照，检查有没有遗漏重要事实、有没有不小心改变了科学含义。

### 为什么这样做有效

直接在原文上改，容易被原文的结构和顺序牵着走——改来改去，段落功能还是错的，只是句子变好看了。闭卷重写强制你从"这段该干什么"出发，而不是从"原来写了什么"出发。

### 什么情况下该用这个方法

- 一个段落改了三遍还是读不通
- 引言的论证链条断了，逐句润色修不好
- 作者自己说"这段我写了好多版，越改越乱"
- 讨论只是在复述结果，换个说法还是在复述

### 注意

闭卷重写的产出要交给作者确认——你是在重新组织论证，不只是在改措辞，有可能改变了重点或弱化了某个作者认为重要的点。重写完之后，把你做的主要调整说清楚，让作者判断。
