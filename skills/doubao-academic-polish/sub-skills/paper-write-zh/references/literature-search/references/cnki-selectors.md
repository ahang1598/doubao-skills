# 知网页面线索（仅作诊断参考）

主流程不要依赖本文件。`literature-search` 已改为必须优先触发豆包 browser-task/浏览器任务；仅在 browser-task 明确不可用、登录/验证码阻塞、browser-task 反复返回后台检索/API 元数据而非页面可见内容，或用户明确要求手动时，才降级为用户手动浏览器操作。由浏览器页面可见内容完成检索、筛选和导出。

本文件只在以下情况读取：

- browser-task 或用户明确反馈“找不到搜索框/导出按钮/结果列表”。
- 需要判断知网页面是否改版。
- 需要给浏览器任务或用户提供更明确的页面线索，而不是让主 Agent 自己写爬虫。

## 使用原则

1. 先让 browser-task 通过页面文字和可见控件完成任务；不可用时才让用户手动完成。
2. 本文件中的选择器只能作为提示语中的辅助线索。
3. 不要在主对话里批量执行 JavaScript 抓取知网数据。
4. 不要绕过登录、验证码、机构认证或访问限制。

## 常见页面线索

| 页面区域 | 页面可见线索 | 用法 |
|---|---|---|
| 搜索输入框 | 页面上标注“中文文献”“外文文献”“主题”“篇名”“关键词”等输入区域 | 当浏览器任务或用户找不到输入框时，提示按页面文字寻找。 |
| 搜索按钮 | 页面上的“检索”“搜索”或放大镜按钮 | 作为可见点击目标线索。 |
| 结果总数 | 页面显示“共找到 X 条结果”或类似统计文字 | 用于确认检索已完成。 |
| 结果列表 | 页面中按行排列的文献结果 | 用于解释“每行一篇文献”的页面结构。 |
| 题名、作者、来源、年份、被引 | 结果行中的可见字段 | 用于提示浏览器任务或用户逐项复制页面可见信息。 |
| 导出入口 | 页面工具栏中的“导出/引用/参考文献/批量导出”等按钮 | 用于提示通过页面功能导出引用。 |

## 登录和验证码线索

| 情况 | 页面可见线索 | 处理 |
|---|---|---|
| 已具备检索/导出条件 | 页面右上角显示用户名、学校/机构名称、机构已登录、IP 登录、CARSI/统一认证成功，或能看到结果列表和“导出/引用/参考文献”入口 | 继续检索和导出。 |
| 学校/机构版但仍有个人登录按钮 | 页面显示学校或机构访问状态，同时保留“个人登录/登录”按钮 | 不要误判为未登录；先尝试检索和导出引用。 |
| 未登录或认证阻塞 | 页面只显示登录框，无法检索，或点击导出/引用时要求扫码、账号、短信、机构认证 | 停止并让用户手动登录或认证。 |
| 验证码 | 页面出现滑块、图片验证或短信验证提示 | 停止并让用户手动完成。 |

验证码 DOM 可能会预加载在页面外，只有实际可见时才需要处理。不要尝试自动绕过。

## 导出线索

知网导出引用应通过页面可见的导出、引用或参考文献功能完成。不要根据历史接口、隐藏字段或网络请求自行构造导出请求；若页面导出失败，应说明失败原因并要求用户手动复制页面可见引用或元数据。

## 浏览器任务诊断提示模板

```text
你正在知网页面上操作。如果找不到控件，请按页面可见文字寻找“检索/搜索”“导出/引用/参考文献”“题名/作者/来源/年份/被引”等区域。请仍以页面可见内容为准，不要绕过登录、验证码或机构认证，不要构造接口请求。
```
