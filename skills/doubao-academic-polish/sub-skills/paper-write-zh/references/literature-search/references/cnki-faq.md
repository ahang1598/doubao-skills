# 知网浏览器任务操作常见问题

本文件服务于 `literature-search` 参考。主流程必须优先触发豆包 browser-task/浏览器任务；仅在 browser-task 明确不可用、登录/验证码阻塞或用户明确要求手动时，才降级为用户手动浏览器操作。不要让主 Agent 手工复刻知网页面点击流程。

## 登录相关

### 为什么必须让用户登录？

知网通常允许未登录用户查看部分检索结果，但导出引用、查看详情或通过机构权限访问全文时，经常需要登录。登录、扫码、手机号验证码、机构统一认证和 CARSI 都属于用户身份行为，不能由 Agent 代填或绕过。注意：学校版、机构版、IP 登录或 CARSI 已认证页面可能仍保留“个人登录/登录”按钮；只要页面显示学校/机构访问状态，或检索与导出/引用入口可用，就不要把该按钮误判为未登录。

### 推荐提示语

```text
知网导出引用需要登录或机构认证。我已停止自动操作，并把浏览器停在需要处理的页面。请你在当前浏览器里完成扫码登录、手机号登录、机构/CARSI认证或验证码；我不会索要或代填账号、密码、短信码。完成后告诉我“已登录”，我再继续检索和导出引用。
```

### 浏览器任务应该做什么？

- 打开 `https://kns.cnki.net`。
- 判断页面是否已经具备检索和导出引用条件，而不是只看是否存在“登录”按钮。
- 如果页面显示学校/机构名称、机构已登录、IP 登录、CARSI/统一认证成功，或能看到结果列表和“导出/引用/参考文献”入口，继续检索和导出。
- 如果无法检索、无法打开导出/引用入口，或页面明确要求扫码、账号、验证码或机构认证，停在该页面，返回“需要用户登录/认证”，并提醒用户接管浏览器完成操作。
- 用户确认已登录后，继续检索和导出。

### 不应该做什么？

- 不索要用户账号、密码、短信验证码。
- 不保存 Cookie 或登录凭据。
- 不尝试绕过机构认证或验证码。

## 验证码相关

### 遇到滑块或拼图验证码怎么办？

停止自动流程，让用户在浏览器中手动完成。完成后继续 browser-task；只有 browser-task 不可用时才让用户回填结果。

### 如何减少触发？

- 检索和翻页动作不要过密。
- 批量导出前先确认候选列表，避免反复导出。
- 建议操作间隔 5-10 秒，导出失败后等待至少 10 秒再重试。
- 不连续翻大量页面；需要大规模综述时分批检索。

## 浏览器任务描述如何写？

好的任务描述应包含目标、站点、关键词、筛选规则、输出格式和人工介入边界。

示例：

```text
打开 https://kns.cnki.net，检查是否已登录。若未登录，停在登录页面并提示需要用户手动登录。登录后，搜索“数字经济 AND 企业创新”，优先返回近5年、CSSCI或北大核心、高相关性的中文文献。请列出前10条候选文献的题名、作者、来源、年份、被引次数和链接。不要下载全文，不要绕过验证码。
```

导出引用示例：

```text
在当前知网检索结果中，先勾选编号 1、3、5 的文献，再使用页面工具栏可见的“导出/引用/参考文献/批量导出”等入口一键导出 GB/T 7714 格式引用。返回引用原文，并附题名、作者、来源和年份用于核对。不要逐条抄元数据拼引用；如批量导出失败，请进入单篇详情页尝试导出；仍失败则说明失败原因。
```

## 检索相关

### 搜索结果为空

1. 将篇名检索改为主题检索。
2. 删除过窄的时间、来源或学科限制。
3. 增加同义词，用 OR 连接。
4. 拆分长句，只保留 2-3 个核心概念。

### 搜索结果太多

1. 增加限定词，用 AND 连接。
2. 限定近 5 年。
3. 优先核心期刊、CSSCI、北大核心、CSCD。
4. 从主题检索改为篇名或关键词检索。

### 如何选择文献？

优先级为：主题相关性 > 来源可靠性 > 近年性 > 被引量。不要只按被引量选文献；综述写作需要覆盖不同主题和研究路径。

## 导出相关

### 导出 API 或页面按钮失败

先让浏览器任务或用户在结果页勾选目标文献，使用页面工具栏的“导出/引用/参考文献/批量导出”等入口一键导出。批量导出失败后，再尝试单篇详情页导出。如果仍失败，输出失败状态和已获得的元数据，不要补写引用。

### 导出的 GB/T 7714 是否可直接使用？

需要核查。知网导出的引用有时存在页码、卷期、作者分隔符或文献类型错误。最终交付前至少检查作者、题名、来源、年份四项。

### 可以只拿摘要写综述吗？

只有 browser-task 从页面提取到可见摘要，或用户手动提供页面可见摘要时，才可做初步综述，并必须标注证据层级。`scholar_search`、普通搜索工具或 API 返回的题名、摘要和元数据不得直接写入综述正文，只能列为待核验候选；evidence map 的 `Risk` 应标注为 `needs-verification`，直到补足页面可见来源。

## 输出与保存

豆包默认不依赖持久文件。优先在对话中输出以下可复制产物，由用户按需保存：

- GB/T 7714 引用池。
- 证据到论点映射（evidence map）。
- 如浏览器任务或用户回填结构化导出数据，可在对话中保留原始字段摘要。

每条引用建议记录：检索词、获取日期、CNKI 页面链接或来源说明。
