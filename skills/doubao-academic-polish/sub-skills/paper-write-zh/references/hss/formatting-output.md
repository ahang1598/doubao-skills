# 排版与输出格式规范（含错误案例对照）

> 本文件针对**生成/修改论文时输出排版层面**的常见错误（尤以 AI 生成稿为甚）给出禁止项与正确做法。每条均配**错误案例**与**规则**。
> 适用场景：生成论文全文/章节、润色、以及"论文格式问题诊断"任务。与 `journal-formats.md`（期刊格式细则）、`submission-checklist.md`（自检清单）配合使用。
>
> **诊断任务铁律**：当本 HSS 参考用于"论文格式问题诊断"时，**每一个指出的问题都必须附原文证据（原文摘录）**；没有原文证据的问题不得列入正式问题清单。

---

## 一、公式格式

### 错误案例
- 纯文本下标：`NQP_{i,t} = α_0 + α_1 × ESG_{i,t} + ΣControls_{i,t} + ε_{i,t}`（`{i,t}`、`_0`、`_1` 以代码式文本显示）。
- HTML 标签：`Trade<sub>ijt</sub> = α<sub>0</sub> + α<sub>1</sub>Geo<sub>ijt</sub> + …`。
- 指数被拆行：`τ<sub>ijt</sub> = (…)` 与 `<sup>1/(2σ-2)</sup>` 分两行。
- 符号解释与公式不一致：公式里是 α₁、β₁、γ₁、γ₂，正文却写成"检验 γ 和 γ 的显著性"。

### 规则
1. **公式必须渲染为正式数学格式**：用公式编辑器/标准 LaTeX；**禁止**输出 `_{i,t}`、`β_0`、`<sub>…</sub>`、`<sup>…</sup>` 这类未渲染的代码式/HTML 痕迹。
2. **核心模型公式独立成行并编号**（式（1）、式（2）……），居中；不得把一个公式机械拆成多行用"+"结尾拼接，**上下标、幂指数、分式须完整渲染在同一公式结构内**。
3. **符号解释与公式严格一致**：公式中出现的每个系数（如 α₁、γ₂）在文字说明中须同名出现，**不得**把 γ₁、γ₂ 简写成重复的"γ 和 γ"。

## 二、表格格式

### 错误案例
- 指标体系被压成一段：`一级指标二级指标具体衡量指标指标属性 劳动者 劳动者素质 本科及以上学历员工占比 正向 …`。
- 描述性统计粘连：`NQP 28654 0.324 0.156 …` 写成 `NQP286540.3240.156…`，分不清观测值与数值。
- 相关性矩阵串成字符串、显著性星号被换行拆断：`Lev 0.089*** 0.123*` 与 `**0.345***` 拆到两行。
- 回归表混排：列名、变量名、系数、t 值连在一起，`观测值2865428654` 无法判断是 28654/28654 两列还是一个长数字。

### 规则
1. **结构化内容必须用表格**：凡出现"一级/二级/衡量指标、指标属性""描述性统计""相关性矩阵""回归结果"等，一律生成**标准表格（建议三线表）**，不得压成连续正文。
2. **表格语义清晰**：描述性统计每变量一行、每统计量一列；相关性输出为矩阵表，变量名作行列标题，**显著性星号紧跟对应系数、不得因换行被拆散**；回归表每个模型单独一列，变量、系数、括号内统计量、固定效应、样本量、R² 分行展示。
3. **数字不粘连**：各单元格数值之间须由表格单元分隔，禁止串成无法切分的长串。

## 三、章节与正文排版

### 错误案例
- 标题与正文混排：`（一）描述性统计表2报告了主要变量的描述性统计结果。`
- 表题/表注/下节标题粘连：`…0.267 表2 主要变量描述性统计（二）相关性分析 表3报告了…`。
- 变量小项连续粘连、层级不清：`…指标具有合理性。2. 解释变量：ESG表现（ESG）… 3. 中介变量（1）融资约束（SA）：…`。

### 规则
1. **标题独立成行**：`一、`/`（一）`/`1.`/`（1）` 等各级标题单独成行，**不得与正文第一句合并在同一段**。
2. **表格前后留白**：表格结束后另起段落；表题、表注与下一节标题不得与表格内容或彼此粘连。
3. **变量定义分层编号**：按"被解释变量 / 解释变量 / 机制（中介/调节）变量 / 控制变量"分类，每类单独成段、编号清晰，不与上一段正文粘连。

## 四、语言与中英文混排

### 错误案例
- `…地缘政治距离的增加会导致贸易政策更加 restrictive，进而抑制贸易。`（英文普通词直接夹在中文句中）

### 规则
1. **中文正文不得无必要夹杂英文普通词**（如 restrictive 应写"限制性更强/更趋收紧"）。
2. 确需使用英文专业术语/缩写时，**首次出现给出中文译名 + 英文括注**（如"环境、社会与治理（ESG）"），其后统一用一种表述。

## 五、参考文献格式

### 错误案例
- 多条挤在同一段：`…2024, 7(3): 1-12.[2] 殷德生… [3] …鲁朝云…`，`[1][2][3]` 未分行。
- 英文条目被硬换行拆断：英文题名、期刊名、页码在条目中间机械换行。

### 规则
1. **逐条换行，一条一段**：每条参考文献独立成段，编号、作者、题名、刊名、年份、卷期、页码结构清晰。
2. **条目内部语义连续**：换行只发生在条目之间；不得把英文题名、期刊名或页码在条目内部机械拆断。
3. 具体著录体例（顺序编码制/著者-出版年制等）见 `citation-styles.md`；条数上限等见 `journal-formats.md`。

## 六、非正式提示 / AI 生成痕迹残留

### 错误案例
- 正文中：`部分内容由豆包生成`。
- 文末：`（注：文档部分内容可能由 AI 生成）` / `（注：内容由 AI 生成，请谨慎参考）`。

### 规则
1. **正式论文不得保留任何生成工具提示**："由 XX 生成""请谨慎参考""部分内容由 AI 生成"等一律删除。
2. 若确需披露 AI 辅助，**按目标期刊规范**转写为规范化的"AI 辅助使用声明"，放入"声明/致谢"等指定位置，不得随意以括注形式散落在标题下或正文中。
3. 交付前**全文扫描**此类残留（含占位符、调试文本、Markdown/HTML 残留标记）并清除。

## 七、输出格式检查速查表（交付前过一遍）

| 检查模块 | 必须满足 | 禁止输出 |
|----------|----------|----------|
| 公式 | 独立成行、编号、上下标/指数完整渲染、符号解释与公式一致 | `NQP_{i,t}`、`β_0`、`<sub>…</sub>`、`<sup>…</sup>` 等未渲染痕迹；"γ 和 γ"式重复 |
| 表格 | 指标体系/描述统计/相关性/回归均为标准表格，语义清晰 | 把变量名、系数、t 值、样本量、R²、表题表注串成字符串 |
| 章节排版 | `一、`/`（一）`/`1.` 标题独立成行，编号层级清晰 | 标题与正文第一句粘连；表题/表注/下节标题粘连 |
| 语言 | 中文正文无必要英文词；术语首现给中文译名+英文括注 | 未翻译英文普通词直接夹在中文句中 |
| 参考文献 | 每条独立成段、条目内部完整 | `[1][2][3]` 挤在同段；英文题名/刊名/页码被硬换行拆断 |
| 正文清理 | 删除非正式 AI 提示；如需声明按期刊规范处理 | "部分内容由豆包生成""内容由 AI 生成，请谨慎参考"等残留 |
