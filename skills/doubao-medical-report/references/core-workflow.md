## 端到端工作流

## 1. 判断用户目标

先根据用户表达判断输出类型。默认目标是 `feishu_doc_report`：

- `feishu_doc_report`：默认输出，创建完整飞书云文档版个人健康管理报告。
- `analysis_summary`：仅当用户明确说“只要口头分析/不要创建文档”时使用。
- `update_report`：提供了新报告，需要更新已有健康管理报告；若用户给出飞书文档 URL，按 `lark-doc` 编辑流程更新。
- `doc_style_report`：强调文档样式、协作、分享或飞书阅读，本质仍是创建飞书云文档。
- `single_exam_report`：用户只提供单张检查单或单次检验报告时使用。不能因为缺少历史数据而省略详细解读；必须输出异常分组、逐项解读和单次指标可视化。

只有在资料缺失、患者/时间段无法区分、关键数值不确定，或用户要求确诊/治疗决策时，才需要先追问。

## 2. 接收资料

可处理的资料包括：

- 体检报告。
- 检验报告。
- 影像报告：超声、CT、MRI、X 光、钼靶等。
- 胃镜、肠镜和病理报告。
- 专科门诊病历。
- 出院小结。
- 用户提供的用药清单。
- 家庭监测或可穿戴数据：血压、血糖、体重、睡眠、运动等。

每份资料记录：

- 文件/页面/来源编号。
- 报告日期。
- 医院和科室。
- 检查类别。
- 原始文本或表格行。
- 抽取是否存在不确定性。

## 3. 资料清晰度与数值复核

不要调用识别工具强行读取不清晰报告。若资料中的数值、单位、参考范围、异常箭头或日期不清晰，不要猜测或补造。

每个数值进入证据表前，必须核对项目名、结果值、小数点、单位、参考范围、异常箭头和报告日期是否对应同一行或同一段原文。低置信度或未复核的数值不得用于图表、趋势、紧急风险判断或强医学结论，只能列入“无法确认项目”。

若不确定数值会影响异常解读、图表、趋势、高风险提示或就医优先级，必须先暂停生成报告，向用户发出数值确认清单。确认清单使用：

`项目 | 当前识别/读取值 | 不确定原因 | 请用户确认`

用户确认或补充清晰原件后，再继续生成报告。若用户明确表示无法确认，可跳过该数值，并在报告中说明该项未纳入解读和图表。

## 4. 先建立证据表

写报告前必须先形成内部证据表。最少包含：

`日期 | 类别 | 原始项目名 | 标准项目名 | 结果 | 单位 | 参考范围 | 异常标记 | 来源 | 原文 | 复核状态 | 置信度`

只有证据表中的内容可以进入医学解读、趋势分析、表格和管理建议。

## 5. 标准化与分组

按临床领域分组：

- 一般情况：年龄、性别、身高、体重、BMI、腰围、血压。
- 代谢：空腹血糖、糖化血红蛋白、胰岛素、尿酸。
- 血脂：总胆固醇、甘油三酯、LDL-C、HDL-C、apoA1、apoB、Lp(a)。
- 肝胆：ALT、AST、GGT、胆红素、脂肪肝、胆囊结石/息肉。
- 肾脏和尿液：肌酐、eGFR、尿素氮、尿蛋白、尿潜血。
- 心血管：心电图、心脏超声、颈动脉超声、ASCVD 相关因素。
- 甲状腺：TSH、FT3、FT4、抗体、结节、TI-RADS。
- 乳腺：超声、钼靶、BI-RADS。
- 肺部：CT 结节、炎症、肺气肿等。
- 消化：胃镜、肠镜、病理、幽门螺杆菌。
- 血液：血常规、贫血线索。
- 肿瘤标志物：必须谨慎解释，不能单项下结论。
- 生活方式：吸烟、饮酒、睡眠、运动、饮食、压力。

## 6. 健康问题分层

生成四类任务：

1. `suggest_emergency_or_urgent_review`
   - 原始报告明确提示需立即联系医生或立即处理。
   - 用户、报告或检验机构材料提供紧急阈值且结果命中。
   - 结果呈极端异常，并通过 `general_search` 或 `MedicalSearch` 核验符合权威立即处理标准。
   - 有急症症状或报告提示可能需要立即处理的情况。
   - 时间窗口：现在，立即联系报告出具机构/主管医生；如伴严重症状或无法及时联系医生，建议急诊或拨打 120。
2. `suggest_timely_medical_review`
   - 可能高风险的影像分级、可疑病理、严重实验室异常、明显症状、反复恶化项目。
   - 时间窗口：非急诊但需尽快，通常 24-72 小时内；相对稳定但仍需专科确认时可写 1 周内。
3. `suggest_follow_up_testing`
   - 轻中度异常，需要复查、专科评估或间隔影像随访。
   - 时间窗口：通常 1-4 周、1-3 个月或按原报告建议。
4. `suggest_long_term_management`
   - 代谢、心血管、体重、睡眠、运动、营养和生活方式问题。
   - 时间窗口：通常 1-3 个月建立管理计划，3-6 个月复盘。

每条任务必须包含：

- 问题是什么。
- 来源证据是什么。
- 为什么值得关注。
- 建议科室或复查方式。
- 合理时间窗口。
- 置信度和不确定性。

内部紧急风险定义与处理：

- 内部紧急风险用于判断哪些结果可能提示患者处于需要立即医学评估的状态；这是内部判断，不是用户可见标题。
- 不得仅凭模型记忆自行认定；优先依据原始报告立即处理标记、检验机构紧急阈值、用户明确提供的标准，或通过 `general_search`/`MedicalSearch` 核验的权威标准。
- 如果识别到内部紧急风险，报告顶部必须出现红色高亮 callout，用户可见标题写“情况紧急：请立即联系医生”，正文写明“现在立即联系报告出具机构/主管医生；如伴严重症状或联系不上医生，立即急诊或拨打 120”。
- 处置说明只提供就医/联系医生路径，不输出诊断、治疗方案、药物或处置方案。
- 如果没有识别到内部紧急风险，不输出任何“未识别到/未发现”的默认说明。

面向普通用户的表达要求：

- 每个健康问题必须先写“普通用户能看懂的一句话”，再列医学证据。
- 避免把专业术语直接作为主要问题标题，例如不要只写“白细胞系异常”“红细胞系异常”“代谢综合征风险”。
- 推荐写法：
  - “免疫防御相关指标偏低，需要尽快让医生确认原因。”
  - “携氧相关指标偏低，可能解释乏力、头晕等不适，需要复查和医生评估。”
  - “血脂相关指标偏高，长期可能增加心血管负担，需要生活方式管理和复查。”
- 每条问题用固定五列输出：`优先级 | 主要问题（一句话） | 这意味着什么 | 对应指标/证据 | 建议动作`。
- 医学术语放在“对应指标/证据”列，不放在“主要问题（一句话）”列。

## 7. 异常指标详细解读

使用 [medical-reasoning.md](medical-reasoning.md)。每个重点问题解释：

- 异常发现。
- 当前结果、单位、参考范围或报告分级。
- 常见临床意义。
- 结合用户数据为什么需要关注。
- 与其他指标或风险因素的关系。
- 哪些需要医生确认。
- 建议动作：限期就医、复查随访或长期管理。

必须优先生成“异常指标摘要表”和“分模块解读”。趋势分析只能补充说明异常变化，不能替代本节。

单张检查单要求：

- 按系统或指标类别分组，例如血常规可分为白细胞系、红细胞系、血小板系；肝肾功能可分为肝胆、肾功能、代谢相关。
- 异常总览必须写明异常项数量、偏高/偏低数量、主要受影响类别。
- 每个异常指标都要单独成段，包含 `当前结果`、`参考范围`、`异常程度`、`解读`、`预警信号`、`提醒/需医生确认`。
- 不输出疾病诊断或治疗方案；只能写“提示需要关注”“建议与医生确认”“可作为就诊沟通重点”。

## 8. 趋势分析

使用 [trend-analysis.md](trend-analysis.md)。对重复项目：

- 按日期排序。
- 统一单位。
- 对照参考范围。
- 判断趋势状态。
- 解释临床意义。

不要过度解读很小的波动。

如果所有证据都只有一个时间点：

- 不生成“重点指标趋势”模块。
- 不输出“趋势状态”“早期值/近期值”等趋势表字段。
- 改为生成“单次异常指标可视化”，展示异常分类分布、偏离程度、风险分层。
- 正文写明“当前资料只有一个时间点，不能判断趋势”。

## 9. 可视化与图表

使用 [visualization-and-charts.md](visualization-and-charts.md)。在数据适合时生成飞书图表助手卡片：

- 2 个及以上时间点的数值指标：趋势折线图。
- 3 个及以上异常项目：异常分布条形图。
- 明确低/中/高风险分层：风险分层图。
- 有复查时间窗口：复查计划时间轴或表格型可视化。

图表必须来自证据表中的结构化数据。影像、内镜、病理文字结论不得被伪造成数值图。每个图表都要配简短图表解读和不确定性说明。

趋势分析中只要存在 2 个及以上可靠时间点，必须运行 `scripts/prepare-health-chart-blocks.js evidence.json > chart-blocks.json` 自动生成飞书图表助手 block JSON，并在飞书文档创建后插入对应图表卡片；不能只写趋势表。

单张检查单没有历史趋势时，也必须运行同一脚本。只要存在异常数值指标，脚本会生成异常分类分布或偏离程度图；不能因为没有趋势而跳过图表。

如果脚本输出 `blocks` 非空但文档没有插入图表，任务视为未完成，不能回复“已生成完整报告”。必须继续插入图表，或明确说明图表助手插入失败的原因。

风险分层表必须用 `<td background-color="...">` 做递进颜色：低 `light-green`，中 `light-yellow`，较高 `light-orange`，高 `light-red`。

## 10. 长期管理方案

使用 [management-plan.md](management-plan.md)。覆盖：

- 医疗建议。
- 复查计划。
- 营养饮食。
- 运动健身。
- 睡眠压力。
- 家庭监测。
- 需提前就医的信号。

若管理建议涉及动作讲解、家庭监测方法、运动动作、饮食搭配或复查流程，必须调用 `image_generation` 生成非诊断性示意图，并插入相应章节。不得生成伪造医学影像、手术/侵入性操作或治疗承诺图。

## 11. 飞书文档生成

使用 [report-structure.md](report-structure.md)、[source-quality-and-links.md](source-quality-and-links.md) 和 [../assets/feishu-doc-template.xml](../assets/feishu-doc-template.xml) 生成飞书文档 XML 草稿。完整报告包括：

- 封面信息。
- 重要提示。
- 报告总览。
- 异常指标详细解读。
- 条件章节：
  - 有 2 个及以上时间点：重点指标趋势。
  - 只有 1 个时间点：单次异常指标可视化，不生成“重点指标趋势”。
- 风险与异常分布可视化。
- 综合健康分析。
- 疾病风险评估。
- 历年异常数据对比。
- 定制健康方案。
- 信息来源。
- 免责声明。

飞书文档生成要求：

- 默认使用 XML，不生成 HTML。
- 使用标题、callout、表格、分栏、任务清单等飞书文档结构增强可读性。
- 异常指标详细解读必须放在趋势章节之前，至少包含异常摘要表和分模块详解。
- 有趋势数据时，趋势可视化优先使用飞书图表助手卡片，辅以表格、箭头、状态标签和简短说明；不要依赖 HTML/SVG。
- 只有单时间点时，不生成“重点指标趋势”；把图表放入“单次异常指标可视化”或“风险与异常分布可视化”。
- 一级标题和关键提示可少量使用严肃 emoji，例如 `📈`、`⚠️`、`📅`、`🧪`，但不要过度装饰。
- 所有正文、表格、趋势、风险、管理方案、信息来源和免责声明均写入飞书文档。
- 如果数据缺失，表格写 `暂无`，正文说明限制，不要用占位符留空。
- 信息来源必须使用可点击链接；外部参考按国家级网站、权威指南/共识、教材、权威期刊文献、专业机构官网、医院官网的顺序选择。输出前检查链接可访问且内容与引用结论匹配。

## 12. 创建与交付

创建飞书文档前，按 `lark-doc` skill 要求读取认证和 XML 规则文件，然后使用：

```bash
lark-cli docs +create --api-version v2 --content '<title>个人健康管理报告</title>...'
```

创建后必须拿到飞书文档 URL。

若报告包含可视化候选数据，创建飞书文档后继续插入飞书图表助手卡片。图表助手卡片插入失败时，不要谎称成功；保留图表数据表并说明限制。

交付前使用 [qa-checklist.md](qa-checklist.md)，并对草稿运行：

```bash
node scripts/validate-feishu-doc-draft.js path/to/draft.xml
```

若草稿包含外部 URL，继续运行：

```bash
node scripts/check-reference-links.js path/to/draft.xml
```

链接不可访问、不可点击或内容不匹配时，替换来源或删除该引用后再交付。

最终回复必须包含：

- 生成完成的简短说明。
- 飞书文档可点击链接，格式为 `[个人健康管理报告](https://.../docx/...)`。
- 如有未能完成的数据项，用一句话说明限制。

不要只写“报告已生成”；用户必须能直接点击打开飞书文档。
