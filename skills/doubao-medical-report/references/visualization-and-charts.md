## 可视化与飞书图表助手

## 目标

在个人健康管理报告中加入严肃、克制、可解释的可视化。图表必须服务于医学理解、趋势判断和长期管理，不做装饰性图表。

## 何时必须生成图表

满足以下任一条件时，应生成图表或图表助手卡片：

- 同一数值指标有 2 个及以上日期点：生成趋势折线图。
- 同一系统有 3 个及以上异常项目：生成异常项目条形图或风险分布图。
- 有明确低/中/高风险分层：生成风险分层条形图或矩阵图。
- 有复查计划和时间窗口：生成复查计划时间轴或表格型可视化。
- 有体重、BMI、血压、血脂、血糖、尿酸、肝肾功能等长期管理指标：优先生成趋势图。

对于趋势分析，只要存在 2 个及以上可靠时间点，不能只写趋势表或文字说明。必须生成飞书图表助手卡片；若插入失败，必须写明失败原因并保留图表数据表。

不生成图表的情况：

- 只有单次、单项文本结论，且没有可比较数值。
- 影像、病理、内镜报告只有文字描述，不能被安全数值化。
- 原始数据不清晰，或日期、单位、参考范围无法确认。
- 图表可能造成确诊暗示或夸大风险。

## 推荐图表

| 场景 | 图表类型 | 字段 | 说明 |
|---|---|---|---|
| 历年数值指标 | line | 日期、指标值、参考上限/下限 | 适合 LDL-C、血糖、尿酸、BMI、血压等 |
| 多个异常项目 | horizontalBar | 异常项目、偏离程度或优先级分 | 适合展示本次异常构成 |
| 风险分层 | bar 或 heatmap | 风险类型、等级、证据数量 | 适合代谢、心血管、肾脏等风险 |
| 复查计划 | bar 或 table | 时间窗口、项目、优先级 | 适合 1个月/3个月/6个月/年度计划 |
| 分类占比 | pie | 管理类别、数量 | 只在类别少于 6 个且占比有意义时使用 |

## 飞书图表助手卡片

参考 `feishu-chart-assistant-skill` 的图表助手结构。插入飞书文档时，优先使用 Docx add-on block：

```json
{
  "block_type": 40,
  "add_ons": {
    "component_type_id": "blk_64df3b277a87c002dafdc52b",
    "record": "{...chartAssistantRecordJson...}"
  }
}
```

其中 `record` 使用图表助手标准结构：

```json
{
  "theme": "fresh-blue-green-health",
  "elements": [
    {
      "type": "chart",
      "position": { "x": 0, "y": 0, "width": 600, "height": 360 },
      "options": {
        "chartType": "line",
        "data": {
          "type": "standard",
          "value": {
            "columns": ["日期", "指标值", "参考上限"],
            "data": [
              { "日期": "2024-06-01", "指标值": 2.92, "参考上限": 3.4 },
              { "日期": "2026-06-01", "指标值": 3.68, "参考上限": 3.4 }
            ]
          }
        },
        "enableDataEdit": true
      }
    }
  ]
}
```

可使用脚本生成图表助手 block JSON：

```bash
node scripts/build-health-chart-blocks.js charts.json > chart-blocks.json
```

更推荐使用证据表自动触发脚本：

```bash
node scripts/prepare-health-chart-blocks.js evidence.json > chart-blocks.json
```

`prepare-health-chart-blocks.js` 会自动扫描证据表中同一指标的多个时间点，生成趋势折线图。报告生成流程必须优先运行这个脚本，而不是依赖手写 `charts.json`。

`charts.json` 示例：

```json
{
  "charts": [
    {
      "title": "LDL-C 历年趋势",
      "chartType": "line",
      "columns": ["日期", "LDL-C", "参考上限"],
      "data": [
        { "日期": "2024-06-01", "LDL-C": 2.92, "参考上限": 3.4 },
        { "日期": "2026-06-01", "LDL-C": 3.68, "参考上限": 3.4 }
      ],
      "insight": "图表解读：LDL-C 较前升高，已接近或超过部分风险人群管理目标，建议结合心血管风险复查评估。"
    }
  ]
}
```

生成后必须把 `block_type: 40` 的 add-on block 插入飞书文档对应章节。草稿复核时，至少应能看到 `block_type: 40`、`add_ons` 或明确的“图表助手卡片未能插入”回退说明。

## 风险分层表底色

风险分层表必须使用单元格背景色展示递进：

| 风险等级 | 单元格背景色 | 语义 |
|---|---|---|
| 低 | `light-green` | 当前以观察或常规管理为主 |
| 中 | `light-yellow` | 需要复查或强化生活方式管理 |
| 较高 | `orange` 或 `light-orange` | 建议限期专科评估 |
| 高 | `light-red` | 需要优先处理或尽快就医确认 |

示例：

```xml
<tr>
  <td>心血管风险</td>
  <td background-color="light-orange">较高</td>
  <td>LDL-C 升高 + 血压偏高</td>
  <td>建议复查血脂并评估 ASCVD 风险</td>
</tr>
```

## 图表生成流程

1. 从证据表中筛选可视化候选：日期、指标、数值、单位、参考范围、异常标记。
2. 去除低置信度、单位不统一、日期不明的数据。
3. 运行 `scripts/prepare-health-chart-blocks.js evidence.json > chart-blocks.json` 自动生成趋势图 block。
4. 需要补充非趋势图时，再手写 `charts.json` 并运行 `scripts/build-health-chart-blocks.js`。
5. 选择 2-5 个最能帮助理解的图表，不要堆满整篇报告。
6. 每个图表必须配一段 `图表解读`：说明看到了什么、管理含义是什么、哪些不能过度解读。
7. 图表插入失败时，在飞书文档中保留图表数据表，并写明“图表助手卡片未能插入，本节保留为图表数据表”。

## 动作示意图

当建议中包含需要用户理解动作或步骤的内容时，应调用 `image_generation` 生成非诊断性示意图，并插入飞书文档：

- 适合生成：居家血压测量姿势、运动动作示范、餐盘搭配、复查流程、睡眠环境调整、家庭监测记录方式。
- 不生成：医学影像、病理图、手术步骤、注射/穿刺/侵入性操作、药品外观伪图、医生诊断场景。
- 风格：清新蓝绿色、医学科普插画、无真实人脸、无品牌、无医院 logo。
- 每张图必须配一句用途说明，例如“示意图用于帮助理解居家血压测量姿势，不作为诊疗操作指导”。

推荐 prompt 模板：

`清新蓝绿色医学科普插画，展示{{动作/场景}}，画面严肃克制，步骤清晰，适合个人健康管理报告，无品牌无真实人脸，不包含诊断或治疗承诺。`

## 严肃 emoji 规则

可以少量使用 emoji 做章节锚点和风险提示：

- `🩺` 医学证据、医生确认。
- `📈` 趋势和图表。
- `⚠️` 重要异常或需限期就医。
- `📅` 复查计划。
- `🧪` 检验指标。
- `❤️` 心血管风险。
- `🥗` 营养饮食。
- `🏃` 运动管理。
- `🌙` 睡眠和生活方式。

使用限制：

- 每个一级章节标题最多 1 个 emoji。
- 不在具体疾病名称前使用夸张 emoji。
- 不使用庆祝、玩笑、恐吓或营销感 emoji。
- 高风险提示使用 `⚠️` 时，语气保持医学谨慎，不制造恐慌。
